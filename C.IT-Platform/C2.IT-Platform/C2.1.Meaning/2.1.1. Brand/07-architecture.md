# 07. Brand Architecture

> **Статус:** draft — Ф7 WP-292
> **Ключевые развилки:** (1) имя продукта выбрано (→ 02-naming.md), (2) схема модулей — единый Suite или самостоятельные имена, (3) «IWE» в публичном имени — включать или нет.
> **Принцип:** IWE — родовое имя категории (не бренд). Аналогия: «персональный компьютер» → «MacBook».

---

## 1. Уровни именования

| Уровень | Аналогия | Текущее | Роль в коммуникации |
|---------|---------|---------|---------------------|
| **Тип продукта (родовое)** | «Персональный компьютер» | **IWE** (Intellectual Work Environment) | Категория. Используется в объяснении, не как бренд |
| **Имя продукта (собственное)** | «MacBook» | **TBD** (кандидат из шорт-листа 02-naming.md) | Главное публичное имя. Стоит в логотипе, в рекламе, в App Store |
| **Пакет документов о бренде** | «Brand Book» | **Brand Compendium** | Внутреннее имя этого набора артефактов |

**Правило:** В публичной коммуникации — **собственное имя + уточнение:**
- «[Имя] — ваш IWE»
- «[Имя] — Intellectual Work Environment для созидателей»
- «[Name] is your IWE — Intellectual Work Environment» (EN)

**Запрещено:** называть продукт «IWE» как брендом без собственного имени. «IWE» = тип продукта, не имя.

---

## 2. Структура: продукт как единое целое

### Основная сборка

```
[Имя продукта]
├── Core (ядро среды)
│   ├── Персональный IWE (ЦД + память + протоколы)
│   ├── Методология (FPF → встроена в workflow)
│   └── MCP-слой (интеграция с AI-клиентами)
├── Atelier (мастерская)
│   ├── Инструменты создания (написание, разбор, планирование)
│   ├── Скиллы и протоколы (Day Open, ОРЗ, АрхГейт)
│   └── Pack (персональная база знаний)
└── Community (сообщество)
    ├── Доступ к сообществу созидателей
    ├── Наставники и peer review
    └── Программы развития (ЛР, РР)
```

### Принцип именования компонентов

**На этапе до выбора имени продукта:** компоненты называются по функции («Персональный IWE», «База знаний», «Протоколы работы»).

**После выбора имени продукта:** компоненты могут получить составные имена:

| Компонент | Шаблон | Пример (если имя = «Noema») |
|-----------|--------|---------------------------|
| Ядро среды | [Имя] Core | Noema Core |
| Мастерская | [Имя] Atelier | Noema Atelier |
| Полная сборка | [Имя] Suite | Noema Suite |
| Сообщество | [Имя] Community | Noema Community |

**Открытый вопрос:** нужны ли публичные имена компонентов — или один продукт без разбивки? Решается после MVP и первой когорты пользователей. Риск дробления: пользователь путается, что включено в подписку.

## 3. Иерархия версионирования

### Версии продукта

| Тип | Формат | Пример |
|-----|--------|--------|
| Мажорная | [Имя] 2 | Noema 2 |
| Минорная | внутренний (не публичный) | 2.3.1 |
| Preview / Beta | [Имя] Preview | Noema Preview |

**Принцип:** публичные мажорные версии — только при существенных изменениях онтологии продукта (новый уровень методологии, новый тип пользователя). Технические версии — не в коммуникации.

### Трек продукта

| Трек | Аудитория | Имя |
|------|-----------|-----|
| **Personal** | Созидатель-одиночка | [Имя] Personal |
| **Team** | Команда 2-10 | [Имя] Team |
| **Professional** | Организация | [Имя] Professional |

**Порядок вывода:** сначала Personal (2026) → Team (2027) → Professional (2028+). Не выводить имена треков раньше, чем трек готов к рынку — не создавать ожиданий.

## 4. Отношение IWE ↔ Aisystant

> Это разграничение критично для бренд-архитектуры: два разных уровня.

| Сущность | Что это | Публичная роль |
|----------|---------|---------------|
| **Aisystant** | Платформа (общая инфраструктура, сообщество, методология) | B2B / партнёры / институциональный уровень |
| **[Имя] IWE** | Персональная рабочая среда для созидателя — продукт поверх Aisystant | B2C / созидатель / конечный пользователь |

**Принцип коммуникации:**
- В адрес конечного пользователя: «[Имя] — ваш IWE». Aisystant не упоминается.
- В адрес инвесторов и партнёров: «[Имя] — флагманский продукт платформы Aisystant».
- В адрес YC/EN: «[Name] is built on Aisystant — the platform for intellectual work. [Name] is the personal environment for individual creators.»

**Запрещено:** называть Aisystant «платформой» в контексте персонального IWE — это создаёт ассоциацию с B2B корпоративным инструментом.
