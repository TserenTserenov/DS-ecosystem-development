# B3.1 PII Field Map

> Карта полей персональных данных во всех сервисах IWE. Обновлять при добавлении новых таблиц.
> Источник: аудит миграций + анализ кода (14 апр 2026).

## Карта таблиц с PII

| Таблица | База/Сервис | PII поля | Назначение | Срок хранения | Правовое основание |
|---------|------------|---------|-----------|---------------|-------------------|
| **users** | aist_bot / Neon platform | `ory_id`, `telegram_id`, `first_name`, `last_name` | Identity + Telegram binding | До удаления аккаунта | Договор + Согласие |
| **user_integrations** | activity_hub / Neon | `user_uuid`, `access_token`, `refresh_token`, `metadata` (GitHub username, WakaTime user_id) | OAuth credentials | Пока активна интеграция | OAuth scope grant |
| **github_installations** | knowledge-mcp / Neon | `user_id` (Ory), `github_user_id`, `github_username`, `installation_id`, `repos` | GitHub App binding + репо | Пока установлено приложение | GitHub App install |
| **user_sources** | knowledge-mcp / Neon | `user_id`, `source`, `github_owner`, `github_repo` | Маппинг личных репо | Indefinite (per-user) | Личные данные пользователя |
| **documents** | knowledge-mcp / Neon | `user_id` (nullable), `content` (может содержать PII если пользователь загружает личные документы) | База знаний L2 + L4 | По TTL документа | DP.SC.012 (подписка) |
| **learner_concept_mastery** | concept_graph / Neon | `user_id`, `concept_id`, `mastery_score` | Прогресс обучения | 1–2 года (когорта) | Договор (образовательные данные) |
| **user_state** | aist_bot / Neon | `user_id`, `current_context` (JSONB — может содержать историю консультаций) | FSM state + контекст сессии | На время сессии | Договор |
| **digital_twins** | aist_bot / Neon | `user_id` (Ory UUID), `data` (JSONB: метрики, профиль, квалификация) | Агрегированный профиль (ЦД) | Indefinite (backbone) | Договор + DPA |
| **user_events** | activity_hub / Neon development | `user_uuid`, `source`, `event_type`, `external_id`, `payload` | Event stream для метрик / ЦД-синк | 90 дней rolling | Телеметрия (consent) |
| **oauth_tokens, github_connections, google_calendar_connections, dt_tokens, ory_tokens** | aist_bot / Neon | `chat_id` (= telegram_id), `access_token`, `refresh_token` | OAuth интеграции | До отзыва | OAuth scope grant |
| **finance_payments** | aist_bot / Neon | `telegram_id`, `status`, `amount` | Транзакции | 3–7 лет (финансовые) | Налоговое законодательство |

## Ключевые риски

### HIGH: OAuth токены в plaintext
- Таблицы: `user_integrations`, `github_connections`, `google_calendar_connections`, `dt_tokens`, `ory_tokens`
- Риск: утечка БД = все токены в открытом виде
- Митигация: RLS + Neon encryption-at-rest (managed). Долгосрочно: B2.5 column-level encryption.

### MEDIUM: Личные документы в OpenAI embeddings
- Таблица: `documents` (user_id IS NOT NULL)
- Риск: пользователь загружает личные документы → текст отправляется в OpenAI для embedding
- Митигация: RLS изоляция ✅. Пользователь контролирует через GitHub repo privacy.
- Open question: OpenAI embeddings нельзя "забыть" после удаления документа → требует решения для GDPR erasure.

### MEDIUM: System prompt с профилем пользователя
- Сервис: gateway-mcp → Claude API
- Риск: IWE_SYSTEM_PROMPT может содержать имя/роль/цели пользователя
- Текущий статус: платформенный промпт, не содержит user context ✅
- Рекомендация: аудит при каждом изменении system prompt.
