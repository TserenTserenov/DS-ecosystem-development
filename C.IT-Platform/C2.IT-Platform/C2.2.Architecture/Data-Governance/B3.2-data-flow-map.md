# B3.2 External Data Flow Map

> Карта потоков данных наружу — что IWE отправляет внешним сервисам.
> Источник: анализ кода knowledge-mcp, personal-knowledge-mcp, gateway-mcp, activity-hub (14 апр 2026).

## Карта потоков

| Назначение | Направление | Данные | Что именно | Минимизация возможна? |
|-----------|------------|--------|-----------|----------------------|
| **OpenAI API** (embeddings) | OUT | Текст документов | Контент базы знаний (L2 платформенные + L4 личные документы пользователя) | Частичная: L4 личные docs контролирует пользователь через GitHub repo privacy |
| **OpenAI API** (reranking gpt-4o-mini) | OUT | Поисковые запросы + 20 сниппетов | Запрос пользователя + фрагменты документов для relevance scoring | Да: ограничить размер сниппетов, убрать метаданные |
| **OpenAI API** (concept analysis) | OUT | Ответы пользователей | Текст ответа на тест/задание для анализа концепций | Да: sanitize — удалять имена/контакты перед отправкой |
| **Anthropic Claude API** | OUT | Контекст пользователя | Системный промпт IWE + контекст из ЦД (роль, цели, если добавляются) | Да: минимизировать личные данные в system prompt, использовать role codes |
| **Ory Hydra** (auth.system-school.ru) | OUT | JWT токен | Верификация подписи, получение `sub` (ory_id) | Inherent: Ory = identity provider. Получаем только `sub`, не email. |
| **Neon PostgreSQL** | BOTH | Все PII | Все таблицы (см. B3.1) | Mitigation: RLS ✅, encryption-at-rest managed ✅, TLS ✅ |
| **GitHub API** | OUT | Репо контент + метаданные | Индексация документов для knowledge-mcp, создание репо, webhook events | Нет: пользователь явно авторизует через OAuth |
| **Aisystant LMS API** | IN | Профиль пользователя | LMS user_id, прогресс курсов, оценки → digital_twins | Нет: LMS = source-of-truth для квалификации |
| **WakaTime API** | IN | Метрики активности | Агрегированное время кодирования (без кода) | Уже минимально: только stats, не source code |
| **Langfuse** (опционально) | OUT | Трейсы LLM | Полный контекст Claude API calls (включая PII если есть в промпте) | Да: отключить если LANGFUSE_SECRET_KEY не задан → трейсы пропускаются |

## Матрица рисков

| Направление | Уровень риска | Обоснование |
|------------|--------------|-------------|
| → OpenAI embeddings (L4 личные docs) | **высокий** | Личные документы пользователя уходят в OpenAI. Нельзя "забыть" после удаления. |
| → Claude API (system prompt с профилем) | **средний** | Зависит от содержимого system prompt. Текущий — платформенный, без PII ✅ |
| → Langfuse (трейсы) | **средний** | Если включён — полный контекст промптов включая user data |
| ↔ Neon (DB) | **управляемый** | RLS + TLS + managed encryption. Основной контроль — B2.5, B9.x |
| → Ory (auth) | **низкий** | Только JWT verify, получаем `sub` |
| ← LMS (входящий) | **низкий** | Pull-only, внутри Aisystant network |
