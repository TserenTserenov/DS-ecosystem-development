# Множители и ограничения системы баллов

> **Source-of-truth (обещание):** SC.105 §3 — Три оси множителей.
> Здесь — детализация для реализации. Конкретные значения — **предварительные, калибровать при WP-121 Ф0**.

---

## 1. Формула начисления

```
effective_points = min(Base × ActionType × Streak × Qualification, daily_cap)
```

Где:
- **Base** — базовые баллы за event_type (из point_rules)
- **ActionType** — множитель по категории действия
- **Streak** — множитель за непрерывность (дни подряд)
- **Qualification** — множитель по стадии развития из ЦД
- **daily_cap** — верхний предел баллов за день (по квалификации)

---

## 2. Три оси множителей

### 2.1. По типу действия (ActionType)

| Категория | Примеры event_type | Множитель | Обоснование |
|-----------|-------------------|-----------|-------------|
| Систематичность | day_open, day_close | Условие (см. §3) | Дисциплина — не бонус, а условие |
| Время | coding_time, commit_created | ×1 | Базовый вклад |
| РП в физическом мире | wp_completed, content_published | ×2 | Результат, а не процесс |
| Качество | knowledge_extracted, distinction_added, method_described | ×3 | Вклад в общее знание |
| Развитие платформы | fmt_commit_merged, community_help, peer_review | ×5 | Вклад в сообщество |

### 2.2. Streak-множитель (удержание)

| День подряд | Множитель |
|-------------|-----------|
| 1 | ×1.0 |
| 2 | ×1.07 |
| 3 | ×1.14 |
| 4 | ×1.21 |
| 5 | ×1.29 |
| 6 | ×1.36 |
| 7+ | ×1.5 (максимум) |
| Пропуск дня | Сброс к ×1.0 |

Рост линейный: +0.07/день до максимума ×1.5 на 7-й день. Удержание ×1.5 на 8-30 день. Пропуск — полный сброс.

### 2.3. По квалификации (стадия развития из ЦД)

| Стадия | Множитель | Daily cap |
|--------|-----------|-----------|
| Random (0) | ×1.0 | 50 баллов/день |
| Practicing (1) | ×1.2 | 80 баллов/день |
| Systematic (2) | ×1.5 | 120 баллов/день |
| Disciplined (3) | ×2.0 | 200 баллов/день |
| Proactive (4) | ×2.5 | 300 баллов/день |

**Квалификация ≠ тир подписки** (DP.D.047). Тир (T0-T4) = уровень доступа (оплата). Квалификация = стадия развития из ЦД (поведение). Участник T4 с квалификацией Random получает множитель ×1.0 и cap 50/день.

## 3. Систематичность как условие

Day Open / Day Close **не дают бонусных баллов**. Вместо этого:

- День **с** Day Open + Day Close → остальные баллы начисляются по формуле
- День **без** Day Open/Close → множитель остальных баллов **×0.5**

Стимулирует дисциплину: не «сделай Day Open ради баллов», а «без Day Open твой вклад ценится вдвое меньше».

> Точное поведение (×0.5 vs ×0 vs только для определённых категорий) — калибровать при реализации.

## 4. Видимость баллов vs участие в скидках

Баллы начисляются **без ограничения** — каждое действие генерирует транзакцию. Пользователь видит **полную сумму** через `/points`.

Но в расчёте скидок участвуют **только баллы в пределах daily cap**. Баллы сверх cap'а:
- **Начисляются** — видны в истории, учитываются в `total_earned`
- **Не участвуют в скидках** — при расчёте скидки учитываются только баллы в пределах дневного лимита

> Пример: Practicing (cap 80/день). Участник с 5-дневным streak (×1.3) публикует качественный контент (×3): Base 10 × 3 × 1.3 × 1.2 = 46.8 → 47 баллов. Если за день набралось 120 — в скидку идут 80. Остальные 40 — «сверхлимитные», отображаются отдельно.

## 5. Anti-abuse ограничения

### 5.1. Cap по одному действию за день

Ограничение по количеству учитываемых событий одного типа за день. Хранится в `point_rules.max_per_day`.

| event_type | max_per_day | Обоснование |
|-----------|-------------|-------------|
| commit_created | 10 | Нельзя спамить пустыми коммитами |
| topic_created | 5 | Нельзя спамить темами |
| comment_posted | 20 | Разумный лимит обсуждения |
| test_passed | Без лимита | Сколько прошёл — столько начислено |

### 5.2. Readiness Gate

Баллы не начисляются, пока данные не подтверждены как полные (WP-109 Ф4). Предотвращает начисление по неполным данным.

### 5.3. Epistemic status как вес

Из SC.105: вклад взвешивается по epistemic status.

| Статус | Вес | Пример |
|--------|-----|--------|
| Предположение | ×0.5 | Гипотеза в обсуждении |
| Воспроизведено | ×1.0 | Применил метод, получил результат |
| Peer-reviewed | ×1.5 | Прошёл ревью сообщества |
| В production | ×2.0 | Используется платформой |

> Реализация epistemic status как множителя — Ф2 WP-121, после базовой системы.

## 6. Конвертация: баллы → скидки

- 1 балл ≈ $0.01 (или эквивалент в рублях по курсу)
- Применяется к: практикумам, семинарам, резидентурам, менторским сессиям, приоритетной ИИ-поддержке
- Скидка оформляется **в юрлице покупки** (РФ или мир)
