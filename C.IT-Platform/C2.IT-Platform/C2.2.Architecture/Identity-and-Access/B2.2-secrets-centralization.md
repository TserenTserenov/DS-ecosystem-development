# B2.2 Secrets Centralization

> Единая модель хранения секретов: где что живёт, как читается, как обновляется.
> Источник: B2.1 Secrets Inventory.

## Модель хранения (целевая)

### Уровень 1: Runtime secrets — Cloudflare Workers

**Что:** DATABASE_URL, OPENAI_API_KEY, ORY_URL, ORY_CLIENT_SECRET, GITHUB_APP_PRIVATE_KEY, и др.

**Где:** `wrangler secret put <NAME>` per-worker. Не в коде, не в wrangler.toml [vars].

**Правило:** если секрет есть в `[vars]` (plaintext в toml) → перенести в `wrangler secret put`.
Текущее нарушение: `CORS_ORIGIN = "*"` в digital-twin-mcp — не секрет, OK.

**Обновление:**
```bash
echo "NEW_VALUE" | wrangler secret put SECRET_NAME
# Применяется без редеплоя. Проверить через wrangler tail.
```

### Уровень 2: CI/CD secrets — GitHub Actions

**Что:** NEON_DIRECTUS_URL, AISYSTANT_PG_URL, NEON_AIST_BOT_URL, BOT_HEALTH_URL.

**Где:** GitHub repo/org → Settings → Secrets → Actions.

**Правило:** Org-level secrets для общих (DB URLs используемые несколькими репо). Repo-level для специфичных.

### Уровень 3: Локальные secrets — `.secrets/`

**Что:** LINEAR_API_KEY, RAILWAY_API_TOKEN, OPENAI_API_KEY (local copy), AISYSTANT_TECH_PASSWORD, GCP_OAUTH_KEYS, N8N_API_KEY.

**Где:** `/Users/tserentserenov/IWE/.secrets/` — plaintext файлы, НЕ в git (IWE не git-репо).

**Структура (текущая):**
```
.secrets/
├── linear-api-key          # читается mcp-linear.sh
├── railway-api-token       # читается mcp-railway.sh
├── openai-api-key          # dev copy
├── aisystant-session-token # time-bound, auto-expire
├── aisystant-tech-password # CRITICAL — shared account
├── gcp-oauth.keys.json     # CRITICAL — private key material
├── gcp-oauth-web.keys.json # CRITICAL — private key material
├── n8n-api-key.txt
└── sync-to-1password.sh    # sync script (статус: неизвестен)
```

**Правило:** `.secrets/` = последний рубеж. Для каждого файла — один потребитель. Перед добавлением нового секрета — проверить, нет ли его уже в `.secrets/`.

### Уровень 4: Сервисные секреты — Railway

**Что:** aist_bot env vars (BOT_TOKEN, DATABASE_URL, ORY_URL, и др.)

**Где:** Railway → Project → Variables.

**Правило:** не дублировать в коде. Все bot-level секреты через Railway Variables.

## Анти-паттерны (запрещено)

| Паттерн | Почему запрещено |
|---------|-----------------|
| Секрет в `[vars]` wrangler.toml | Plaintext в git-репо |
| Секрет в `.env` файле в git-репо | История git не очищается без BFG |
| Секрет в коде (`const KEY = "sk-..."`) | Hardcode — ротация = code push + deploy |
| Секрет в MCP config (`.mcp.json`) | Файл может попасть в git или логи |
| Секрет в MEMORY.md или `.claude/` | Попадает в контекст агента = логи Anthropic |
