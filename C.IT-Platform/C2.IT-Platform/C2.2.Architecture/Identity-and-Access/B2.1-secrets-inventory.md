## Проблемы

### Захардкоженные секреты
| Секрет | Файл | Риск |
|--------|------|------|
| FSM_OPENAI_CHALLENGE_TOKEN | `DS-MCP/fsm-mcp/src/index.ts` | Низкий (публичный challenge), но ротация требует code push |

### Отсутствует политика ротации
- Нет централизованного расписания ротации
- Нет таблицы "последняя ротация" для каждого секрета
- Локальные `.secrets/` файлы: нет автоматического синка с vault

### Рекомендации по ротации (по приоритету)

**Немедленно (если есть подозрение на утечку):**
- GITHUB_APP_PRIVATE_KEY — RSA key, широкий blast radius
- GCP_OAUTH_KEYS — private key material
- AISYSTANT_TECH_PASSWORD — shared account, высокий blast radius

**Ближайший квартал:**
- Все HIGH и CRITICAL: квартальная ротация
- Ввести Google Sheet / Notion table "Secret Rotation Log"

**Среднесрочно:**
- FSM_OPENAI_CHALLENGE_TOKEN → перенести в env var
- Локальные `.secrets/` → синк с 1Password (скрипт `sync-to-1password.sh` уже есть)

## Критический путь (blast radius)

**Если утёк DATABASE_URL:**
→ Затрагивает: knowledge-mcp, digital-twin-mcp, personal-knowledge-mcp, guides-mcp, gateway-mcp, activity-hub, payment-registry
→ Воздействие: все пользовательские данные, auth state, подписки

**Если утёк GITHUB_APP_PRIVATE_KEY:**
→ Затрагивает: gateway-mcp, personal-knowledge-mcp
→ Воздействие: несанкционированное создание репо, подмена knowledge tree

**Если утёк AISYSTANT_PG_URL:**
→ Затрагивает: payment-registry workflows
→ Воздействие: обход подписки, манипуляция платёжным состоянием
