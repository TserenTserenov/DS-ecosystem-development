# B2.3 Secrets Rotation Policy

> Политика ротации секретов экосистемы IWE. Источник истины: B2.1 Secrets Inventory.
> **Принцип:** ротация ДО истечения, не после инцидента.

## Расписание ротации

| Класс | Частота | Триггер | Ответственный |
|-------|---------|---------|--------------|
| **Критические** (RSA keys, service account passwords, DB credentials) | Квартально (каждые 90 дней) | Плановая + при смене команды | Владелец репо |
| **Высокие** (API keys, OAuth secrets, webhook secrets) | Каждые 6 месяцев | Плановая + при подозрении на утечку | Владелец сервиса |
| **Средние** (endpoint URLs, client IDs) | Ежегодно или при смене инфраструктуры | Плановая | Владелец сервиса |
| **Немедленно** — любой класс | При увольнении/смене доступа члена команды | Событийная | Всегда немедленно |
| **Немедленно** — любой класс | При подозрении на утечку (публичный коммит, подозрительная активность) | Событийная | Всегда немедленно |

## Процедура ротации по типу хранения

### Cloudflare Workers secrets
```bash
# Обновить секрет в Cloudflare
cd DS-MCP/<service>
echo "NEW_SECRET_VALUE" | wrangler secret put SECRET_NAME

# Проверить деплой
wrangler tail  # убедиться что новые запросы успешны
```
**Downtime:** нет (секреты обновляются без редеплоя воркера)
**Rollback:** повторить с предыдущим значением

### GitHub Actions secrets
```
GitHub UI → Settings → Secrets and variables → Actions → Update secret
```
**Downtime:** нет (применяется к следующему workflow run)
**Rollback:** обновить значение обратно

### Локальные `.secrets/` файлы
```bash
# Обновить файл
echo "NEW_VALUE" > ~/.IWE/.secrets/<secret-name>
# или через 1Password sync
.secrets/sync-to-1password.sh
```
**Downtime:** нет (читается при старте процесса)

### GITHUB_APP_PRIVATE_KEY (RSA)
```bash
# 1. GitHub → Settings → Developer settings → GitHub Apps → <app> → Generate new private key
# 2. Скачать PEM файл
# 3. Обновить в Cloudflare (gateway-mcp + personal-knowledge-mcp):
cat new-key.pem | wrangler secret put GITHUB_APP_PRIVATE_KEY
# 4. Удалить старый ключ в GitHub App settings
# 5. Проверить create_repository flow
```
**Координация:** одновременно обновить в gateway-mcp И personal-knowledge-mcp

### DATABASE_URL (Neon)
```bash
# 1. Neon console → Branches → Reset password (генерирует новый CONNECTION_STRING)
# 2. Обновить во ВСЕХ сервисах:
for svc in knowledge-mcp digital-twin-mcp personal-knowledge-mcp guides-mcp gateway-mcp; do
  echo "NEW_DB_URL" | wrangler secret put DATABASE_URL -c DS-MCP/$svc/wrangler.toml
done
# 3. Обновить в GitHub Actions (payment-registry, activity-hub)
# 4. Обновить в Railway (aist_bot)
# 5. Smoke test каждого сервиса
```
**Downtime риск:** да — нужна координация. Обновлять все сервисы за одну сессию (~15 мин).

## Признаки утечки (мониторинг)

- GitHub secret scanning alert → немедленная ротация
- Необъяснённые API calls в логах (Cloudflare, Railway)
- Dependabot security advisory на пакет, работающий с secrets
- Смена или увольнение члена команды с доступом
