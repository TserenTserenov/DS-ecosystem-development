# Концепция разделения платформы IWE: Россия и мир

> **Для кого:** архитектор, владелец продукта, инвестор — любой, кто хочет понять, как IWE будет работать в двух юрисдикциях одновременно.
>
> **Детали:** [01 — Экономика](01-economic-model.md) · [02 — Инфраструктура](02-infrastructure-and-services.md) · [03 — Сообщество](03-community-and-culture.md) · [04 — Исследование](04-research-patterns.md)

---

## 1. Проблема

IWE — платформа развития, работающая с пользователями в России и за рубежом. Три давления одновременно:

| Давление | Суть | Источник |
|----------|------|---------|
| **Юридическое** | Россия требует хранить ПД граждан в РФ (152-ФЗ). ЕС требует GDPR. Два режима несовместимы в одной инфраструктуре | Регуляторы |
| **Финансовое** | Российские и международные платёжные системы изолированы (ЮKassa/Мир vs Stripe). Инвесторы не хотят видеть карту «Мир» на международном сайте | ИТ-встреча 5, Олег Царёв |
| **Инфраструктурное** | Доступ из РФ к зарубежным сервисам нестабилен (блокировки, замедления). Neon EU может стать недоступна из РФ | Ильшат, опыт Telegram 2018-2020 |

Но IWE — не просто SaaS-продукт с пользователями в разных странах. IWE создаёт **созидателей, которые работают на улучшение мира в целом**, а не на вендоров и страны. Цель — усиление общечеловеческого интеллекта. Границ в сообществе быть не должно.

### Почему единое сообщество — архитектурный инвариант

> «Угрозы и технологии растут намного быстрее, чем растёт общечеловеческий интеллект.»
> — Манифест экосистемы

Сообщество IWE — не аудитория продукта. Это **среда развития**: человек развивается через взаимодействие с другими (peer review, менторство, команды Б5, культурная диффузия). Разделить сообщество по юрисдикциям = разделить среду = замедлить развитие каждого.

Три аргумента:
1. **Вклад не имеет гражданства.** Системное мышление одинаково ценно независимо от паспорта
2. **Среда развития неделима.** Уменьшить пул = замедлить каждого участника
3. **Культурная диффузия глобальна.** Участники несут системное мышление в общество — на рабочие места, в стартапы, корпорации, государственные организации

> «Каждый человек талантлив. Но не каждому повезло раскрыть свой потенциал — для этого нужен доступ к лучшим знаниям, к среде единомышленников и к ресурсам развития. Мы создаём экосистему, которая даёт это каждому — **независимо от стартовой точки, географии или социального статуса**.»
> — Манифест экосистемы

> Подробное обоснование: [03-community-and-culture.md §0](03-community-and-culture.md)

### Противоречие

**Финансы и данные вынуждают разделить. Сообщество и культура требуют объединить.** Политики и государства нарезают мир на зоны — мы обязаны подчиняться финансовому и data-регулированию. Но сообщество — наше пространство, и здесь границ быть не должно.

## 2. Как это делают другие

> Подробный разбор: [04-research-patterns.md](04-research-patterns.md)

| Компания | Что сделали | Урок для IWE |
|----------|-----------|-------------|
| **Telegram** | Юрлицо в ОАЭ, единый глобальный сервис, игнорирование 152-ФЗ (штраф 4 млн руб). Блокировку 2018-2020 пережили через domain fronting | Можно игнорировать регуляторов, если масштаб позволяет. У IWE масштаб не позволяет |
| **Miro** | Началась в Перми, перенесла HQ в США, после 2022 отключила РФ-пользователей полностью | Миграция юрисдикции работает, но мы не хотим терять РФ-рынок |
| **JetBrains** | Головная в Чехии, дочки по регионам. Open-source ядро (IntelliJ CE) доступно всем. Kotlin Foundation — совместно с Google | **Близкий аналог:** открытое ядро + коммерция по юрлицам |
| **Wise** | Юрлицо в каждой юрисдикции (10+), единый UX. Маршрутизация платежей прозрачна для пользователя | Паттерн маршрутизации: пользователь не думает, в каком контуре он |
| **Linux Foundation** | Некоммерческий фонд = сообщество (код, governance). Red Hat, Google = коммерция (support, cloud) | **Целевая модель:** Foundation владеет сообществом, компании — коммерцией |

### Ключевой паттерн

Все успешные примеры разделяют **одно и то же**: коммерция локальна, сообщество глобально. Telegram, JetBrains, Linux Foundation — финансы по юрлицам, community без границ.

## 3. Решение: три уровня разделения

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  УРОВЕНЬ 1: ЭКОНОМИКА И КОМПАНИЯ                               │
│  Разделяем полностью                                            │
│                                                                 │
│  ООО (Россия)  ◄── лицензия ──►  Inc (Delaware, USA)           │
│  ЮKassa, Мир                      Stripe, PayPal               │
│  Рубли                            USD                           │
│                                                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  УРОВЕНЬ 2: ИНФРАСТРУКТУРА И СЕРВИСЫ                            │
│  Дублируем по контурам, связываем протоколами                   │
│                                                                 │
│  Бот-RU    Ory-RU    CRM-RU       Бот-EU   Ory-EU   CRM-EU    │
│  PG-RU (PII)                       Neon-EU (PII)               │
│       │                                 │                       │
│       └────────── протоколы ─────────────┘                      │
│                                                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  УРОВЕНЬ 3: СООБЩЕСТВО И КУЛЬТУРА                               │
│  Единое, без границ                                             │
│                                                                 │
│  ┌───────────────── Community Layer ──────────────────┐         │
│  │  Баллы (Points Engine)    — единая система вклада  │         │
│  │  Цифровой двойник (ЦД)    — один человек = один ЦД │         │
│  │  Знания (Knowledge MCP)  — общие курсы и материалы │         │
│  │  Identity (canonical ID) — один аккаунт навсегда   │         │
│  │  Команды (Б5)            — без границ по составу   │         │
│  └────────────────────────────────────────────────────┘         │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### Как это работает технически

**Псевдонимизация** — ключевой ход. В Community Layer нет персональных данных (ФИО, email, телефон). Только `ory_id` (UUID). Ключ «ory_id → ФИО» хранится в контурном Ory, в юрисдикции пользователя.

```
КОНТУР РФ                    COMMUNITY LAYER               КОНТУР МИР
┌────────────────┐     ┌──────────────────────┐     ┌────────────────┐
│ ory_id → ФИО   │     │ ory_id → баллы       │     │ ory_id → name  │
│ ory_id → email │     │ ory_id → прогресс    │     │ ory_id → email │
│ ory_id → тел.  │     │ ory_id → квалификация│     │ ory_id → addr  │
│ (PII, 152-ФЗ)  │     │ (псевдоним, ОК)      │     │ (PII, GDPR)    │
└────────────────┘     └──────────────────────┘     └────────────────┘
```

Результат: баллы, ЦД, знания — юридически не являются персональными данными. Не требуют локализации ни по 152-ФЗ, ни по GDPR.

## 4. Текущая ситуация и эволюция

### Что есть сейчас

- **РФ-юрлицо** — активное, принимает оплаты
- **US-юрлицо (Delaware)** — зарегистрировано, неактивное
- **Одна Neon EU** — все данные, один бот, один Ory
- **Единое сообщество** — уже работает, без разделения

### Эволюционный путь

| Этап | Что | Триггер | Детали |
|------|-----|---------|--------|
| **Сейчас** | Одна Neon, одно активное юрлицо, один бот | — | Работает |
| **Этап 1** | Активировать Delaware. CRM-сервис с `source_region`. Лицензия между юрлицами | Приём международных оплат | [01-economic](01-economic-model.md) §2 |
| **Этап 2** | Два бота, два Ory, outbox для RU→EU. Community Layer как API | Рост международных пользователей | [02-infra](02-infrastructure-and-services.md) §2 |
| **Этап 3** | PG в РФ для PII. Foundation для сообщества | Блокировка Neon из РФ / 1000+ участников | [01-economic](01-economic-model.md) §1.3 |

### Что закладываем уже сейчас

| Что | Зачем | Где |
|-----|-------|-----|
| `source_region` в таблицах | Аудит + готовность к разделению | [02-infra](02-infrastructure-and-services.md) §6 |
| `canonical_ory_id` | Готовность к двум Ory | [02-infra](02-infrastructure-and-services.md) §3 |
| Баллы без PII | Юридическая чистота Community Layer | [03-community](03-community-and-culture.md) §2 |
| Outbox-паттерн | Устойчивость при проблемах связности | [02-infra](02-infrastructure-and-services.md) §4 |

## 5. Сценарии

### 5.1. Участник платит из-за рубежа

```
Maria (Берлин) → Бот @aist_me → /subscribe
  → CRM-EU → Stripe → IWE Inc (Delaware) → подписка активна
  → Событие: subscription_created (source_region='world')
  → Activity Hub → Points Engine → баллы начислены
  → Maria видит баллы в /points (общие, не «международные»)
```

### 5.2. Участник переезжает из РФ за рубеж

1. Баллы, ЦД, прогресс — **ничего не меняется** (привязаны к canonical_ory_id)
2. Подписка переоформляется: ЮKassa → Stripe, ООО → Inc
3. Identity: Ory-RU деактивируется, Ory-EU активируется, identity_links обновляется
4. Для участника: один переключатель. Для сообщества: невидимо

### 5.3. Neon EU недоступна из РФ

```
Событие в RU → Outbox (PG-RU) → retry → Neon EU (когда доступна)
```

Баллы начисляются с задержкой (часы). Для нефинансовой метрики — допустимо. Пользователь не замечает.

## 6. Риски и митигации

| Риск | Вероятность | Последствие | Митигация |
|------|------------|-------------|-----------|
| Роскомнадзор потребует локализацию | Средняя | Нужен PG в РФ для PII | Этап 3: PG-RU, PII в контуре. Community Layer (псевдоним) остаётся в EU |
| Блокировка Neon из РФ | Средняя | CRM-RU не пишет, баллы не начисляются | Outbox + retry. PG-RU как буфер |
| Delaware не подходит для EU-данных | Низкая | Нужно EU-юрлицо дополнительно | Этап 2: добавить NL/EE-дочку при фокусе на EU |
| Два Ory → конфликт identity | Средняя | Один человек = два аккаунта | canonical_ory_id + identity_links |
| Сложность системы растёт | Высокая | Overhead на поддержку | Эволюционный подход: Этап 1 = минимум. Усложняем только по триггеру |
