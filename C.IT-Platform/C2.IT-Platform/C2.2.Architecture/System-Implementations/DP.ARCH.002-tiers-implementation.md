## 8. Реализация разделения по тирам

**Механизм — динамический набор tools + system prompt:**

```python
def build_claude_context(user) -> dict:
    system_prompt = DOMAIN_PROMPT
    tools = [search_knowledge, search_guides]

    if user.tier >= 1:  # T1+: есть ory_id, кросс-системный профиль
        system_prompt += load_profile(user.ory_id)

    if user.tier >= 3:  # T3: Персонализация+
        system_prompt += load("standard/CLAUDE.md")
        system_prompt += load("standard/memory/*")
        tools += [read_digital_twin, write_digital_twin]

    # T0: нет ory_id, данные по telegram_id (ограниченный контекст)
    # T4+ работает через Claude Code нативно (не через бота)
    # TD1: доступ к исходникам + деплой (ортогональная ось, не тир учащегося)

    return {"system": system_prompt, "tools": tools}
```

**Ключевое:** Claude получает разный набор tools и разный system prompt в зависимости от тира пользователя. Нет tool = нет доступа к данным.

## 12. Конверсионная воронка бота

> Бот — главная точка входа на платформу. Задача воронки: провести пользователя от /start до программ ЛР/РР и подписки, показывая ценность каждого шага.

### 12.1. Единый продукт — подписка БР

```
Пользователь
    │
    └──→ Подписка БР (system-school.ru)  ← конвертируем ИЗ бота НАРУЖУ
           Все функции бота (лента, консультация, заметки, планы)
           + Программы (ЛР, РР, руководства)
           + Сообщество, ДЗ-чекер, клуб

    (TG Stars — опциональные донаты, не гейтят функции)
```

### 12.2. Триггеры конверсии (7 типов)

| # | Триггер | Когда | Что предложить | Канал |
|---|---------|-------|---------------|-------|
| C1 | **Марафон завершён** | `marathon_status = 'completed'` (28 тем) | «Что дальше?» → 3 варианта: Лента, ЛР, РР | Подписка БР |
| C2 | **Консультация «что дальше?»** | Пользователь спрашивает «что изучать?», «куда расти?», «что дальше?» | Claude рекомендует программу на основе профиля/ЦД | Программы |
| C3 | **Активность N дней** | 7 / 14 / 30 / 60 / 90 дней с момента регистрации | Milestone-сообщение: прогресс + следующий шаг | Подписка БР |
| C4 | **Подписка не оформлена** | Пользователь активен N дней без подписки БР | Напоминание: ценность + подписка БР | Подписка БР |

### 12.8. Cooldown и анти-спам

> **Принцип:** предлагать, не навязывать.

| Правило | Значение |
|---------|----------|
| **Между предложениями** | Не менее 7 дней между конверсионными сообщениями одного типа |
| **Отказ = cooldown** | Если пользователь закрыл/проигнорировал предложение → +14 дней cooldown |
| **Максимум в неделю** | Не более 2 конверсионных сообщений в неделю (включая milestones) |
| **Трекинг** | Таблица `conversion_events(chat_id, trigger_type, shown_at, action)` |

## 13. Progressive UI per Tier (WP-52 v4)

> **Source-of-truth дизайна:** `DS-my-strategy/inbox/WP-52-progressive-ui-tiers.md`
> **Реализация:** `aist_bot_newarchitecture/core/tier_config.py`, `tier_detector.py`, `tier_ui.py`

### 13.1. Два слоя навигации

| Слой | Механизм | Когда |
|------|----------|-------|
| **mode_select ReplyKeyboard** | 2×2 grid (tier-dependent) | При возврате в главное меню |
| **SM-contextual ReplyKeyboard** | Row 1: контекст стейта + Row 2: `[🏠 Меню] [⚙️]` | Внутри SM reply-стейтов |
| **Menu ☰** | Per-user `BotCommandScopeChat` | Всегда доступно (гамбургер) |

### 13.2. mode_select Keyboards

```
T0:  [📚 Марафон]   [🧪 Тест]    / [📊 Прогресс]  [⚙️ Настройки]
T1:  [📚 Марафон]   [🧪 Тест]    / [📊 Прогресс]  [⚙️ Настройки]
T2:  [📖 Лента]     [🧪 Тест]    / [📊 Прогресс]  [⚙️ Настройки]
T3:  [🧬 ЦД]        [📖 Лента]   / [📊 Прогресс]  [⚙️ Настройки]
T4:  [📋 Мой план]   [🏛 Клуб]   / [🧬 ЦД]        [⚙️ Настройки]
TD1: = T{N} keyboard + dev-commands в menu (bot.py)
```

### 13.3. Ключевые принципы

- **⚙️ Настройки** — с текстом, в каждой клавиатуре (Language первый)
- **Help = gateway** — /help показывает все тиры (locked/unlocked), все команды доступны по вводу
- **SM-contextual** — SM не удаляет KB, а заменяет на контекстную
- **TD1 menu = dev-only** — stats, usage, qa, health, latency, errors, analytics, ... (задаётся в bot.py)
- **Тир падает** при истечении подписки → T1 (или T0, если нет Ory)

### 13.4. Payment-first Tier Detection

```python
# Ось T (учащийся):
T4:  (подписка БР ИЛИ trial) + GitHub подключён
T3:  (подписка БР ИЛИ trial) + ЦД подключён (dt_connected_at IS NOT NULL)
T2:  подписка БР активна (system-school.ru). Trial НЕ даёт T2 — только T0/T1 с разблокированными функциями
T1:  ory_id IS NOT NULL, нет подписки, trial истёк
T0:  ory_id IS NULL (нет регистрации в Ory), trial истёк

# Trial (30 дней от /start):
T0+trial:  все функции бота разблокированы (feed, consultation, notes, plans)
T1+trial:  то же + кросс-системный UUID
# После trial без подписки → LOCKED_SERVICES заблокированы, марафон бесплатно

# Ортогональные оси (назначаются, хранятся в БД):
TD1: DEVELOPER_CHAT_ID match (или role в public.users)
TA1-TA4: role в public.users (назначение владельцем)
TM1-TM3: role в public.users (аттестация + назначение)
```

**Единый детектор:** `core/tier_detector.py:detect_ui_tier(chat_id)` — async, один на весь бот.
**Тир динамический:** вычисляется из текущего состояния, не хранится. Подписка истекла → T1 (или T0, если нет Ory).
