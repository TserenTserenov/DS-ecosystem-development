# 1.1.4. User Research — раздел исследовательских интервью

> Раздел экосистемы, в котором живёт регулярная работа с пользователями: подготовка интервью, транскрипты, разбор по направлениям, реестр задач и связь с РП.

## Зачем этот раздел

Регулярные интервью — основной канал обратной связи от пользователей IWE и продуктов МИМ. Без явного процесса интервью становятся одноразовыми артефактами, и идеи теряются между разговором и реализацией. Этот раздел замыкает цикл «интервью → задача → РП → результат».

## Структура

| Папка/файл | Что хранится |
|------------|--------------|
| `process.md` | Концепция процесса: Service Clause, сценарии, роли, рабочая процедура, метрики |
| `templates/` | Шаблоны вопросов под 7 классов (C1 онбординг IWE / C2 подписка МИМ / C3 наставник / C4 тестирование IWE / C5 содержание программ / C6 пилот ЛР / C7 практическая сессия) + чеклист подготовки и проведения |
| `transcripts/` | Транскрипты по дате: `YYYY-MM-DD-<class>-<initials>.md` (см. конвенцию в `process.md § Транскрипты`) |
| `analysis/` | Сводки по направлениям, посегментные разборы, кросс-интервью паттерны |
| `tasks-registry.md` | **Единый реестр задач** из всех интервью с маршрутизацией в РП (новые фазы / новые WP / отказ) |
