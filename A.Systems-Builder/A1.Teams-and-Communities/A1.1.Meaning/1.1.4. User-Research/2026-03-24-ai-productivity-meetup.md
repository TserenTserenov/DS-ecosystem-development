<summary><b>Контекст: зачем этот митап</b></summary>

Джедайская Академия (Максим Дорофеев) — сообщество вокруг системы личной продуктивности («Джедайские техники»). Участники («джедаи») проходят 12-недельные спринты. Многие начали внедрять ИИ в свои рабочие процессы. Решили провести «смотрины» — каждый показывает, что сделал для себя.

**Ключевой посыл Дорофеева:** «Мы все сейчас новички. Формат не "гуру учит", а "вместе разбираемся". Мы вам — процесс, а дальше вместе учимся.»

Планируется отдельная бэди-группа в спринте, где участники вместе будут строить ИИ-инструменты.

<summary><b>1. Севара — «Второй мозг» через Claude Code (не-программист)</b></summary>

**Проблема:** Выгорела 2 года назад, не могла вернуться к планированию. Todoist + Singularity не хватало — нужна не только задачная система, а «история» (наблюдения, знакомства, размышления).

**Что сделала:**
- Посмотрела ролик на YouTube, решила повторить, хотя «ноль в коде»
- Развернула бота на своём VPS (изначально для VPN, в Кыргызстане) через **Claude Code**
- Подключила Obsidian, Todoist (бесплатное API), голосовую распознавалку (DeepGram API, бесплатно)

**Как работает:**
- Пишет боту, что делала за день — он раскладывает: задачи → Todoist, наблюдения → Obsidian, бизнес-активности → отдельная категория
- Голосовые заметки на ходу → бот сам определяет по контексту: идея, наблюдение, задача
- Утренний дайджест: напоминает о невыполненном, показывает бизнес-активности, CRM-пробелы
- Синхронизация 2 раза в сутки (экономия токенов)
- Есть видение на 3 года, цели на год, задачи на 2 месяца — бот держит это в фокусе

**Сколько заняло:** 5 часов (вместо обещанных 2). Главный урок: **«Когда Claude говорит "иди, сделай это на сервере" — говори "сделай всё сам". Не ведись.»**

**Стоимость:** Подписка Claude Pro ($20/мес).

<details>
<summary><b>2. Ирина — Strobery: армия «миньонов» без кода</b></summary>

**Инструмент:** [Strobery](https://strobery.ai/) — агентский браузер (стартап из Стокгольма). Агенты работают прямо в браузере, эмулируя действия пользователя. Системы не определяют, что это агент.

**Команда миньонов:**
- **LinkedIn-скаут** — раз в 2 часа ходит в LinkedIn, ищет посты для комментирования. Критерии заданы в промпте.
- **Комментатор** — пишет вдумчивые комментарии под найденными постами.
- **Утренние/вечерние рутины** — напоминает о задачах, предлагает «хочешь, я за тебя сделаю?»

**Как пишет промпты:**
- Берёт любую думающую модель (Claude 4.6) или Whisper для наговаривания
- Описывает задачу голосом → модель генерирует промпт → вставляет агенту
- Итерирует: если агент тупит → смотрит, где застрял → дописывает в промпт конкретную инструкцию (например, «нажми три точки → copy link to post»)

**Кейс с квартирой:**
- Искала квартиру в Болгарии. Посадила Strobery на сайт недвижимости
- За ~2 часа агент отправил 73 запроса риэлторам
- На следующий день отзвонились ~20-30 агентов
- За 36 часов: 30 актуальных вариантов → 8 просмотров → 1 идеальная квартира

**Кейс с Telegram-архивом:**
- Выгрузила весь архив «сохранёнок» из Telegram (с 2022 года)
- Через **Code Cowork** (low-code инструмент) за 30 минут всё рассортировано в Obsidian: по темам, тегам, связям между материалами
- Финансовое моделирование венчурного фонда — тоже через Code Cowork, работа с реальными Excel-файлами

**Совет по оптимизации расходов:** Блогер **Velvet Shark** — как настроить OpenClaw с дешёвыми моделями на простые задачи и дорогими на сложные.

**Стоимость:** Strobery $20/мес.

<summary><b>3. Станислав — Obsidian + Claude Code: питание, тренировки, финансы</b></summary>

**Проблема:** Рутинные сферы проваливаются — питание (что купить, что готовить), тренировки (перетрен/недотрен), финансы (не рассчитал зарплату/премии).

**Что сделал:**

*Питание:*
- Создал Obsidian Vault, занёс данные о здоровье, предпочтениях, составе семьи
- Транскрибировал видео доверенного тренера → Claude систематизировал принципы → построил программу тренировок + дневник
- Claude генерирует PDF с 7 карточками A5: меню на неделю, чеклист для магазина, рецепты
- Закрыл ~90% потребностей

*Тренировки:*
- Claude построил программу по принципам из видео: упражнения, прогрессия, периодизация
- Шаблон дневника тренировок в Obsidian → заполняет с телефона

*Финансы:*
- Вайбкодил приложение за 4 часа
- Дал Claude API зарплатной системы + принципы расчёта
- Приложение рассчитывает зарплату, премии, отпускные на год вперёд (точность до 100 руб.)
- Экспортирует плановые транзакции в Дзен-мани

*Синхронизация:* Obsidian Vault в iCloud → Mac + iPhone.

*Obsidian-плагин:* **Claudium** (только с GitHub, нет в Community Plugins) — Claude Code прямо внутри Obsidian.

<summary><b>4. Ольга — GPT-ассистенты для коучинга и привычек</b></summary>

**Подход:** Не кодит, не использует API. Создаёт кастомных GPT-ботов в ChatGPT. Пишет промпты, итерирует, скармливает книги.

**Любимчик — «Тренер сосредоточенности»:**
- Промпт: «Бережный поддерживающий тренер. Без давления, стыда, морализаторства. На моей стороне. Отвлечение — не провал.»
- Когда отвлекается — идёт жаловаться боту вместо того, чтобы листать соцсети
- Бот помогает распутать момент: что отвлекло (мысли, вкладка, эмоции), и мягко возвращает к задаче
- За 1.5 месяца — «очень сильные изменения». При паузе — откат. Вернулась.

**Другие боты:**
- **Мастер по внедрению привычек** — скормлены «Атомные привычки», «Нано-привычки». Раскладывает привычку на лесенку шагов.
- **Кот Баюнчик** — помогает вовремя лечь спать. Уговаривает, напоминает, гладит.
- **Мистер Фикс** — декомпозитор проектов на стартовые задачи. Скормлены «Джедайские техники» + посты Дорофеева.
- **Небекайза** — обучает других пользоваться ChatGPT (показывает разницу между тупым и хорошим промптом).

**Ключевой инсайт:** «Когда хочу отвлечься — иду к нему жаловаться вместо того, чтобы пойти листать.»

<details>
<summary><b>5. Михаил — Геймификация продуктивности (вайбкод-приложение)</b></summary>

**Что сделал:** Telegram-бот с геймификацией задач. Вайбкодил за 2-3 недели.

**Механика:**
- Герой с уровнем и энергией
- Энергия тратится на задачи, восстанавливается (2 мин передышки = немного энергии)
- XP за задачи. Если перерабатываешь — XP начисляется меньше
- Навыки-паутинка: выбираешь, какой навык прокачивает задача. Ежедневная практика = быстрый рост. 5+ дней без практики = навык увядает
- **Рандомные награды**: 3 типа (отдых, развитие, приключение), 3 уровня редкости (обычная 80%, редкая, эпическая). Вероятностная система — эффект казино.

**Дорофеев:** «Раньше я был уверен — невозможно сделать нормального тудушника. Сейчас чешутся руки.»

<summary><b>6. Даниил — Claude Code как полноценный экзокортекс (IWE-подход)</b></summary>

**Контекст:** Инжиниринг-менеджер в финтех (GMT+0). Работает через Claude Code весь рабочий день. Использует систему, основанную на подходах Церена Церенова (Школа системного менеджмента), с доработками.

**Архитектура:**
- **Git-репозиторий** с правилами, протоколами, паками знаний по доменам
- **Паки знаний** разделены: IT Operations, дата-центры для ИИ на Ближнем Востоке, etc.
- **MCP-интеграции:** Google Calendar, Granola (запись митингов), Singularity, Telegram

**Ритуалы:**
- **Начало недели:** Сессия стратегирования → ревью → инбоксы → план
- **Начало дня:** «Давай, начали день» → план из недельных РП → проверка инбоксов
- **Архгейт:** Если начинает работать над чем-то вне плана → Claude проверяет: есть ли в плане? Связано ли? >15 мин? Если нет — стоп.
- **Конец дня:** Настроение/энергия/фокус (3 цифры), синхронизация git, captures, вечерняя рефлексия (по встречам с конца в начало)

**После митингов:**
- Granola записывает и транскрибирует
- Claude Code стягивает транскрипцию → экстрактор: action items → бэклог, новые знания → пак

**Ключевая идея:** Формирует систему так, чтобы через пару недель передать команде: «Возьмите git-репозиторий, подключите, пропускайте задачи через пак знаний по протоколу.»

**Безопасность:** Хуки в Claude Code: блокировка пуша при обнаружении API-ключей, ограничения по ключевым словам.

**Стоимость OpenClaw:** В первый день настройки ушло 100M токенов (~€1500 на Sonnet). Решение: китайские модели по подписке.

**Ресет навыков и лидеров:**
- Stack Overflow — посещаемость в ноль. «Он умер, чтобы они жили.»
- Авторитет сеньор-разработчиков размывается
- «Училка стала кодером, домохозяйка кодит — и код работает.»

**Присвоение результатов ИИ:**
- Люди склонны присваивать результат ИИ: «это я сделал» (как с Google: найденная информация ощущается как собственное знание)
- «Как в анекдоте: мужики, я уголь привёз! — хера себе он привёз, подумала лошадь.»

**Собственный мозг — ключевой ресурс:**
- «Нужен навык на лету понимать, не чухню ли тебе продают.»
- Без дисциплины ИИ-инструменты не работают. Всё держится на привычке начинать/закрывать день.

**Вайбкодинг — казино:**
- Затягивает: «шлёп, сделал, не то — переделал, попал — кайф.»
- «Если сделал плохо — ты такой "а-а-а, надо переделать!" и переделку уже не прокрастинируешь.»

**Каждый инструмент — старый ботинок:**
- «Все выемки и неидеальности стопы промялись — удобен. Другому — мерзкий и плохо пахнет. С мобильным приложением так же.»
