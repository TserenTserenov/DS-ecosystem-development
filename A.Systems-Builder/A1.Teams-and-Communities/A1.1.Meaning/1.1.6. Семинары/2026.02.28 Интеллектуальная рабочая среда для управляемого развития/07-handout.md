# Четыре ступени образования агента

Claude, GPT, Gemini — это выпускник **школы**, не вуза. Чтобы агент стал полезным, нужны ещё три ступени:

| # | Ступень | Кто даёт | Аналогия |
|---|---------|----------|----------|
| 0 | **Способности** — читать, писать, рассуждать | Anthropic / OpenAI | Выпускник школы |
| 1 | **Мышление и метод** (ZP + FPF) | Платформа (Aisystant) | Выпускник вуза — методолог |
| 2 | **Знание домена** (Pack) | Вы как руководитель | Специалист в должности |
| 3 | **Задачи и РП** (DS) | Вы как руководитель | Специалист с заданием |

> **Base** = платформа выдаёт (ступени 0–1).
> **Pack и DS** = вы создаёте (ступени 2–3).

Большинство пользователей делают ступени 0 и 3 — оплатили подписку и сразу к промптам. Пропускают **мышление** и **домен**. Это как дать школьнику задание финансового аналитика — без вуза и без ввода в должность.

---

# Четыре роли при взаимодействии с ИИ-агентом

| Роль | Что делает | Аналогия |
|------|-----------|----------|
| **Изготовитель** | Обучает модель (pre-training) | Школа-Вуз |
| **Дообучатель** | Специализирует (fine-tuning) | Учебный центр |
| **Инженер среды** | Проектирует обвязку: знания, инструменты, протоколы | Администратор + HR |
| **Пользователь** | Ставит задачи и принимает результат | Заказчик работы |

В IWE вы играете **обе последние роли**:
- **Инженер среды** — при развёртывании и настройке (CLAUDE.md, память, роли)
- **Пользователь** — каждый день (даёте задачи, принимаете результат)

> Качество работы ИИ определяется не моделью, а **средой** — обвязкой, которую вы создали.

---

# Три режима работы с ИИ-агентом

Почему IWE, а не просто чат:

| | **Чат** | **IWE** | **Fine-tuning** |
|---|---------|---------|-----------------|
| **Знание** | В голове пользователя | В файлах (Pack, MCP) | В весах модели |
| **Онбординг** | Вручную, каждый раз | Авто, при запуске | Не нужен |
| **Забывает?** | Да | Нет | Нет |
| **Аналогия** | Объясняете каждое утро | Инструкция на столе | После курсов |

> Вы описываете один раз — среда онбордит агента каждую сессию.

# CLAUDE.md — файл инструкций для ИИ-агента

**CLAUDE.md** — текстовый файл с правилами, которые ИИ-агент читает **автоматически** при каждом запуске.

| Вопрос | Ответ |
|--------|-------|
| Где лежит? | В корне рабочей директории (`~/Github/CLAUDE.md`) |
| Кто читает? | Claude Code — автоматически при старте сессии |
| Кто пишет? | Вы (инженер среды) |
| Зачем? | Агент помнит правила, не нужно повторять |
| Аналогия | Должностная инструкция + внутренний регламент на столе сотрудника |

> Без CLAUDE.md каждая сессия — с нуля. С CLAUDE.md — агент уже знает правила.

# CLAUDE.md — пакет онбординга для ИИ-агента

Аналогия: вы **нанимаете** выпускника (Claude), а не создаёте его. CLAUDE.md — это всё, что новый сотрудник получает в первые недели:

| Аспект | Аналогия в компании | В CLAUDE.md |
|--------|---------------------|-------------|
| **Корпоративная культура** | «У нас так принято» | Принципы, ArchGate, Capture-to-Pack |
| **Должностная инструкция** | Роли, зоны ответственности | Каталог ролей, гейты |
| **СОПы / регламенты** | Стандартные процедуры | ОРЗ (Open→Work→Close), WP Gate |
| **Ubiquitous Language** | «У нас "спринт" значит вот это» | Термины: «сервис», «РП», различения |
| **Оргструктура** | Кто кому, где что лежит | Иерархия репо (Base→Pack→DS) |
| **Система качества** | ISO, чеклисты перед релизом | ЭМОГССБ ≥ 8, Pre-action Gates |
| **Институциональная память** | «Мы пробовали X — не работает» | memory/*.md, уроки |

> CLAUDE.md — не «корпоративная культура», а **организационная операционная система**: культура + процессы + язык + память.

---

# Структура CLAUDE.md (из шаблона)

CLAUDE.md в шаблоне IWE содержит 6 секций:

| § | Секция | Что описывает |
|---|--------|---------------|
| 1 | **Архитектура репозиториев** | 3 типа репо (Base, Pack, DS), fallback chain, иерархия принципов |
| 2 | **Стадии сессии — ОРЗ** | Открытие → Работа → Закрытие, блокирующие правила, Capture-to-Pack |
| 3 | **Описания методов** | Что такое сервис, процесс, сценарий; когда нужен PROCESSES.md |
| 4 | **Memory (Слой 3)** | Навигация по справочным файлам, политика лимитов |
| 5 | **АрхГейт** | Обязательная оценка архитектурных решений (7 характеристик, порог ≥8) |
| 6 | **Обновление** | Куда писать новые правила |

> Это **slim-ядро**: триггеры и правила. Детали протоколов — в `memory/protocol-*.md` (подгружаются по триггеру, не занимают контекст постоянно).

---

# Как работает CLAUDE.md: пример

**Вы запускаете Claude Code в терминале:**

```
$ cd ~/Github
$ claude
```

**Claude Code автоматически:**
1. Читает `~/Github/CLAUDE.md` → загружает все правила
2. Читает `MEMORY.md` → видит ваши задачи и уроки
3. Вы даёте задание → Claude проверяет по WP Gate (§2)
4. Работает по протоколу ОРЗ → фиксирует результат

**Без CLAUDE.md:** «Напиши код» → пишет код → забыл. Каждый раз заново.
**С CLAUDE.md:** «Напиши код» → проверил план → объявил роль → работает → фиксирует знания → обновляет память.

> CLAUDE.md превращает ИИ-ассистента в **среду**, которая работает по вашим правилам.

# Обзор: три слоя памяти экзокортекса

| Слой | Файл | Что хранит | Загрузка | Лимит |
|------|------|-----------|----------|-------|
| **1** | `MEMORY.md` | «Что на столе» — задачи, статусы, уроки | **Всегда** (каждая сессия) | ≤100 строк |

| **3** | `memory/*.md` | «Справочники» — различения, SOTA, протоколы | **По запросу** (по триггеру) | ≤10 файлов, ≤100 строк каждый |

Поэтому ИИ **никогда не начинает с нуля**.

> **Аналогия:** Слой 1 = стикеры на мониторе. Слой 2 = регламент на столе. Слой 3 = папки в шкафу (достаёшь когда нужно).

# Память экзокортекса: аналогия с наймом

Каждая сессия Claude Code — это **первый рабочий день нового сотрудника**. Он компетентен (модель мощная), но не знает ваш проект. Память — это то, что лежит на его столе.

| Аналогия | Что в системе |
|----------|--------------|
| **Диплом** (базовые знания) | Модель Claude — знает программирование, языки, фреймворки |
| **Регламент предприятия** | `CLAUDE.md` — правила, протоколы, архитектура |
| **Памятка на столе** | `MEMORY.md` — задачи этой недели, уроки, статусы |
| **Папки в шкафу** | `memory/*.md` — достаёшь нужную, когда возникает задача |
| **Записная книжка** | Контекстное окно сессии — всё, что обсудили за день |
| **Дневник уроков** | Раздел «Уроки» в MEMORY.md — «мы уже наступали на эти грабли» |

> Сотрудник каждый день приходит «с чистой головой» (новая сессия = пустой контекст). Но на столе лежит памятка, а в шкафу — справочники.

---

# CLAUDE.md vs MEMORY.md: в чём разница?

| | **CLAUDE.md** (регламент) | **MEMORY.md** (памятка) |
|---|---|---|
| **Горизонт** | Постоянно — пока существует «предприятие» | Неделя-день — текущий контекст |
| **Что внутри** | Архитектура репо, стадии ОРЗ, ArchGate, правила | РП этой недели, уроки, статусы, деплой-инфо |
| **Кто меняет** | Редко, осознанно (как пересмотр регламента) | Каждую сессию — статусы, уроки |
| **Пример** | «У нас код-ревью обязательно» | «На этой неделе — семинар 28 фев» |
| **Хранится в** | Git (в репо, коммитится) | `.claude/` (локально, не в репо) |

> **CLAUDE.md** растёт медленно — добавили правило, оно там навсегда.
> **MEMORY.md** ротируется — done-задачи удаляются, устаревшие уроки убираются.

# ОРЗ: Открытие → Работа → Закрытие

**ОРЗ** — цикл каждой рабочей сессии. Три стадии, три протокола.

| Стадия | Зачем | Триггер |
|--------|-------|---------|
| **Открытие** | Проверить план, согласовать работу | Любое задание |
| **Работа** | Фиксировать знания на лету | После Открытия |
| **Закрытие** | Зафиксировать результат, обновить память | «Закрывай» |

> **Пропуск Открытия** = незапланированная работа (хаос).
> **Пропуск Закрытия** = незафиксированный результат (потеря).

Контур замкнут: план → работа → фиксация → обновление плана.

<div class="ref">

**Полные протоколы:** `memory/protocol-open.md`, `memory/protocol-work.md`, `memory/protocol-close.md`

</div>

# Открытие: WP Gate

**WP Gate** — блокирующая проверка. Claude выполняет её **до** любого действия.

### Алгоритм:

1. Прочитать MEMORY.md → секция «РП текущей недели»
2. Задание совпадает с РП? → **Да:** продолжить. **Нет:** СТОП.

### Если не совпадает:

1. «Этой задачи нет в плане на неделю.»
2. Вывести таблицу РП
3. Спросить: артефакт, формулировка, репо, бюджет
4. Записать в три места:
   - `MEMORY.md` → новая строка в таблице РП
   - `DS-strategy/current/WeekPlan` → новая строка
   - `DS-strategy/inbox/WP-{N}-{slug}.md` → контекстный файл

### Исключения (WP Gate не нужен):

- Задачи ≤15 мин
- Вопросы без изменений файлов
- Экстренные баг-фиксы

# Открытие: Ритуал согласования

После WP Gate Claude **объявляет** план работы и ждёт согласования:

> *«**Роль:** [из каталога]. **Работа:** [что]. **РП:** [артефакт]. **Метод:** [как]. **Оценка:** [~Xh]. **Модель:** [текущая] — рекомендую [модель] ([причина]).»*

### Рекомендация модели:

| Модель | Когда |
|--------|-------|
| **Opus** | Архитектура, сложный код, стратегия |
| **Sonnet** | Типовые задачи, одно-файловые правки, контент |
| **Haiku** | Быстрые поиски, простые вопросы |

Пользователь может не согласиться и сменить модель.

> **Зачем ритуал?** Агент не начинает работу молча. Вы всегда знаете: что будет сделано, в какой роли, за сколько времени.

# Работа: Capture-to-Pack

Во время работы, на каждом рубеже (завершена подзадача, найден паттерн, принято решение), Claude проверяет: **есть ли знание для записи?**

| Тип знания | Куда | Когда | Нужен Экстрактор? |
|------------|------|-------|--------------------|
| Правило для всех репо (1-3 строки) | `~/Github/CLAUDE.md` | Сразу | Нет |
| Правило для одного репо (1-3 строки) | `<repo>/CLAUDE.md` | Сразу | Нет |
| Доменное (архитектура, паттерны) | Соответствующий Pack | При Close | Да |
| Различение, метод, FM, РП | Соответствующий Pack | При Close | Да |
| Крупный урок | `memory/<topic>.md` | При Close | Нет |

**Формат объявления:** *«Capture: [что] → [куда]»*

> Знания не теряются в чате — они маршрутизируются в нужное место.

# Работа: Pre-action Gates

Перед определёнными действиями Claude **обязан** проверить:

| Перед чем | Что проверить |
|-----------|--------------|
| Началом работы над задачей | Какие сервисы затронуты? |
| Ответом на доменный вопрос | Поиск по базе знаний (MCP) |
| Записью в Pack | Нет дубликата? Нет противоречия? |
| `git commit` | Прочитать CLAUDE.md репо |
| Архитектурным предложением | АрхГейт: таблица ЭМОГССБ, порог ≥8 |
| Добавлением инструмента/агента | IntegrationGate: тип, контур, роли, продукты, процессы |

> Gates — это «контрольные точки», предотвращающие типичные ошибки.

# Закрытие: алгоритм Close

При слове **«закрывай»** Claude выполняет 8 шагов:

| # | Шаг | Что делает |
|---|-----|-----------|
| 0 | **Pull** | `git pull --rebase` в DS-strategy |
| 1 | **Извлечение знаний** | Собрать captures → классифицировать → применить |
| 2 | **MEMORY.md** | Обновить статусы РП |
| 3 | **Фиксация** | Записать что сделано, что осталось |
| 4 | **Git commit** | Закоммитить (с подтверждением) |
| 5 | **План** | Обновить DS-strategy/current/Plan |
| 6 | **Backup** | `memory/ + CLAUDE.md → DS-strategy/exocortex/` |
| 7 | **WP Context** | done → архив. in_progress → обновить контекст |
| 8 | **Остатки** | Недоделки → context file. Идеи → MAPSTRATEGIC |

# Что такое MCP

**MCP** (Model Context Protocol) — открытый стандарт от Anthropic. Позволяет ИИ-агенту **подключаться к внешним данным** через единый протокол.

| | Без MCP | С MCP |
|---|---------|-------|
| **Режим** | Push: вы решаете, что дать | Pull: агент сам ищет нужное |
| **Контекст** | Копипаст → не всё влезает | 5 400+ документов, подгружается релевантное |
| **Ошибки** | Галлюцинации в домене | Ответы опираются на **ваши** знания |
| **Аналогия** | Стопка бумаг на столе | Библиотека с поисковой системой |

> MCP превращает знания из «вручную копирую в промпт» в «агент находит сам».

---

# Как подключить MCP-сервер знаний

### Платформенный сервер (5 400+ документов):

```
URL: https://knowledge-mcp.aisystant.workers.dev/mcp
```

Подключается к Claude Code, Cursor, VS Code и другим инструментам с поддержкой MCP.

### Три способа дать агенту ваши знания:

| Способ | Как | Когда |
|--------|-----|-------|
| **Локальные файлы** | Claude Code читает файлы в репо | Всегда (базовый уровень) |
| **CLAUDE.md + memory/** | Загружается при каждом запуске | Правила и контекст |
| **Свой MCP-сервер** | Индексатор строит поисковую базу | Когда >100 документов |
