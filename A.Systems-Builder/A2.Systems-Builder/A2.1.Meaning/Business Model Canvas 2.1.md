# Business Model Canvas продукта Aisystant

## 0. Назначение документа

Этот документ описывает **бизнес-модель продукта** — как платформа Aisystant создаёт, доставляет и захватывает ценность. Отвечает на вопрос: «Как устроен бизнес, который предлагает эту ценность клиентам?»

**Семейство:** 2.1 (SoI × Meaning) — почему люди захотят покупать и пользоваться нашим продуктом.

**Важно:** Это бизнес-модель **продукта** (платформы Aisystant), а не экономика экосистемы в целом (→ [[Экономика экосистемы 3.1]]) и не personal business model участника.

## 2. Ключевая логика: открытый контент + платная практика

Бизнес-модель построена по принципу **Red Hat** (контент бесплатен, инфраструктура платная):

```
БЕСПЛАТНО (привлечение)              ПЛАТНО (ценность)
─────────────────────                ─────────────────────
Тексты руководств                    Рабочая тетрадь (кейсы, упражнения)
Доступ к знаниям (MCP)               ИИ-проверка заданий (ДЗ-чекер)
Базовый ИИ-ассистент (T1)            Персонализированный ИИ-Проводник
Публичная документация               Цифровой двойник (полный)
                                     Наставничество
                                     Сертификация
                                     Командные инструменты
```

**Почему это работает:**
- Бесплатный контент создаёт воронку: миллионы читают → тысячи конвертируются
- Конкуренты НЕ могут скопировать: у них контент = продукт. Открыть его = потерять выручку
- Мы конкурируем не контентом (commodity), а **инфраструктурой практики** (defensible)

## 3. Два слоя ценообразования

> **Источник:** [[Тарифы и пакеты 2.1]] — детальные цены, баллы, путь участника.

### Слой 1: Подписка БР (recurring) — одна подписка на всё

> **Принцип: одна подписка, нет отдельных платных тиров.** T3 и T4 включены — активируются действиями пользователя. Цена растёт постепенно по мере роста ценности платформы.

| Тир | Что получает | Оплата | Конверсия |
|-----|-------------|--------|-----------|
| **T0** Старт | Trial 30 дней: полный доступ T1–T4 | бесплатно | 100% (вход) |
| **T1** Регистрация | Руководства, клуб, марафон, консультант | бесплатно (email) | — |
| **T2** Изучение | + программы, ДЗ-чекер, полный ЦД, все функции бота | **подписка БР** | >5% от T1 |
| **T3** Персонализация | + типовая методология, персональные ответы, публикация в клубе | включено в БР (заполнить ЦД) | 30-40% от T2 |
| **T4** Созидание (IWE) | + Gateway, personal CLAUDE.md, агенты, git-репо, WakaTime | включено в БР (GitHub OAuth) | 10-20% от T3 |

**Путь пользователя:** T0 (5 мин) → T1 (5 мин) → T2 (подписка) → T3 (20 мин) → T4 (5 мин). Около 40 минут активного времени.

### Слой 1b: LLM-биллинг (отдельный от подписки)

> Платформа предоставляет AI-модели как сервис. Базовый лимит токенов включён в подписку БР. Превышение — отдельный биллинг.

| Что | Механизм | Биллинг |
|-----|----------|---------|
| Базовый лимит | Включён в подписку БР | Нет доплаты |
| Превышение | Уведомление → пакет токенов | Отдельная оплата |
| Свой API-ключ (T4) | Direct MCP | Пользователь платит напрямую провайдеру |

**Экономика:** себестоимость ~$3-5/мес на активного пользователя (при оптимальном routing). Маржа на AI-вызовах — дополнительный поток дохода.

### Слой 2: События (on-demand) — трансформация

| Продукт | Формат | Цена | Для кого |
|---------|--------|------|----------|
| **Практикум** | 6-12 недель, deliberate practice | $399-799 | T2+ (навык) |
| **Резидентура** | 6 недель, шедевр с наставником | $299-2000 | T3+ (результат) |
| **Семинар** | 2-6 часов, новое знание | ₽5000-12000 | T2+ (фронтир) |
| **Марафон** | 14 дней, микро-задания | $99-199 | Все (вход) |

> **Баллы вместо скидок:** за активные действия начисляются баллы (множитель по тиру: T2 ×1, T3 ×1.5, T4 ×2). Баллами можно оплатить события полностью или частично (1 балл ≈ $1). Это мотивирует активное участие и апгрейд тира.

**Принцип:** подписка = инструменты (recurring), события = трансформация (one-time). Цена = результат трансформации, не доступ к контенту. Наставник НЕ включён в подписку.

## 5. Revenue Sharing (как платформа стимулирует стороны)

| Участник | Доля | За что |
|----------|------|--------|
| **Платформа** | 30% | Инфраструктура, ИИ, маркетинг, поддержка |
| **Автор руководства** | 50% | Создание и обновление контента |
| **Наставник** | 15% | Проверка ДЗ, менторинг, обратная связь |
| **Куратор** | 5% | Организация потока, контроль качества |

Revenue Sharing работает как **маховик**: больше выручка → больше заработок наставников и авторов → больше качественного контента и проверки → больше retention → больше выручка.

## 6. Шесть маховиков роста

### Маховик 1: Контент → пользователи → вклад → контент
```
Бесплатные руководства → привлекают читателей →
  часть конвертируется → делают ДЗ →
    лучшие ответы становятся примерами →
      руководства улучшаются → привлекают ещё
```

### Маховик 2: Данные → ИИ → персонализация → данные
```
Пользователи заполняют ЦД → ИИ-Проводник адаптирует маршрут →
  пользователь учится эффективнее → больше данных в ЦД →
    ИИ становится точнее → retention растёт
```

### Маховик 3: Наставники → качество → retention → выручка → наставники
```
Revenue Sharing привлекает наставников →
  наставники улучшают качество проверки →
    retention учеников растёт →
      больше выручки → больше наставников
```

### Маховик 4: AI Self-Learning Flywheel (артефакты → обучение ИИ → лучшая помощь)

> Аналогия с Anthropic RLHF: как качественный человеческий feedback улучшает ИИ-модели, артефакты участников с подтверждённым эпистемическим статусом улучшают ИИ-агентов платформы.

```
Участники создают артефакты (ДЗ, кейсы, проекты) →
  наставники / peer-review подтверждают качество →
    артефакты с высоким epistemic status → тренировочные данные ИИ
      (вес = квалификация автора × качество артефакта) →
        ДЗ-чекер точнее проверяет, Проводник точнее маршрутизирует,
        Консультант глубже отвечает →
          участники учатся эффективнее → создают ещё лучшие артефакты ↺
```

**Множитель веса автора:** Ученик ×1.0, Интеллектуал ×1.5, Профессионал ×2.0, Исследователь ×3.0, Просветитель ×4.0. Чем зрелее сообщество, тем умнее ИИ. Подробнее: [WP-73 §3.7](../../0.OPS/0.9.Inbox/WP-73-aisystant-platform-architecture.md).

### Маховик 5: Финансовая устойчивость
```
Выручка от подписок и событий → инвестиции в инфраструктуру →
  лучшие продукты → больше ценности для участников →
    выше retention → больше выручки
```

### Маховик 6: Культурная диффузия
```
Созидатели применяют мышление в своих организациях →
  организации видят результаты → заказывают корпоративные программы →
    новые участники приходят из организаций → сообщество растёт
```
