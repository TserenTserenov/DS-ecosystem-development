# Business Model Canvas продукта Aisystant

## 0. Назначение документа

Этот документ описывает **бизнес-модель продукта** — как платформа Aisystant создаёт, доставляет и захватывает ценность. Отвечает на вопрос: «Как устроен бизнес, который предлагает эту ценность клиентам?»

**Семейство:** 2.1 (SoI × Meaning) — почему люди захотят покупать и пользоваться нашим продуктом.

**Важно:** Это бизнес-модель **продукта** (платформы Aisystant), а не экономика экосистемы в целом (→ [[Экономика экосистемы 3.1]]) и не personal business model участника.

## 2. Ключевая логика: открытый контент + платная практика

Бизнес-модель построена по принципу **Red Hat** (контент бесплатен, инфраструктура платная):

```
БЕСПЛАТНО (привлечение)              ПЛАТНО (ценность)
─────────────────────                ─────────────────────
Тексты руководств                    Рабочая тетрадь (кейсы, упражнения)
Доступ к знаниям (MCP)               ИИ-проверка заданий (ДЗ-чекер)
Базовый ИИ-ассистент (T1)            Персонализированный ИИ-Проводник
Публичная документация               Цифровой двойник (полный)
                                     Наставничество
                                     Сертификация
                                     Командные инструменты
```

**Почему это работает:**
- Бесплатный контент создаёт воронку: миллионы читают → тысячи конвертируются
- Конкуренты НЕ могут скопировать: у них контент = продукт. Открыть его = потерять выручку
- Мы конкурируем не контентом (commodity), а **инфраструктурой практики** (defensible)

## 3. Тиры как модель ценообразования

| Тир | Что получает | Цена | Конверсия |
|-----|-------------|------|-----------|
| **T1** (бесплатно) | Тексты, базовый ИИ, чат | $0 | 100% (вход) |
| **T2** (изучение) | Рабочая тетрадь, ДЗ-чекер, ЦД (базовый) | $10-30/мес | >5% от T1 |
| **T3** (персонализация) | ИИ-Проводник, полный ЦД, наставник | $30-100/мес | 30-40% от T2 |
| **T4** (IWE) | Экзокортекс, агенты, командные инструменты | $100+/мес | 10-20% от T3 |

**Принцип:** цена = результат трансформации, не доступ к контенту. Каждый следующий тир = больше ценности = больше готовности платить.

| Метрика | Значение | Комментарий |
|---------|----------|-------------|
| **CAC** | $50-150 | Органический (бесплатный контент + WoM) |

## 5. Revenue Sharing (как платформа стимулирует стороны)

| Участник | Доля | За что |
|----------|------|--------|
| **Платформа** | 30% | Инфраструктура, ИИ, маркетинг, поддержка |
| **Автор руководства** | 50% | Создание и обновление контента |
| **Наставник** | 15% | Проверка ДЗ, менторинг, обратная связь |
| **Куратор** | 5% | Организация потока, контроль качества |

Revenue Sharing работает как **маховик**: больше выручка → больше заработок наставников и авторов → больше качественного контента и проверки → больше retention → больше выручка.

---

## 6. Три маховика роста

### Маховик 1: Контент → пользователи → вклад → контент
```
Бесплатные руководства → привлекают читателей →
  часть конвертируется → делают ДЗ →
    лучшие ответы становятся примерами →
      руководства улучшаются → привлекают ещё
```

### Маховик 2: Данные → ИИ → персонализация → данные
```
Пользователи заполняют ЦД → ИИ-Проводник адаптирует маршрут →
  пользователь учится эффективнее → больше данных в ЦД →
    ИИ становится точнее → retention растёт
```

### Маховик 3: Наставники → качество → retention → выручка → наставники
```
Revenue Sharing привлекает наставников →
  наставники улучшают качество проверки →
    retention учеников растёт →
      больше выручки → больше наставников
```
