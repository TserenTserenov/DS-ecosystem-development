# Карта ролей и ответственности — мастер-реестр команды

## 0. Назначение документа

Мастер-реестр членов команды Aisystant: ФИО, должности, общая зона ответственности, приоритетные системы. **Source-of-truth для персональной идентичности.**

Per-kernel срезы (кто в какой роли работает над конкретной системой) живут в:
- [`B3.Aisystant-Ecosystem/.../B3.3.1. Team-and-Roles/`](../../../../B.Aisystant-Ecosystem/B3.Aisystant-Ecosystem/B3.3.Operations/3.3.1.%20Team-and-Roles/) — для экосистемы
- [`C3.Platform-Dev-Team/.../C3.3.1. Team-and-Roles/`](../../../../C.IT-Platform/C3.Platform-Dev-Team/C3.3.Operations/3.3.1.%20Team-and-Roles/) — для ИТ-платформы

**Правило:** per-kernel файлы используют ссылки на этот реестр (`[Андрей](../../../A/.../#андрей)`), не дублируют ФИО/должности.

## 1. Команда (ядро внутреннее)

Организована в 4 функциональные службы. Приоритетные системы — ядра S2R, в которых член команды работает основное время.

### 1.1. ИТ-команда (6 чел)

| Имя | Должность | Приоритетные системы | Зона ответственности |
|-----|-----------|---------------------|---------------------|
| **Андрей** | Архитектор | C (ИТ-платформа) | Архитектурные встречи, ADR, Ory-инфра, серверы |
| **Паша** | DevOps | C (ИТ-платформа) | Деплой Ory (Kratos/Hydra/Keto), Railway, серверная инфраструктура |
| **Дима** | Backend (LMS) | C (ИТ-платформа) | Aisystant PG, `AccessService.java`, Payment Registry |
| **Ильшат** | Backend / менеджер | C (ИТ-платформа) | CRM, Payment Registry (совместно с Димой), декомпозиция монолита |
| **Наталья** | Frontend-разработчик | C (ИТ-платформа) | Фронтенд. Связана с Ory-инфрой (Андрей разблокирует) |
| **Инга** | UX / UI дизайнер | B (Экосистема) + C (ИТ-платформа) | UX, CRO, продуктовое развитие. Точечная загрузка (не full-time). См. WP-199 |
