# Приложение 4. Задачи по службам (Q2)

> Приложение к [Стратегии экосистемы 2026](Стратегия%20экосистемы%202026.md).
> Персоналии — в мастер-реестре [`A3.3.1/Карта ролей и ответственности 3.3.md`](../../A3.3.Operations/3.3.1.%20Team-and-Roles/Карта%20ролей%20и%20ответственности%203.3.md).

## ИТ-команда (6 чел)

**Главная задача:** MVP IWE к 1 мая в проде + D30 замер к 30 мая.

- **Бэкенд LMS:** характеристики деятеля ЦД v3 (WP-151 Ф7b), прямые платежи + autopay (WP-246)
- **Бэкенд CRM / будущий менеджер продукта:** реестр подписок (WP-231), CRM + воронка метрик (WP-183), **к 1 июня** — приём продуктовых решений (handoff от основателя, WP-252)
- **Фронтенд:** фронт онбординга ЦД, landing под два трека (МИМ + холодный)
- **UX-дизайнер:** UX онбординга, процессы бота (WP-199)
- **Архитектор:** архитектурное ревью, карта данных Neon (WP-228), ТО IWE (WP-217)
- **DevOps:** безопасность продукта (WP-212), инфраструктура Gateway (WP-187)

**Q2 режим:** развиваем только то, что в MVP. Разделение РФ/мир (WP-215) заморожено до подтверждения гипотезы. К 30 июня — вся команда на экзоскелете (Claude Code + IWE Template), WakaTime ≥50%.

## Служба продвижения (5 чел)

**Главная задача:** валидация сегментов + запуск холодного канала + воронка метрик.

- **Руководитель службы:** 20 интервью + 20 «голых продаж» к 31 мая (WP-251), контент-план, атрибуция CAC, еженедельный sync с ИТ-командой
- **Соцсети:** упаковка сообщений под сегменты С1-С4
- **Таргет:** после Э2 — если CAC<1500 ₽ в органике, масштабируем платный
- **YouTube + дизайн:** контент для С1 Knowledge worker
- **Продюсирование:** конвейер публикаций (WP-155 ContentPipeline)

## Служба координации программ (2 чел)

**Главная задача:** связка МИМ × IWE, встраивание IWE в курсы, ручная адаптация руководств.

- **Руководитель службы (деканат):** встроить IWE как обязательный элемент в 1-2 ключевых курса МИМ (кандидат — СМ2026), замер cross-sell, адаптация персональных руководств для первой когорты MVP
- **Администратор:** администрирование программ, деканат

## Бухгалтерия (1 чел)

**Главная задача:** runway, MRR, когортный анализ, unit-экономика.

- Сверка доходов LMS / Клуб / IWE через платёжный конвейер (YooKassa + Stars + autopay)
- Metabase-отчёты для еженедельного ревью с основателем
- К 31 декабря — прозрачный P&L по юрлицам

## Основатель

**Главная задача:** передача продуктовых решений менеджеру + холодная валидация + стратегия связки МИМ×IWE.

- **Месяц 1 (апрель-май):** 20 интервью ведёт менеджер, основатель присутствует молча
- **Месяц 2 (май):** менеджер принимает все продуктовые решения. Основатель имеет 1 вопрос + 1 возражение в неделю
- **Цель к 1 июня:** ни одного продуктового решения основателя о функциях IWE
- **Что остаётся на основателе:** стратегия связки МИМ×IWE, личный бренд и публикации (холодный канал), методологический фундамент

## 4 блока стратегии

Каждый блок = физический объект + методы + подсистемы + служба-владелец.

| Блок | Физический объект | Служба-владелец |
|---|---|---|
| **Продукт** | Сам IWE: инсталляции, активные пользователи, функции в проде, артефакты ЦД | ИТ-команда + Основатель (до 1 июня, потом менеджер продукта) |
| **Клиенты** | Живые люди: подписчики, лиды, сообщество ~13 500 → 20 000, партнёры-консультанты | Служба продвижения + Служба координации программ |
| **Финансы** | Деньги на счету, runway, MRR, P&L, налоги | Бухгалтерия + Основатель (стратегия) |
| **Организация** | 14 человек, ИТ-инфраструктура, юрлица, договоры, ритуалы культуры | Основатель + менеджер продукта (передача) |

**Primary/secondary owners для пограничных направлений:**

- **Retention:** Клиенты (primary, удержание) + Продукт (secondary, инженерия)
- **Ценообразование:** Финансы (primary) + Продукт (secondary, атрибут обещания)
- **География (РФ/мир):** Клиенты (primary, рынок) + Финансы + Организация (secondary)
- **Ров:** Продукт (primary) + Организация (secondary, инфраструктура)
