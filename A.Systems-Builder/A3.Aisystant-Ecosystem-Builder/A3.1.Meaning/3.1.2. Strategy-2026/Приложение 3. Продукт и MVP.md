# Приложение 3. Продукт и MVP

> Приложение к [Стратегии экосистемы 2026](Стратегия%20экосистемы%202026.md).

## Что такое общая платформа и персональный IWE

**Наш продукт — двухслойная архитектура:**

- **Общая платформа** — инфраструктура и знания, общие для всех созидателей: шлюз для ИИ-агентов, база знаний сообщества, методология FPF, единая аутентификация, учёт подписок, общие компоненты цифрового двойника (таксономии характеристик, модели ступеней, справочники). Одна для всех, обновляется централизованно.
- **Персональный IWE** (Intellectual Work Environment) — среда созидателя под конкретного человека поверх платформы: его цифровой двойник, его документы, его персональное руководство, его история и настройки агентов. Один IWE — один созидатель.

Методология системного мышления встроена как невидимая архитектура на обоих слоях. Пользователь получает единую точку, где его знания, его контекст и ИИ-агенты работают связно, а не разрозненно в 10 инструментах.

Снаружи это выглядит как современный ИИ-ассистент. Внутри — методология, цифровой двойник и персональный маршрут развития.

## Три опоры продукта

1. **Цифровой двойник (ЦД)** — единая память о пользователе: роль, цели, характеристики деятеля (агентность, квалификация, мастерство), документы, история. Общие схемы и таксономии — на общей платформе; содержимое конкретного ЦД — в персональном IWE. Обновляется из активности пользователя + через синхронизатор.
2. **Персональное руководство** — маршрут развития «под меня»: где я сейчас (ступень 1-5), что делать на неделе, какие методы практиковать. Живёт в персональном IWE, использует методологические шаблоны с общей платформы.
3. **ИИ-агенты через MCP** — работают с контекстом пользователя. Навигатор, Диагност, Помощник, Портной (генератор руководств), Оркестратор. Агенты запускаются на общей платформе, контекст берут из персонального IWE. Единый шлюз знаний.

## MVP к 1 мая 2026

**Идея MVP:** минимальная связка «общая платформа + персональный IWE», которая удерживает новую когорту с удержанием на 30-й день > 60%. Если не удерживает — любой холодный канал утечёт, привлечение бессмысленно.

| Компонент | Что включает | Кто делает | Статус |
|---|---|---|---|
| **Ввод ЦД (онбординг)** | Диалог 15-20 мин: роль, цели, контекст работы, уровень ИИ-зрелости, 2-3 актуальные задачи | ИТ-команда: фронт + промпт агента | WP-151 + WP-218 |
| **Загрузка документов** | 3-5 документов в личную базу знаний через knowledge-mcp | ИТ-команда: бэк | WP-187 (готово) |
| **1 агент в контексте** | «Помощник» — читает ЦД и отвечает в контексте через шлюз MCP | ИТ-команда: промпт + MCP | WP-187 + WP-222 |
| **Персональное руководство (шаблон)** | 3-5 шаблонов по сегментам + ручная адаптация 10-15 мин | Методолог (шаблоны) + Служба координации (адаптация) | WP-149 + WP-245 |
| **Единый вход через бота** | Единый вход (Ory SSO) + подписка шлюза, ввод в Telegram | ИТ-команда: бэк | WP-73, WP-231 (готово) |

## Что MVP НЕ включает (откладывается до Q3)

- Обучающиеся агенты
- Автогенерация руководства через ИИ-модели
- Интеграции Gmail / Calendar / Notion / Obsidian
- Мультиагентные сценарии
- Веб-приложение (пока бот + VS Code)
- Англоязычная версия

## Критерий готовности MVP (1 мая)

- [ ] Новый пользователь с 1 мая: бот → ввод ЦД → 3-5 документов → персональное руководство → диалог с Помощником в контексте
- [ ] Менеджер обрабатывает каждого нового пользователя за 15-20 мин (адаптация руководства)
- [ ] Удержание на 30-й день первой когорты (1-15 мая) замеряется к 30 мая. Цель: >60%
