# Приложение 3. Продукт и MVP

> Приложение к [Стратегии экосистемы 2026](Стратегия%20экосистемы%202026.md).

## Что такое IWE

**IWE (Intellectual Work Environment)** — персональная рабочая среда созидателя, в которой методология системного мышления встроена как невидимая архитектура. Пользователь получает единую точку, где его знания, его контекст и ИИ-агенты работают связно, а не разрозненно в 10 инструментах.

Снаружи IWE выглядит как современный ИИ-ассистент. Внутри — методология, цифровой двойник и персональный маршрут развития.

## Три опоры продукта

1. **Цифровой двойник (ЦД)** — единая память о пользователе: роль, цели, характеристики деятеля (agency, qualification, mastery), документы, история. Обновляется из активности в IWE + через синхронизатор.
2. **Персональное руководство** — маршрут развития «под меня»: где я сейчас (ступень 1-5), что делать на неделе, какие методы практиковать. Живой документ, адаптируется по мере роста.
3. **ИИ-агенты через MCP** — работают с контекстом пользователя. Навигатор, Диагност, Помощник, Портной (генератор руководств), Оркестратор. Единый Knowledge Gateway.

## MVP к 1 мая 2026

**Идея MVP:** минимальный IWE, который удерживает новую когорту D30 > 60%. Если не удерживает — любой холодный канал утечёт, привлечение бессмысленно.

| Компонент | Что включает | Кто делает | Статус |
|---|---|---|---|
| **Онбординг ЦД** | Диалог 15-20 мин: роль, цели, контекст работы, уровень ИИ-зрелости, 2-3 актуальные задачи | ИТ-команда: фронт + промпт агента | WP-151 + WP-218 |
| **Загрузка документов** | 3-5 документов в личную базу знаний через knowledge-mcp | ИТ-команда: бэк | WP-187 (готово) |
| **1 агент в контексте** | «Помощник» — читает ЦД и отвечает в контексте через Gateway MCP | ИТ-команда: промпт + MCP | WP-187 + WP-222 |
| **Персональное руководство (шаблон)** | 3-5 шаблонов по сегментам + ручная адаптация 10-15 мин | Методолог (шаблоны) + Служба координации (адаптация) | WP-149 + WP-245 |
| **Единый вход через бота** | Ory SSO + подписка Gateway, онбординг в TG | ИТ-команда: бэк | WP-73, WP-231 (готово) |

## Что MVP НЕ включает (откладывается до Q3)

- Обучающиеся агенты
- Автогенерация руководства через LLM
- Интеграции Gmail / Calendar / Notion / Obsidian
- Мультиагентные сценарии
- Web App (пока бот + VS Code)
- EN-версия

## Критерий готовности MVP (1 мая)

- [ ] Новый пользователь с 1 мая: бот → онбординг ЦД → 3-5 документов → персональное руководство → диалог с Помощником в контексте
- [ ] Менеджер обрабатывает каждого нового пользователя за 15-20 мин (адаптация руководства)
- [ ] D30 первой когорты (1-15 мая) замеряется к 30 мая. Цель: >60%
