# Система баллов и лояльности

Баллы — внутренняя валюта экосистемы Aisystant. Начисляются за активные действия, тратятся на программные продукты. Вместо скидок и крипто-токенов — простая, прозрачная модель.

## 1. Принцип

> **Баллы не покупаются — они зарабатываются через вклад.**

| Свойство | Описание |
|----------|----------|
| **Назначение** | Признание вклада участника, мотивация активного участия и апгрейда тира |
| **Единица** | 1 балл ≈ $1 |
| **Начисление** | За активные действия (обучение, сообщество, проекты) |
| **Трата** | Полная или частичная оплата программных продуктов |
| **Хранение** | PostgreSQL (Neon), таблица `finance.points_ledger` |

Баллы — **не** криптовалюта, **не** стейблкоин, **не** токен. Это внутренний механизм лояльности, аналогичный бонусным баллам банков или авиамилям.

## 2. Начисление баллов

### 2.1. Таблица весов

Source-of-truth: [Тарифы и пакеты 2.1](../../A2.Systems-Builder/A2.1.Meaning/2.1.1.%20Продуктовые%20предложения/Тарифы%20и%20пакеты%202.1.md) §4.

| Действие | Баллы | Лимит/день | Описание |
|----------|-------|------------|----------|
| Завершённый слот обучения | 5-10 | 10 | Прочитал раздел, прослушал лекцию |
| Завершённый модуль | 20-50 | 3 | Завершил логический блок курса |
| Выполнение задания | 10-30 | 5 | Упражнение в рабочей тетради |
| Перенос в практику (артефакт) | 20-100 | 2 | Создал рабочий продукт на основе знания |
| Peer-review чужой работы | 10-50 | 3 | Рецензия на артефакт другого участника |
| Вклад в сообщество (пост, кейс) | 10-30 | 2 | Публикация в Клубе (Discourse) |
| Проверка ДЗ (наставник) | 5-20 | — | Фиксированный, без множителя |
| Проект / рабочий продукт | 50-500 | 1 | Завершённый проект, одобренный ментором |

### 2.2. Множитель по тиру (квалификации)

Чем выше тир подписки — тем больше множитель. Это мотивирует апгрейд и активное участие.

| Тир | Множитель | Пример: слот (10 баллов) |
|-----|-----------|--------------------------|
| T2 Изучение | x1.0 | 10 баллов |
| T3 Персонализация | x1.5 | 15 баллов |
| T4 Созидание | x2.0 | 20 баллов |

> T1 (бесплатный) — баллы не начисляются (нет подписки).
> Наставнические действия (проверка ДЗ) — фиксированный размер, без множителя.

### 2.3. Формула

```
Баллы за действие = Вес действия × Множитель тира
Баллы за неделю = Σ(Баллы за каждое действие в рамках лимитов)
```

### 2.4. Пример расчёта

**Участник:** T3 Персонализация (множитель x1.5)

| Действие | Кол-во | Вес | Множитель | Баллы |
|----------|--------|-----|-----------|-------|
| Завершённые слоты | 5 | 10 | x1.5 | 75 |
| Завершённый модуль | 1 | 30 | x1.5 | 45 |
| Задания | 3 | 20 | x1.5 | 90 |
| Артефакт (перенос в практику) | 1 | 50 | x1.5 | 75 |
| Пост в Клубе | 1 | 20 | x1.5 | 30 |

**Итого за неделю: 315 баллов** (~$315 эквивалент)

## 3. Трата баллов

Баллами можно полностью или частично оплачивать **программные продукты** (Слой 2 из [Тарифов](../../A2.Systems-Builder/A2.1.Meaning/2.1.1.%20Продуктовые%20предложения/Тарифы%20и%20пакеты%202.1.md)).

| Продукт | Цена | Эквивалент в баллах |
|---------|------|---------------------|
| Практикум (4-8 нед) | $300-500 | 300-500 |
| Резидентура (6-12 нед) | $800-2000 | 800-2000 |
| Семинар (2-4 ч) | ₽5000-12000 | 50-120 |
| Менторская сессия | Договорная | По каталогу |

### 3.1. Правила траты

- Минимальная трата: 10 баллов
- Можно комбинировать: часть баллами + часть деньгами
- Баллы **не конвертируются** в деньги (нет вывода)
- Баллы **не передаются** между участниками (не P2P)
- Баллы **не сгорают** (бессрочные, пока активна подписка)

### 3.2. При отмене подписки

- Баланс баллов замораживается (не сгорает)
- При возобновлении подписки — баланс восстанавливается
- Начисление прекращается на период отсутствия подписки

## 4. Архитектура (подсистема #11: Loyalty / Proof-of-Impact)

### 4.1. Место в платформе

```
Activity Hub (#7)          Event Bus (#8)            Loyalty (#11)
──────────────────  event  ─────────────────  event  ──────────────────
 Агрегирует          ──►   Маршрутизирует     ──►    Начисляет баллы
 события                    по подписчикам            обновляет баланс
 (обучение, клуб,                                     записывает в
 проекты)                                              audit log
```

### 4.2. Модель данных (PostgreSQL / Neon)

```sql
-- Схема: finance

CREATE TABLE finance.points_ledger (
    id          BIGSERIAL PRIMARY KEY,
    user_id     UUID NOT NULL REFERENCES public.users(id),
    amount      INTEGER NOT NULL,          -- + начисление, - списание
    balance     INTEGER NOT NULL,          -- баланс после операции
    action_type VARCHAR(50) NOT NULL,      -- 'section_completed', 'purchase', ...
    source_id   VARCHAR(100),             -- ID события-источника
    tier        VARCHAR(10),              -- 'T2', 'T3', 'T4'
    multiplier  NUMERIC(3,1) DEFAULT 1.0,
    created_at  TIMESTAMPTZ DEFAULT now(),
    metadata    JSONB                     -- доп. контекст
);

CREATE INDEX idx_points_user ON finance.points_ledger(user_id);
CREATE INDEX idx_points_created ON finance.points_ledger(created_at);

-- Представление: текущий баланс
CREATE VIEW finance.points_balance AS
SELECT user_id,
       SUM(amount) AS balance,
       COUNT(*) FILTER (WHERE amount > 0) AS total_earnings,
       COUNT(*) FILTER (WHERE amount < 0) AS total_spendings
FROM finance.points_ledger
GROUP BY user_id;
```

### 4.3. Источники событий

| Источник | События | Как получаем |
|----------|---------|-------------|
| **LMS Aisystant** | SectionCompleted, ExerciseSave, CourseCompleted | API / webhook |
| **Discourse (Клуб)** | PostCreated, CommentCreated | Webhook |
| **Бот (Telegram)** | TrainingCompleted, ArtifactSubmitted | Event Bus |
| **Web App** | ProjectMilestone, PeerReview | Event Bus |

### 4.4. Процесс начисления

1. Участник совершает действие (например, завершает слот)
2. Activity Hub фиксирует событие
3. Event Bus доставляет событие в Loyalty Service
4. Loyalty Service:
   - Проверяет дневной лимит по типу действия
   - Определяет тир участника → множитель
   - Рассчитывает баллы
   - Записывает в `points_ledger` (append-only)
5. Уведомление участнику (через Notification Service)

### 4.5. Audit log

Каждая запись в `points_ledger` — это и есть audit log (append-only ledger). Никогда не UPDATE, только INSERT. Баланс = SUM(amount).

## 5. Proof-of-Impact: перспектива

Баллы — первый уровень. В перспективе (Phase 3, 2027+) система эволюционирует:

| Уровень | Что измеряет | Когда |
|---------|-------------|-------|
| **L1: Баллы** | Активность (сделал действие) | Сейчас |
| **L2: Компетенции** | Качество (подтверждённые навыки через ESG) | Phase 2 |
| **L3: Влияние** | Эффект (какой результат принесло знание в проекте) | Phase 3 |

Эпистемический граф (ESG, подсистема #12) — зависимость для L2. Пока ESG не построен, работаем на L1 (чистые баллы за действия).

## 6. Связь с Revenue Sharing

Баллы и Revenue Sharing — **две разные системы**:

| | Баллы | Revenue Sharing |
|---|-------|----------------|
| **Кому** | Участникам (ученики, интеллектуалы) | Создателям контента (авторы, наставники, кураторы) |
| **За что** | Активные действия | Продажу созданного контента |
| **Чем платим** | Внутренние баллы | Реальные деньги ($) |
| **Формула** | Вес × множитель тира | Platform 30%, Author 50%, Instructor 15%, Curator 5% |

## Приложение A. Крипто-токенизация (перспектива, не реализуется)

> Этот раздел сохранён для будущего рассмотрения. Переход к крипто-токену возможен как надстройка поверх системы баллов, если экосистема вырастет до масштаба, оправдывающего сложность.

### A.1. Концепция токена STK

Токен SysTok (STK) — потенциальный крипто-токен для экосистемы, построенный на принципе: «Награждаем не за потребление (покупки), а за вклад».

**Функции (потенциальные):**

| Функция | Описание |
|---------|----------|
| Средство расчёта | Оплата обучающих программ, доступа к ИИ-ассистентам |
| P2P платежи | Расчёты между участниками за услуги и менторство |
| Единица ценности | Измерение реального вклада участника в экосистему |
| Средство доступа | Персональные ИИ-агенты, премиум-функции |

### A.2. Эмиссия

```
Initial supply: 1 000 000 000 STK
Halving: каждые 4 года
```

| Период | Годовая эмиссия | Дневная эмиссия |
|--------|-----------------|-----------------|
| Год 0-3 | 1 000 000 000 | 2 739 726 |
| Год 4-7 | 500 000 000 | 1 369 863 |
| Год 8-11 | 250 000 000 | 684 932 |

Распределение: Rewards Pool 50%, Treasury 20%, Startup Fund 15%, Team 10%, Liquidity 5%.

### A.3. Блокчейн

- Стандарт: SPL Token (Solana)
- Кошельки: Phantom, Solflare
- Преимущества: высокая пропускная способность, низкие комиссии

### A.4. Дополнительные механизмы (не реализованы)

- **Vesting:** заморозка баллов до подтверждения эффекта (30-90 дней по типу)
- **Soft slashing:** прогрессивное перераспределение от неактивных (5-15%/квартал)
- **Стабилизация:** buyback, burn, emission adjustment
- **DeFi:** cross-chain bridges, стейкинг, DAO governance

### A.5. Условия перехода к крипто

Переход оправдан, когда:
1. Экосистема > 10 000 активных участников
2. Объём внутренних транзакций > $100K/мес
3. Потребность в P2P-расчётах между участниками
4. Юридическая ясность (регуляторика)

До этого момента — система баллов (§1-8) полностью покрывает потребности.
