# Описание клуба

Социальная подсистема экосистемы — место для тренировки мышления письмом, вхождения в культуру и публичной демонстрации вклада. Клуб функционирует как **открытый репозиторий активностей участника**, где каждое действие оставляет след, формируя видимую историю роста и влияния на мир с полной трассировкой эпистемического статуса.

## Назначение

Клуб — это одновременно:
- **Песочница для новичков** — безопасное место для практики системного мышления через письмо
- **Витрина достижений** — публичная демонстрация вклада и эпистемического статуса всех участников
- **Социальный граф экосистемы** — сеть связей между участниками, идеями, проектами
- **Источник репутации** — все активности вознаграждаются токенами и влияют на статус

### Ключевая метафора

> Клуб — это как GitHub для интеллектуального развития: не код, а мысли, не коммиты, а публикации, не pull requests, а взаимная критика. История активности полностью открыта и показывает путь от новичка до эксперта.

### Три роли Клуба

1. **Обучающая среда**
   - Практика мышления письмом (think-by-writing)
   - Безопасное пространство для экспериментов
   - Быстрая обратная связь от сообщества
   - Вхождение в культуру и нормы экосистемы

2. **Витрина репутации**
   - Эпистемический статус участников виден всем
   - История вклада прослеживается до первой публикации
   - Связи между идеями, проектами, результатами
   - Доказательства компетенций через артефакты

3. **Токеномическая мотивация**
   - Все действия вознаграждаются токенами по Proof-of-Impact
   - Качественные публикации получают больше вознаграждений
   - Peer review и критика также вознаграждаются
   - История заработка привязана к истории роста

## Архитектура

### Компоненты

#### 1. Publication Engine (Движок публикаций)

**Назначение:** Создание, публикация и версионирование контента.

**Типы публикаций:**
- **Posts** — короткие заметки, размышления, гипотезы
- **Articles** — развёрнутые статьи, исследования
- **Logs** — логи практики (daily logs, weekly reviews)
- **Comments** — комментарии к публикациям других
- **Reviews** — рецензии на артефакты (peer review)

**Форматы:**
- Markdown с расширениями (таблицы, диаграммы, формулы)
- Вложения (изображения, PDF, Jupyter notebooks)
- Embedded контент (видео, презентации)

**Функции:**
- Черновики и публикация
- Версионирование (подобно Git)
- Связи между публикациями (citations, references)
- Теги и категории

**Интеграции:**
- Memory Bank (4.12) — сохранение как Episteme
- Epistemic Graph (4.16) — присвоение статусов
- Activity Hub (4.6) — фиксация событий публикации

#### 2. Reputation Tracker (Трекер репутации)

**Назначение:** Отслеживание и визуализация репутации участников.

**Компоненты репутации:**
- **Эпистемический статус** — качество знаний и доказательств
- **Activity Score** — объём и регулярность вклада
- **Impact Score** — влияние на других и проекты
- **Peer Rating** — оценки от сообщества

**Метрики:**
- Количество публикаций по типам
- Качество публикаций (на основе peer review)
- Цитирования и использование другими
- Вклад в проекты (через кейсы)
- Менторство и помощь другим

**Визуализация:**
- Профиль участника с историей активности
- График роста эпистемического статуса
- Карта связей (кто с кем взаимодействует)
- Timeline вклада (хронология достижений)

**Интеграции:**
- Proof-of-Impact (4.17) — расчёт impact score
- Digital Twin (4.11) — обновление профиля
- Epistemic Graph (4.16) — статусы артефактов

#### 3. Social Graph (Социальный граф)

**Назначение:** Управление связями между участниками.

**Типы связей:**
- **Following** — подписка на публикации
- **Mentorship** — отношения ментор-ученик
- **Collaboration** — совместная работа над проектами
- **Peer Review** — взаимная критика
- **Citation** — ссылки на работы других

**Функции:**
- Лента активности (feed) от подписок
- Рекомендации участников для follow
- Поиск менторов и экспертов
- Discovery новых публикаций и проектов

**Интеграции:**
- CRM (4.7) — управление контактами
- Activity Hub (4.6) — события взаимодействий

#### 4. Discourse Platform (Платформа дискуссий)

**Назначение:** Организация обсуждений, дебатов, критики.

**Форматы:**
- **Threads** — ветки обсуждений под публикациями
- **Debates** — структурированные дебаты с ролями
- **Study Groups** — групповое изучение тем
- **Project Channels** — каналы проектов

**Механики:**
- Голосование за ценные комментарии
- Модерация (community-driven + автоматическая)
- Нотификации и подписки на threads
- Эпистемический статус для комментариев

**Интеграции:**
- Activity Hub (4.6) — фиксация дискуссий
- Proof-of-Impact (4.17) — вознаграждение за вклад в дискуссии

#### 5. Events & Activities (События и активности)

**Назначение:** Организация и проведение событий сообщества.

**Типы событий:**
- **Meetups** — встречи и воркшопы
- **Webinars** — вебинары и лекции
- **Sprints** — интенсивные рабочие сессии
- **Challenges** — челленджи и конкурсы
- **Reviews** — публичные защиты работ

**Функции:**
- Календарь событий
- Регистрация и управление участниками
- Записи и материалы событий
- Feedback и оценки

**Интеграции:**
- LMS (4.4) — связь с обучением
- Case Management (4.13) — связь с проектами
- Activity Hub (4.6) — логирование участия

#### 6. Gamification & Rewards (Геймификация и награды)

**Назначение:** Мотивация через игровые механики и вознаграждения.

**Механики:**
- **Badges** — значки за достижения
- **Streaks** — серии регулярных действий
- **Leaderboards** — рейтинги по метрикам
- **Quests** — челленджи с наградами
- **Levels** — уровни участников (Новичок → Мастер)

**Вознаграждения:**
- Токены за публикации (по Proof-of-Impact)
- Бонусы за качественные peer reviews
- Награды за менторство
- Призы за победы в challenges

**Интеграции:**
- Tokenomics (4.10) — начисление токенов
- Proof-of-Impact (4.17) — расчёт вознаграждений

## Процессы

### П1. Онбординг новичка в Клуб

**Назначение:** Быстрое вхождение нового участника в культуру.

**Этапы:**
1. **Регистрация** — создание профиля через ORY
2. **Welcome Tour** — интерактивный тур по Клубу
3. **Первая публикация** — guided процесс создания первого поста
4. **Знакомство с сообществом** — рекомендации участников для follow
5. **Первые задания** — простые quests для освоения механик

**Метрики:**
- Time to First Post (целевое: <30 минут)
- Конверсия регистрация → первая публикация (>80%)
- Retention Day 7 (>50%)

**Интеграции:**
- ORY (4.14) — identity и согласия
- Guide (4.5) — персональный маршрут онбординга
- Digital Twin (4.11) — инициализация профиля

### П2. Публикация и получение обратной связи

**Назначение:** Основной цикл создания контента и улучшения через critique.

**Этапы:**
1. **Создание черновика**
   - Markdown редактор с подсказками
   - Связывание с другими публикациями
   - Выбор категорий и тегов

2. **Публикация**
   - Выбор видимости (public/followers/draft)
   - Автоматический анализ качества (языковые модели)
   - Присвоение предварительного эпистемического статуса

3. **Распространение**
   - Публикация в ленты подписчиков
   - Рекомендации заинтересованным участникам
   - Кросс-постинг в каналы/проекты

4. **Обратная связь**
   - Комментарии и дискуссии
   - Peer review (если запрошен)
   - Голосования и реакции

5. **Итерация**
   - Обновление публикации на основе feedback
   - Версионирование (подобно Git)
   - Обновление эпистемического статуса

**Вознаграждения:**
- Токены за публикацию (базовая ставка)
- Бонус за качество (peer review rating)
- Бонус за impact (цитирования, использование)
- Вознаграждение рецензентам за critique

**Интеграции:**
- Memory Bank (4.12) — сохранение публикации
- Epistemic Graph (4.16) — статус и связи
- Proof-of-Impact (4.17) — расчёт вознаграждений
- Activity Hub (4.6) — события публикации

### П3. Менторство и помощь

**Назначение:** Передача знаний от опытных участников к новичкам.

**Роли:**
- **Ментор** — опытный участник с высоким статусом
- **Менти** — обучающийся участник
- **Peer** — участник того же уровня (peer support)

**Механики:**
- Запрос менторства (менти → ментор)
- Назначение ментора (автоматически или вручную)
- Сессии менторства (встречи, reviews)
- Feedback loops (двусторонняя обратная связь)

**Метрики:**
- Количество активных менторских пар
- Удовлетворённость менти (ratings)
- Прогресс менти (рост статуса)
- Время ментора (компенсируется токенами)

**Вознаграждения:**
- Токены ментору за время (по SplitRule)
- Бонус за успехи менти
- Reputation boost для ментора

**Интеграции:**
- CRM (4.7) — управление менторскими парами
- Tokenomics (4.10) — выплаты менторам
- Digital Twin (4.11) — обновление ролей

### П4. Организация событий

**Назначение:** Проведение meetups, webinars, sprints.

**Этапы:**
1. **Планирование** — создание события, дата, формат
2. **Анонс** — публикация в Клубе, приглашения
3. **Регистрация** — сбор участников, подтверждения
4. **Проведение** — сама активность (online/offline)
5. **Запись и материалы** — сохранение контента
6. **Feedback** — сбор отзывов и метрик
7. **Follow-up** — задания после события, продолжение

**Роли:**
- **Организатор** — инициатор и координатор
- **Спикеры** — докладчики, эксперты
- **Модераторы** — ведущие дискуссий
- **Участники** — слушатели, активные участники

**Вознаграждения:**
- Токены организатору и спикерам
- Баджи участникам
- Reputation boost для организаторов

**Интеграции:**
- Calendar integration (Google Calendar, iCal)
- Video platform (Zoom, Meet)
- LMS (4.4) — связь с курсами
- Activity Hub (4.6) — фиксация участия

## Хранилище данных

### Схема данных

**Таблицы (PostgreSQL):**

```sql
-- Публикации
CREATE TABLE Post (
  id UUID PRIMARY KEY,
  author_id UUID NOT NULL,
  title VARCHAR NOT NULL,
  content TEXT NOT NULL,
  type ENUM('post', 'article', 'log', 'review'),
  status ENUM('draft', 'published', 'archived'),
  visibility ENUM('public', 'followers', 'private'),
  created_at TIMESTAMP,
  updated_at TIMESTAMP,
  published_at TIMESTAMP,
  tags VARCHAR[],
  metadata JSONB
);

-- Версии публикаций
CREATE TABLE PostVersion (
  id UUID PRIMARY KEY,
  post_id UUID REFERENCES Post(id),
  version INT NOT NULL,
  content TEXT NOT NULL,
  changed_by UUID,
  created_at TIMESTAMP
);

-- Комментарии
CREATE TABLE Comment (
  id UUID PRIMARY KEY,
  post_id UUID REFERENCES Post(id),
  author_id UUID NOT NULL,
  parent_id UUID REFERENCES Comment(id),  -- для nested comments
  content TEXT NOT NULL,
  votes INT DEFAULT 0,
  created_at TIMESTAMP,
  updated_at TIMESTAMP
);

-- Reviews
CREATE TABLE Review (
  id UUID PRIMARY KEY,
  post_id UUID REFERENCES Post(id),
  reviewer_id UUID NOT NULL,
  rating INT CHECK (rating >= 1 AND rating <= 5),
  content TEXT,
  dimensions JSONB,  -- оценки по разным измерениям
  created_at TIMESTAMP
);

-- События
CREATE TABLE Event (
  id UUID PRIMARY KEY,
  organizer_id UUID NOT NULL,
  title VARCHAR NOT NULL,
  description TEXT,
  type ENUM('meetup', 'webinar', 'sprint', 'challenge'),
  start_time TIMESTAMP,
  end_time TIMESTAMP,
  location VARCHAR,  -- physical or online
  max_attendees INT,
  metadata JSONB
);

-- Регистрации на события
CREATE TABLE EventRegistration (
  id UUID PRIMARY KEY,
  event_id UUID REFERENCES Event(id),
  user_id UUID NOT NULL,
  status ENUM('registered', 'attended', 'cancelled'),
  registered_at TIMESTAMP,
  attended BOOLEAN DEFAULT false
);

-- Баджи
CREATE TABLE Badge (
  id UUID PRIMARY KEY,
  name VARCHAR NOT NULL,
  description TEXT,
  icon_url VARCHAR,
  criteria JSONB  -- условия получения
);

-- Баджи участников
CREATE TABLE UserBadge (
  id UUID PRIMARY KEY,
  user_id UUID NOT NULL,
  badge_id UUID REFERENCES Badge(id),
  earned_at TIMESTAMP,
  evidence_id UUID  -- ссылка на доказательство
);
```

**Граф (Neo4j):**

```cypher
// Узлы
(:User {id, name, email, joined_at, level})
(:Post {id, title, type, published_at})
(:Event {id, title, start_time})
(:Tag {name})
(:Badge {id, name})

// Связи
(:User)-[:FOLLOWS]->(:User)
(:User)-[:POSTED]->(:Post)
(:User)-[:COMMENTED_ON]->(:Post)
(:User)-[:REVIEWED]->(:Post)
(:User)-[:ATTENDED]->(:Event)
(:User)-[:EARNED]->(:Badge)
(:User)-[:MENTORS]->(:User)
(:Post)-[:TAGGED_WITH]->(:Tag)
(:Post)-[:REFERENCES]->(:Post)
(:Post)-[:BUILDS_ON]->(:Post)
```

### Retention policy

- **Публикации:** Бессрочно (архивирование старых)
- **Комментарии:** Бессрочно
- **События:** 2 года (архив бессрочно)
- **Activity logs:** 1 год (агрегаты бессрочно)

### Алерты

**Критические:**
- Platform unavailable
- Database connection lost
- Massive spam detected
- Security breach

**Предупреждения:**
- Declining engagement (DAU/MAU trend down)
- Low peer review rate (<20%)
- High bounce rate for newcomers (>50%)
- Event cancellation spike
- Moderation queue overflow (>100 items)
