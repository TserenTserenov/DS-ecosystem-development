# Описание системы биллинга

Система управления подписками, тарифами, доступами и распределением доходов. Центральная точка монетизации экосистемы — управляет всеми финансовыми отношениями между участниками: от студентов и преподавателей до менторов и разработчиков контента. Реализует принципы FPF: эволюционность через гибкие тарифы и программу лояльности, сквозную целостность от покупки до распределения доходов, дидактическую прозрачность каждой транзакции и начисления.

## Назначение

Система биллинга — это **финансовый координатор экосистемы**, который превращает платежи в доступы, а доходы — в мотивацию для всех участников.

### Ключевые функции

1. **Управление тарифами и подписками**
   - Гибкие тарифные планы (freemium, subscription, pay-per-course)
   - Подписки с trial periods и grace periods
   - Upgrade/downgrade с prorated billing
   - Семейные и корпоративные планы
   - Промокоды, скидки, акции

2. **Управление доступами (Access Control)**
   - Выдача доступа к контенту на основе оплаты
   - Feature gating (функции по тарифам)
   - Quota management (лимиты по тарифам)
   - Временные доступы (trial, limited access)
   - Автоматическое отключение при неоплате

3. **Программа лояльности**
   - Бонусы за повторные покупки
   - Скидки для активных участников
   - Реферальная программа (cashback за приглашения)
   - Накопительные скидки (чем больше курсов, тем больше скидка)

4. **Распределение доходов (Revenue Sharing)**
   - Автоматические расчёты с создателями контента (авторы, преподаватели)
   - Split rules (доли распределения между ролями)
   - Менторские выплаты (за сопровождение стажёров)
   - Роялти за продажи курсов
   - Прозрачный dashboard для всех ролей

5. **Финансовая отчётность и юнит-экономика**
   - Детализация доходов по продуктам, каналам, когортам
   - Метрики: MRR, ARR, LTV, CAC, churn
   - Прогнозирование выручки
   - Налоговая отчётность
   - Выгрузки для бухгалтерии

### Роль в архитектуре

Биллинг — центр монетизации:
- Получает платежи от **Payment Gateway** (4.8)
- Управляет тарифами и доступами для всех систем (**LMS**, **AI OS**, **Club**)
- Рассчитывает и начисляет доходы создателям через **Tokenomics** (4.10)
- Синхронизирует подписки с **CRM** (4.7) для customer success
- Логирует все billing события в **Activity Hub** (4.6)

#### 1. Subscription Manager (Менеджер подписок)

**Назначение:** Управление lifecycle подписок участников.

**Функции:**
- **Subscription Creation** — создание подписки при оплате
- **Plan Management** — управление тарифными планами
- **Lifecycle Management** — trialing, active, past_due, cancelled, expired
- **Renewal Handling** — автоматическое продление подписок
- **Change Management** — upgrade/downgrade с proration

**Типы подписок:**
- **Freemium** — бесплатный доступ с ограничениями
- **Basic** — базовая подписка (ограниченный доступ)
- **Standard** — стандартная подписка (полный доступ к курсам)
- **Premium** — премиум подписка (все курсы + ментор + AI features)
- **Enterprise** — корпоративный план (для компаний/вузов, custom pricing)

**Proration logic:**
```python
def calculate_proration(old_plan, new_plan, days_used, days_total):
    # При upgrade: immediate charge за разницу
    if new_plan.price > old_plan.price:
        unused_value = (old_plan.price / days_total) * (days_total - days_used)
        new_period_value = (new_plan.price / days_total) * (days_total - days_used)
        charge_now = new_period_value - unused_value
        return charge_now

    # При downgrade: credit на следующий период
    else:
        credit = ((old_plan.price - new_plan.price) / days_total) * (days_total - days_used)
        return -credit  # negative означает credit
```

- **Trial Access** — временные доступы для trial users
- **Access Revocation** — отзыв доступа при неоплате

**Access matrix (пример):**

| AI Assistant | ❌ | 10 calls/day | 50 calls/day | Unlimited | Unlimited |
| Personal Guide | ❌ | Basic | Advanced | Advanced + mentors | Custom paths |

**Check access flow:**
```python
def check_access(user_id, resource_type, resource_id):
    subscription = get_user_subscription(user_id)

    if subscription.status not in ['active', 'trialing']:
        return AccessDenied("Subscription inactive")

    plan = subscription.plan

    # Feature gating
    if resource_type == 'ai_assistant':
        quota = get_quota(user_id, 'ai_calls', period='day')
        if quota.consumed >= plan.limits['ai_calls_per_day']:
            return AccessDenied("Quota exceeded")
        increment_quota(user_id, 'ai_calls')

    # Course access
    if resource_type == 'course':
        if plan.course_access == 'all':
            return AccessGranted()
        elif has_purchased_course(user_id, resource_id):
            return AccessGranted()
        else:
            return AccessDenied("Course not in subscription")

    return AccessGranted()
```

**Loyalty program logic:**
```python
def calculate_loyalty_discount(user_id):
    total_spent = get_total_lifetime_spend(user_id)
    courses_completed = get_courses_completed_count(user_id)

    # Tier based on spend
    if total_spent >= 1000:
        tier_discount = 0.15  # 15% для VIP
    elif total_spent >= 500:
        tier_discount = 0.10  # 10% для Gold
    elif total_spent >= 200:
        tier_discount = 0.05  # 5% для Silver
    else:
        tier_discount = 0

    # Bonus за completed courses
    completion_bonus = min(courses_completed * 0.01, 0.10)  # до 10% максимум

    return min(tier_discount + completion_bonus, 0.25)  # максимум 25% скидка
```

#### 4. Revenue Distribution Engine (Движок распределения доходов)

**Назначение:** Автоматические расчёты и выплаты создателям контента.

- **Revenue Calculation** — расчёт доходов для каждой роли
- **Payout Scheduling** — планирование выплат (monthly, quarterly)
- **Withholding** — удержания (platform fee, taxes)
- **Transparency Dashboard** — прозрачный dashboard для всех

**Роли в распределении доходов:**
- **Course Author** — разработчик курса (контент, методология)
- **Instructor** — преподаватель потока (ведение, проверка ДЗ)
- **Mentor** — ментор стажёра (личное сопровождение)
- **Curator** — куратор программы (дизайн траектории)
- **Platform** — комиссия платформы

**Split rules (пример для курса):**
```
Цена курса: $100
- Platform fee: 30% → $30
- Course Author: 50% → $50
- Instructor: 15% → $15
- Curator: 5% → $5

При менторском сопровождении (Premium):
Цена: $200
- Platform fee: 25% → $50
- Course Author: 35% → $70
- Instructor: 15% → $30
- Mentor: 20% → $40
- Curator: 5% → $10
```

**Payout flow:**
```python
def calculate_monthly_payout(creator_id, month):
    # 1. Собрать все продажи за месяц
    sales = get_sales_for_creator(creator_id, month)

    # 2. Применить split rules
    total_revenue = 0
    for sale in sales:
        split_rule = get_split_rule(sale.product_id, creator_id)
        creator_share = sale.amount * split_rule.percentage
        total_revenue += creator_share

    # 3. Вычесть withholdings
    platform_fee = total_revenue * 0.05  # admin fee
    tax_withholding = total_revenue * get_tax_rate(creator_id)

    net_payout = total_revenue - platform_fee - tax_withholding

    # 4. Создать payout request
    create_payout(creator_id, net_payout, month)

    return net_payout
```

**Cohort analysis example:**
```python
def cohort_retention_analysis(cohort_month):
    # 1. Получить всех пользователей, зарегистрированных в cohort_month
    users = get_users_registered_in(cohort_month)

    # 2. Для каждого месяца после регистрации посчитать retention
    retention_by_month = {}
    for month_offset in range(0, 12):  # первые 12 месяцев
        active_count = count_active_subscriptions(
            users,
            cohort_month + month_offset
        )
        retention_pct = active_count / len(users)
        retention_by_month[month_offset] = retention_pct

    return retention_by_month
```

5. **Payment Success** — Payment Gateway подтверждает оплату
6. **Grant Access** — Billing выдаёт доступ к курсу
7. **Notify Creator** — уведомление создателю о продаже
8. **Calculate Revenue Split** — расчёт долей для автора/платформы
9. **Send Receipt** — отправка чека на email
10. **Enroll in LMS** — автоматическая запись на курс в LMS

**Revenue split:**
```
Цена курса: $100
- Platform: $30
- Author: $50
- Instructor (если есть поток): $15
- Referrer (если был реферал): $5
```

### П2. Подписка на платформу (subscription)

**Назначение:** Оформление recurring подписки.

**Этапы:**
1. **Select Plan** — выбор тарифа (Basic, Standard, Premium)
2. **Trial Setup** — если есть trial, установка trial period
3. **First Payment** — оплата через Payment Gateway
4. **Create Subscription** — создание subscription entity
5. **Grant Access** — выдача доступа ко всем функциям плана
6. **Onboarding** — trigger onboarding sequence в CRM
7. **Schedule Renewal** — запланировать следующее списание

**Trial handling:**
```
Trial period: 14 дней
- Day 0: Trial starts, full access
- Day 11: Email reminder (3 дня до окончания)
- Day 13: Last email reminder (1 день до окончания)
- Day 14: Автоматическое списание или требование добавить карту
- Если payment fails: grace period 3 дня
```

**Интеграции:**
- Payment Gateway (4.8) — recurring payments

- All systems (LMS, AI OS, Club) — выдача доступов
- CRM (4.7) — onboarding campaigns

### П3. Upgrade/downgrade подписки

**Назначение:** Изменение тарифного плана.

**Этапы (Upgrade):**
1. **Select New Plan** — пользователь выбирает более дорогой план
2. **Calculate Proration** — расчёт доплаты (prorated billing)
3. **Charge Difference** — immediate charge за разницу
4. **Update Subscription** — обновление subscription record
5. **Grant New Access** — выдача доступа к новым функциям
6. **Notify User** — email с подтверждением upgrade

**Этапы (Downgrade):**
1. **Select Lower Plan** — пользователь выбирает более дешёвый план
2. **Schedule Downgrade** — downgrade применится в конце текущего периода
3. **Calculate Credit** — расчёт credit на следующий период
4. **Notify User** — email с подтверждением (downgrade в конце периода)
5. **On Period End** — применить downgrade и revoke доступы

**Proration example:**
```
Current plan: Standard ($50/month)
New plan: Premium ($100/month)
Days used: 10 days out of 30

Unused value of old plan: ($50/30) * 20 = $33.33
Value of new plan for remaining days: ($100/30) * 20 = $66.67
Charge now: $66.67 - $33.33 = $33.34
```

**Интеграции:**
- Payment Gateway (4.8) — прием дополнительного платежа
- All systems — обновление access прав

### П4. Месячный расчёт с создателями (monthly payout)

6. **Generate Payout Statements** — формирование отчётов
7. **Send Notifications** — email с отчётом и суммой к выплате
8. **Initiate Payouts** — инициация выплат через Payment Gateway или Tokenomics
9. **Confirm Payouts** — подтверждение выплат

**Transparency dashboard для создателя:**

- Доля в каждой продаже (split breakdown)

**Интеграции:**

```
GET  /billing/creators/:id/payouts         # История выплат
POST /billing/payouts/initiate             # Инициация payout
```

**Invoices:**
```
GET  /billing/invoices/:id                 # Получить счёт
GET  /billing/invoices/user/:id            # Счета пользователя
POST /billing/invoices/:id/pay             # Оплатить счёт
```

### События

**Subscription Events:**
- `billing.subscription.created` — подписка создана
- `billing.subscription.trial_started` — начался trial
- `billing.subscription.activated` — подписка активирована
- `billing.subscription.upgraded` — upgrade плана
- `billing.subscription.downgraded` — downgrade плана
- `billing.subscription.renewed` — подписка продлена
- `billing.subscription.cancelled` — подписка отменена
- `billing.subscription.expired` — подписка истекла

**Purchase Events:**
- `billing.purchase.completed` — покупка завершена
- `billing.access.granted` — доступ выдан
- `billing.access.revoked` — доступ отозван

**Revenue Events:**
- `billing.revenue.allocated` — доход распределён по создателям
- `billing.payout.scheduled` — выплата запланирована
- `billing.payout.completed` — выплата завершена

**Quota Events:**
- `billing.quota.exceeded` — превышена квота
- `billing.quota.reset` — квота сброшена (новый период)

## Хранилище данных

### Схема данных

**Таблицы (PostgreSQL):**

```sql
-- Тарифные планы
CREATE TABLE Plan (
  id UUID PRIMARY KEY,
  name VARCHAR NOT NULL,
  tier VARCHAR,  -- freemium, basic, standard, premium, enterprise
  price DECIMAL NOT NULL,
  currency VARCHAR(3) DEFAULT 'USD',
  interval VARCHAR,  -- month, year
  features JSONB,  -- список фич
  limits JSONB,  -- лимиты (ai_calls, storage, etc)
  active BOOLEAN DEFAULT true,
  created_at TIMESTAMP
);

-- Подписки пользователей
CREATE TABLE UserSubscription (
  id UUID PRIMARY KEY,
  user_id UUID NOT NULL,
  plan_id UUID REFERENCES Plan(id),
  status VARCHAR,  -- trialing, active, past_due, cancelled, expired
  current_period_start TIMESTAMP,
  current_period_end TIMESTAMP,
  trial_end TIMESTAMP,
  cancel_at_period_end BOOLEAN DEFAULT false,
  cancelled_at TIMESTAMP,
  created_at TIMESTAMP,
  updated_at TIMESTAMP
);

-- Покупки курсов (one-time)
CREATE TABLE Purchase (
  id UUID PRIMARY KEY,
  user_id UUID NOT NULL,
  product_type VARCHAR,  -- course, bundle, etc
  product_id UUID NOT NULL,
  price DECIMAL NOT NULL,
  currency VARCHAR(3),
  discount DECIMAL DEFAULT 0,
  final_price DECIMAL NOT NULL,
  promo_code VARCHAR,
  purchased_at TIMESTAMP,
  access_expires_at TIMESTAMP  -- NULL для lifetime access
);

-- Доступы пользователей
CREATE TABLE UserAccess (
  id UUID PRIMARY KEY,
  user_id UUID NOT NULL,
  resource_type VARCHAR,  -- course, feature, ai_assistant, etc
  resource_id UUID,
  source VARCHAR,  -- subscription, purchase, trial, promo
  granted_at TIMESTAMP,
  expires_at TIMESTAMP,
  active BOOLEAN DEFAULT true
);

-- Квоты
CREATE TABLE Quota (
  id UUID PRIMARY KEY,
  user_id UUID NOT NULL,
  resource_type VARCHAR,  -- ai_calls, storage, compute_hours
  limit_value DECIMAL,
  consumed DECIMAL DEFAULT 0,
  period_start TIMESTAMP,
  period_end TIMESTAMP,
  reset_schedule VARCHAR  -- daily, weekly, monthly
);

-- Split Rules (правила распределения доходов)
CREATE TABLE SplitRule (
  id UUID PRIMARY KEY,
  product_id UUID NOT NULL,
  role VARCHAR,  -- author, instructor, mentor, curator, platform
  user_id UUID,  -- конкретный человек (или NULL для роли)
  percentage DECIMAL NOT NULL,  -- доля в %
  active BOOLEAN DEFAULT true,
  created_at TIMESTAMP
);

-- Распределение доходов (revenue allocations)
CREATE TABLE RevenueAllocation (
  id UUID PRIMARY KEY,
  purchase_id UUID REFERENCES Purchase(id),
  recipient_id UUID NOT NULL,
  role VARCHAR,
  amount DECIMAL NOT NULL,
  currency VARCHAR(3),
  status VARCHAR,  -- pending, processed, paid
  payout_id UUID,
  created_at TIMESTAMP
);

-- Выплаты создателям
CREATE TABLE Payout (
  id UUID PRIMARY KEY,
  recipient_id UUID NOT NULL,
  amount DECIMAL NOT NULL,
  currency VARCHAR(3),
  period_start DATE,
  period_end DATE,
  status VARCHAR,  -- scheduled, processing, completed, failed
  payment_method VARCHAR,  -- bank_transfer, paypal, crypto, tokens
  paid_at TIMESTAMP,
  created_at TIMESTAMP
);

-- Промокоды
CREATE TABLE PromoCode (
  id UUID PRIMARY KEY,
  code VARCHAR UNIQUE NOT NULL,
  type VARCHAR,  -- percentage, fixed_amount, trial_extension
  value DECIMAL,
  max_uses INT,
  used_count INT DEFAULT 0,
  valid_from TIMESTAMP,
  valid_until TIMESTAMP,
  active BOOLEAN DEFAULT true
);
```

### Retention policy

- **Subscriptions:** Бессрочно (пока активны), 7 лет после cancellation
- **Purchases:** 7 лет (налоговые требования)
- **Invoices:** 7 лет
- **Revenue Allocations:** 7 лет
- **Payouts:** 7 лет
- **Promo Codes:** 2 года после expiration

### Алерты

**Критические:**
- Billing system unavailable
- Failed subscription renewals spike (>10%)
- Payout processing failure
- Access control system down
- Database connection lost

**Предупреждения:**
- High churn rate (>7%)
- Low upgrade rate (<5%)
- Failed payment rate increasing (>5%)
- Creator payout delays (>7 дней)
- Promo code abuse detected
- Quota system slow (>1 sec response)
