# Описание распределителя токенов

Система автоматического начисления и распределения токенов за активности участников экосистемы. Получает данные из Цифрового двойника об активных действиях (завершение курсов в LMS, публикации в Клубе, участие в проектах), применяет алгоритм расчёта вознаграждений на основе Proof-of-Impact (цепочка Work → Evidence → Metrics), и переводит токены на кошельки пользователей в блокчейне Solana. Использует окна регрессии для валидации эффекта и автоматически распределяет вознаграждения между авторами, менторами и treasury через SplitRule.

## Назначение

Распределитель токенов — это **экономический мост между активностями участников и блокчейном**, который автоматически вознаграждает участников за их вклад в экосистему.

### Ключевые функции

1. **Сбор данных о активностях из Цифрового двойника**
   - Активные действия из Клуба (публикации, комментарии, peer reviews)
   - Образовательные достижения из LMS (завершение курсов, выполнение заданий, сертификации)
   - Вклад в проекты (через систему управления кейсами)
   - Метрики личного развития (OKR, навыки, прогресс)
   - Агрегированные показатели из Activity Hub

2. **Алгоритм расчёта вознаграждений (Proof-of-Impact)**
   - Трассировка Work → Evidence → Metrics — полная прозрачность от работы до измеримых результатов
   - Расчёт веса доказательств на основе эпистемического статуса
   - Оценка влияния (impact), а не просто активности
   - Применение мультипликаторов (качество, сложность, социальный эффект)

3. **Валидация и окна регрессии**
   - Окна регрессии (60-90 дней) — валидация эффекта во времени
   - Обнаружение ложно-позитивных результатов
   - Частичная заморозка вознаграждений до подтверждения эффекта

4. **Распределение токенов на кошельки в Solana**
   - Автоматический перевод токенов на кошельки пользователей в блокчейне Solana
   - Использование SPL Token Standard
   - Low fees, высокая пропускная способность
   - On-chain прозрачность всех транзакций
   - SplitRule для автоматического разделения вознаграждений (автор/ментор/treasury)

5. **Защита от злоупотреблений**
   - Бюджеты и политики для ограничения начислений
   - Конформанс-чеклисты для валидации
   - Обнаружение аномалий (fraud detection)
   - Ручная модерация подозрительных случаев

6. **Интеграция с экосистемой**
   - Валидация доказательств через Эпистемический граф
   - Синхронизация с Финансовой системой токеномики
   - Обновление балансов в Цифровом двойнике

## Архитектура

### Компоненты

1. **Impact Tracker**
   - Сбор Work из Activity Hub
   - Агрегация событий в цепочки (workflows)
   - Идентификация ключевых точек измерения (metrics checkpoints)
   - Трекинг прогресса по артефактам

2. **Evidence Validator**
   - Валидация доказательств через ESG (4.16)
   - Проверка наличия evidence-bindings
   - Расчёт веса доказательств (weight calculation)
   - Определение уровня уверенности (confidence scoring)

3. **Metrics Engine**
   - Измерение конкретных показателей (KPI, OKR)
   - Связывание метрик с Work
   - Автоматический сбор метрик (API integration)
   - Агрегация метрик во времени

4. **Regression Window Manager**
   - Управление окнами валидации (60-90 дней)
   - Мониторинг изменений метрик в окне
   - Обнаружение регрессий и ложно-позитивных результатов
   - Автоматическое закрытие окон

5. **Token Distributor**
   - Расчёт вознаграждений на основе весов доказательств
   - Применение SplitRule (автор/ментор/treasury)
   - Частичная заморозка до закрытия окна регрессии
   - Интеграция с Token Economy (4.10) для начисления
   - Формирование транзакций для Solana

6. **Solana Integration (Интеграция с блокчейном)**
   - Управление кошельками пользователей (SPL Token)
   - Создание и отправка транзакций в сеть Solana
   - Мониторинг статуса транзакций (confirmation)
   - Обработка ошибок и retry logic
   - Синхронизация on-chain балансов с off-chain базой данных
   - Поддержка multi-signature для крупных транзакций treasury

7. **Anti-Gaming System**
   - Бюджеты и политики для ограничения начислений
   - Конформанс-чеклисты для валидации
   - Обнаружение аномалий (fraud detection)
   - Ручная модерация подозрительных случаев

### Технологический стек

- **Backend:** Python (метрики, ML), Go (API, distributor), Rust (validator, Solana integration)
- **Frontend:** React dashboard для визуализации impact цепочек
- **База данных:** PostgreSQL (трассы, метрики), TimescaleDB (временные ряды), Neo4j (граф связей Work→Evidence→Metrics)
- **Blockchain:** Solana (on-chain транзакции), SPL Token Standard, Anchor Framework (смарт-контракты)
- **Wallet Integration:** Phantom, Solflare, Sollet (кошельки пользователей)
- **Инфраструктура:** Kubernetes, Kafka (событийный поток), Airflow (периодические джобы для окон регрессии), RPC nodes (Solana cluster)

## API и контракты

### Основные эндпоинты

**Impact Tracking:**
```
GET  /poi/work/:id/impact-chain              # Цепочка Work → Evidence → Metrics
POST /poi/work/:id/submit                    # Отправка Work на оценку impact
GET  /poi/work/:id/status                    # Статус оценки impact
GET  /poi/user/:id/impact-score              # Общий impact score пользователя
```

**Evidence & Metrics:**
```
POST /poi/work/:id/link-evidence             # Связывание Work с доказательствами
POST /poi/work/:id/add-metric                # Добавление метрики к Work
GET  /poi/work/:id/evidence-weight           # Вес доказательств
GET  /poi/work/:id/metrics                   # Все метрики по Work
```

**Regression Windows:**
```
GET  /poi/window/:id/status                  # Статус окна регрессии
GET  /poi/window/:id/metrics-trend           # Тренд метрик в окне
POST /poi/window/:id/close                   # Закрыть окно вручную
GET  /poi/windows/active                     # Активные окна регрессии
```

**Token Distribution:**
```
POST /poi/work/:id/calculate-reward          # Расчёт вознаграждения
GET  /poi/work/:id/split-rule                # Правило разделения
POST /poi/work/:id/distribute                # Распределение токенов
GET  /poi/user/:id/pending-rewards           # Замороженные вознаграждения
GET  /poi/user/:id/earned-tokens             # Заработанные токены
```

**Admin & Governance:**
```
GET  /poi/suspicious-activities              # Подозрительные активности
POST /poi/activity/:id/review                # Ручная проверка
GET  /poi/stats                              # Статистика по системе
POST /poi/policy/update                      # Обновление политик
```

### События

Система генерирует следующие события:
- `poi.work.submitted` — Work отправлен на оценку
- `poi.evidence.linked` — доказательство привязано к Work
- `poi.metric.added` — метрика добавлена
- `poi.weight.calculated` — вес доказательств рассчитан
- `poi.window.opened` — открыто окно регрессии
- `poi.window.closed` — окно регрессии закрыто
- `poi.reward.calculated` — вознаграждение рассчитано
- `poi.reward.distributed` — токены распределены
- `poi.reward.unfrozen` — вознаграждение разморожено
- `poi.anomaly.detected` — обнаружена аномалия
- `poi.regression.detected` — обнаружена регрессия метрик

## Хранилище данных

### Схема данных

Основные сущности и их поля:

**Impact Chain (граф в Neo4j):**
- **WorkNode**: id, user_id, type, created_at, submitted_at, status
- **EvidenceNode**: id, artifact_id, esg_status, weight, confidence
- **MetricNode**: id, metric_type, value, measured_at, source
- **Link (Edge)**: work_id → evidence_id, evidence_id → metric_id, weight, created_at

**Таблицы (PostgreSQL):**
- **ImpactSubmission**: id, work_id, user_id, submitted_at, status, evidence_count, total_weight
- **RegressionWindow**: id, work_id, opened_at, closes_at, status, initial_metrics (JSON), final_metrics (JSON)
- **RewardCalculation**: id, work_id, total_tokens, author_share, mentor_share, treasury_share, calculated_at
- **TokenDistribution**: id, reward_id, user_id, amount, frozen, unfrozen_at, distributed_at
- **SplitRule**: id, work_type, author_pct, mentor_pct, treasury_pct, conditions (JSON)
- **AnomalyDetection**: id, work_id, anomaly_type, confidence, detected_at, reviewed, resolution

**Временные ряды (TimescaleDB):**
- **MetricTimeSeries**: timestamp, work_id, metric_type, value, window_id

### Retention policy

- **Активные трассы (Work → Evidence → Metrics):** Бессрочно
- **Закрытые окна регрессии:** 2 года
- **История начислений:** 5 лет (для налоговой отчётности)
- **Аномалии:** 3 года
- **Детальные метрики:** 1 год (агрегированные — бессрочно)

### Ключевые метрики

| Avg time Work → Reward distribution | <7 дней | — |
| Avg regression window duration | 60-90 дней | — |
| False positive rate (регрессии) | <5% | — |
| Avg evidence weight per Work | >0.6 | — |
| Total tokens distributed per month | [целевое] | — |
| Fraud detection accuracy | >95% | — |

### Алерты

**Критические:**
- Reward distribution pipeline остановлен
- Database unavailable
- Kafka stream lag > 10 минут
- Fraud detection down

**Предупреждения:**
- Regression window closing rate < expected
- High volume of low-weight evidence (possible gaming)
- Anomaly detection queue > 50 items
- Token distribution delay > 24 часа
- Avg evidence weight declining trend
