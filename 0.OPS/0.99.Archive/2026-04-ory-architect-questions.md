## Вопрос 1: Какой реальный URL у ORY?

`auth.aisystant.com` возвращает 404 на всех endpoint'ах (`/oauth2/auth`, `/.well-known/openid-configuration` и др.).

**Нужно:** актуальный базовый URL Ory Hydra (публичный endpoint).

**Зачем:** бот уже использует `auth.aisystant.com` в коде — если URL изменился, нужно обновить. WP-187 Ф0 не может начаться без рабочего endpoint.

## Вопрос 3: Как из Ory получить статус подписки пользователя?

Бот должен при запросе T2+ функции проверить: есть ли у пользователя активная подписка БР.

Варианты реализации разные:
- **А.** Subscription status хранится в Kratos traits (профиль пользователя) — бот читает через `/sessions/whoami`
- **Б.** Это проверка через Ory Keto (check: `user X has permission Y`) — отдельный HTTP-запрос
- **В.** Ory здесь вообще не нужен — бот проверяет подписку напрямую в Neon

**Нужно:** какой вариант принят архитектором.

**Зачем:** от этого зависит реализация paywall в WP-187 (Бот + Ory).
