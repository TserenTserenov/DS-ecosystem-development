# ТЗ Рекомендации по развитию хранилища 0.4

## 0. Назначение документа

Техническое задание для ИИ по формированию отчёта «Рекомендации по развитию». Отчёт агрегирует данные из всех предыдущих отчётов и формирует приоритизированный бэклог действий по улучшению хранилища.

---

## 1. Запуск генерации отчёта

### 1.1. Автоматический запуск

**Расписание:** Каждое воскресенье в 08:00 по московскому времени (05:00 UTC)

**Механизм:** GitHub Actions workflow

**Порядок генерации:** Этот отчёт генерируется **последним**, после всех остальных отчётов, так как агрегирует их данные.

### 1.2. Ручной запуск

```bash
python3 .ops/build_report.py --report recommendations
```

или через общий скрипт:

```bash
./.ops/build_all_reports.sh
```

## 2. Тепловая карта статуса

**ВАЖНО:** Общая оценка здоровья рассчитывается на основе взвешенных показателей из всех отчётов. Нельзя присваивать высокий статус без анализа всех критериев.

В начале отчёта обязательно размещается тепловая карта общего здоровья хранилища:

| Индикатор | Значение | Описание |
|-----------|----------|----------|
| 🟢 | **Здоровое** | **ВСЕ критерии:** ≥90 полных документов (каждый >500 слов с примерами, диаграммами, не заглушки) И средняя взвешенная оценка по всем показателям ≥90 баллов И ≥80% документов имеют текстовые связи (wikilinks) с другими документами И <10 критических проблем (🔴) И тренд улучшения (рост показателей за 3 периода) |
| 🟡 | **Требует внимания** | 60-89 полных документов (с примерами) ИЛИ средняя оценка 75–89 баллов ИЛИ 40-79% документов имеют связи ИЛИ 10-30 критических проблем ИЛИ тренд стабильный или незначительное улучшение |
| 🔴 | **Критическое** (по умолчанию) | **ЛЮБОЕ из условий:** <60 полных документов ИЛИ средняя оценка <75 баллов ИЛИ <40% документов имеют связи ИЛИ >30 критических проблем (🔴) ИЛИ тренд ухудшения (снижение показателей) ИЛИ наличие блокирующих проблем |

**Формат тепловой карты в отчёте:**

```markdown
## Тепловая карта здоровья хранилища

| Измерение | Оценка | Вес | Вклад | Тренд | Статус |
|-----------|--------|-----|-------|-------|--------|
| Полнота | 85% | 25% | 21.25 | ↑ +2% | 🟡 |
| Связность | 78% | 20% | 15.6 | → 0% | 🟡 |
| Согласованность | 72% | 25% | 18.0 | ↓ -3% | 🟡 |
| Терминология | 88% | 15% | 13.2 | ↑ +4% | 🟡 |
| Актуальность | 76% | 15% | 11.4 | → 0% | 🟡 |
| **Итого** | — | 100% | **79.45** | — | 🟡 |

**Общая оценка: 79/100** 🟡 Требует внимания
```

### 2.1. Показатель динамики улучшения

В отчёте обязательно включается раздел с анализом динамики по сравнению с предыдущими отчётами:

```markdown
## Динамика здоровья хранилища

| Измерение | 2 периода назад | Предыдущий | Текущий | Тренд 3 периода |
|-----------|-----------------|------------|---------|-----------------|
| Полнота | 83% | 85% | 85% | → Стабильно |
| Связность | 76% | 78% | 78% | ↑ Улучшение |
| Согласованность | 75% | 72% | 72% | ↓ Ухудшение |
| Терминология | 84% | 88% | 88% | ↑ Улучшение |
| Актуальность | 74% | 76% | 76% | ↑ Улучшение |
| **Общий балл** | **77.4** | **79.8** | **79.45** | **↑ Незначительное улучшение** |

**Анализ тренда:**
- **Краткосрочный тренд (1 период):** ↓ Незначительное ухудшение (79.8 → 79.45)
- **Долгосрочный тренд (3 периода):** ↑ Улучшение (77.4 → 79.45, +2.05 балла)
- **Самое улучшающееся измерение:** Терминология (+4% за 3 периода)
- **Самое ухудшающееся измерение:** Согласованность (-3% за 3 периода)
```

## 4. Анализ (процесс обработки)

### 4.1. Алгоритм формирования

1. **Сбор метрик из отчётов**
   - Извлечь ключевые метрики из каждого отчёта
   - Рассчитать оценку по каждому измерению (0–100%)
   - Вычислить взвешенную общую оценку

2. **Агрегация проблем**
   - Собрать все проблемы из отчётов
   - Удалить дубликаты
   - Классифицировать по типу и критичности

3. **Приоритизация рекомендаций**
   - Оценить влияние каждой проблемы
   - Оценить трудоёмкость исправления
   - Рассчитать приоритет = влияние / трудоёмкость
   - Ранжировать по приоритету

4. **Формирование бэклога**
   - Критические (эта неделя): влияние ≥ 8, трудоёмкость ≤ 4
   - Важные (этот месяц): влияние ≥ 5
   - Желательные (бэклог): остальные

5. **Анализ трендов**
   - Сравнить с предыдущими отчётами (4 недели)
   - Рассчитать изменение каждой метрики
   - Выявить улучшения и ухудшения

### 4.2. Формула расчёта приоритета

```
Приоритет = (Влияние × Критичность) / (Трудоёмкость × Риск)

где:
- Влияние: 1–10 (сколько документов/пользователей затрагивает)
- Критичность: 1–3 (🟢=1, 🟡=2, 🔴=3)
- Трудоёмкость: 1–10 (оценка в часах)
- Риск: 1–3 (риск сломать что-то при исправлении)
```
