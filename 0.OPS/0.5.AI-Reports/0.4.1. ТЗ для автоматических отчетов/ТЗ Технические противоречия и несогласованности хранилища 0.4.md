# ТЗ Технические противоречия и несогласованности хранилища 0.4

## 0. Назначение документа

Техническое задание для ИИ по формированию отчёта «Технические противоречия и несогласованности хранилища». Отчёт фокусируется **исключительно на технических** проблемах хранилища:

- Дублирование названий папок и документов
- Дублирование текста между документами
- Битые wikilinks
- Несогласованность метаданных (YAML frontmatter)
- Нарушения структуры папок
- Пустые папки и документы без frontmatter

**ВАЖНО: Разделение ответственности между отчётами**

| Тип проблемы | Где проверяется |
|--------------|-----------------|
| **Технические** (структура, ссылки, метаданные) | Этот отчёт |
| **Содержательные** (терминологические, логические, фактические противоречия) | [[ТЗ Содержательная полнота описания 0.4]] |
| **Ошибки классификации** (документ не для той аудитории) | [[ТЗ Содержательная полнота описания 0.4]] |

Содержательные противоречия (когда один документ утверждает одно, а другой — противоположное) и ошибки классификации по целевой аудитории проверяются в отчёте [[ТЗ Содержательная полнота описания 0.4]].

## 3. Вход (исходные данные)

### 3.1. Источники данных

| Источник | Путь | Что извлекаем |
|----------|------|---------------|
| Все документы хранилища | `content/**/*.md` | Пути, содержимое, wikilinks |
| Структура папок | `content/` | Имена папок, иерархия |
| Frontmatter документов | YAML frontmatter | type, status, family, cell, created |
| Стандарты | `Структура хранилища 0.1.md` | Ожидаемая структура папок |
| История Git | `git log` | Даты изменений |

### 3.2. Типы технических проблем для проверки

| Тип | Описание | Приоритет |
|-----|----------|-----------|
| **Дублирование названий папок** | Папки с одинаковыми или похожими названиями в разных местах | 🔴 Критический |
| **Дублирование названий документов** | Файлы с одинаковыми названиями в разных папках | 🔴 Критический |
| **Дублирование текста** | Одинаковые или очень похожие (>80%) фрагменты текста в разных документах | 🟡 Важный |
| **Битые wikilinks** | Ссылки `[[...]]` на несуществующие документы | 🟡 Важный |
| **Несогласованность метаданных** | Отсутствие обязательных полей, неверные значения | 🟡 Важный |
| **Нарушения структуры папок** | Документы в неправильных разделах, неверная нумерация | 🟡 Важный |
| **Пустые папки** | Папки без документов | 🟢 Низкий |
| **Документы без frontmatter** | Файлы без YAML-метаданных | 🟡 Важный |

### 4. Критерии оценки статуса

| Тип проблемы | 🟢 Чисто (проверено содержание) | 🟡 Требует внимания (документы частичные) | 🔴 Критично (по умолчанию, нет полных документов) |
|--------------|----------|---------------------|-------------|
| Дублирование названий папок | 0 + проверено содержание | 1-2 + содержание похоже но не идентично | ≥3 ИЛИ дубль имеет полное идентичное содержание |
| Дублирование названий документов | 0 + проверено содержание | 1-5 + содержание имеет 50-90% совпадений | ≥6 ИЛИ содержание >90% идентично |
| Дублирование текста (>200 символов) | 0 + текст проанализирован на смысл | 1-10 + есть переформулировки похожего смысла | ≥11 ИЛИ текст идентичен дословно |
| Битые wikilinks | 0 + все ссылки актуальны | 1-5 + целевые документы существуют но устаревшие | ≥6 ИЛИ целевые документы не существуют |
| Несогласованность метаданных | 0 + метаданные полные | 1-10 + не все метаданные заполнены но связаны корректно | ≥11 ИЛИ метаданные противоречивы с содержанием |
| Нарушения структуры папок | 0 + структура соответствует стандартам | 1-2 + структура частично не соответствует | ≥3 ИЛИ структура не соответствует вообще |
| Пустые папки | ≤2 + остальные папки с документами | 3-5 + есть папки без полных документов | ≥6 ИЛИ папки только с заглушками |
| Документы без frontmatter | 0 + все документы имеют полные метаданные | 1-5 + большинство документов имеют frontmatter | ≥6 ИЛИ frontmatter не структурирован вообще |

**Общий статус:** Определяется по наиболее критичному типу проблем. Если есть хотя бы один 🔴 тип, общий статус 🔴. Если нет 🔴 но есть 🟡, статус 🟡. Если все типы 🟢, статус 🟢.

## 4. Анализ (процесс обработки)

### 4.1. Алгоритм проверки дублирования названий папок

1. **Сбор всех папок**
   - Рекурсивный обход `content/`
   - Извлечение названий папок (без номера: `0.1. Логика хранилища` → `Логика хранилища`)

2. **Поиск дублей**
   - Точное совпадение названий
   - Fuzzy matching (расстояние Левенштейна ≤ 2)
   - Нормализация: приведение к нижнему регистру, удаление спецсимволов

3. **Формирование отчёта**
   - Группировка дублей с указанием полных путей
   - Рекомендация по объединению или переименованию

### 4.2. Алгоритм проверки дублирования названий документов

1. **Сбор всех документов**
   - Извлечение имён файлов без расширения и номера (`Концепция созидателя 2.2.md` → `Концепция созидателя`)

2. **Поиск дублей**
   - Точное совпадение названий
   - Fuzzy matching (сходство >90%)
   - Учёт синонимов из глоссария

3. **Классификация дублей**
   - Полный дубль: одинаковое название в разных папках
   - Частичный дубль: похожее название (например, `Описание X` и `X описание`)

### 4.3. Алгоритм проверки дублирования текста

1. **Извлечение контента**
   - Парсинг документов, удаление frontmatter
   - Разбиение на абзацы (>200 символов)

2. **Сравнение абзацев**
   - Хеширование (simhash) для быстрого поиска кандидатов
   - Детальное сравнение кандидатов (косинусное сходство >0.8)
   - Исключение: цитаты, таблицы стандартных форматов

3. **Формирование отчёта**
   - Группировка дублей с указанием документов и строк
   - Рекомендация: вынести в отдельный документ или использовать ссылку

### 4.4. Алгоритм проверки wikilinks

1. **Извлечение всех ссылок**
   - Поиск паттерна `[[...]]` во всех документах
   - Нормализация: удаление alias (`[[Doc|Alias]]` → `Doc`)

2. **Проверка существования**
   - Поиск документа по названию (fuzzy)
   - Учёт переименований через git history

3. **Классификация проблем**
   - Битая ссылка: документ не существует
   - Неточная ссылка: найден документ с похожим названием
   - Устаревшая ссылка: документ переименован

### 4.5. Алгоритм проверки метаданных

1. **Обязательные поля YAML**
   - `type` — тип документа
   - `status` — статус (active/draft/obsolete)
   - `created` — дата создания

2. **Валидация значений**
   - `type` ∈ {doc, decision, index, guide, spec, ...}
   - `status` ∈ {active, draft, obsolete}
   - `family` ∈ {F0, F1, ..., F9}
   - `created` — корректная дата

3. **Проверка согласованности**
   - Соответствие `family` расположению в папке
   - Соответствие `cell` номеру раздела

### 4.6. Алгоритм проверки структуры папок

1. **Ожидаемая структура**
   - Корневые папки: 0, 1, 2, 3
   - Подразделы с номерами: X.Y. Название
   - Документы с суффиксом раздела: `Документ X.Y.md`

2. **Проверки**
   - Документ в правильном разделе (по family/cell)
   - Нумерация папок последовательна
   - Нет папок без индекса

## 8. Рекомендации по устранению

### 8.1. Критические (устранить немедленно) 🔴

1. **Объединить дублирующиеся папки**
   - Действие: Переместить документы, удалить дубль
   - Затронуто документов: X
   - Потребует обновления ссылок: Y

2. **Исправить нумерацию папок**
   - Действие: Переименовать `0.4.1. Структура автоматических отчётов по хранилищу/`
   - Предложение: `0.4.2. Архив структуры отчётов/`

1. **Убрать дублирование текста**
   - Определить каноническое место
   - Заменить дубли на ссылки

2. **Исправить битые ссылки**
   - Автофикс для опечаток
   - Ручная проверка удалённых документов

### 8.3. Желательные (бэклог) 🟢

1. Добавить недостающие метаданные
2. Удалить пустые папки
