<summary><b>1. Проблема: два мира</b></summary>

Сейчас оплаты хранятся в двух несвязанных базах. Бот не знает об оплатах через Tilda/Монету. Бухгалтер и маркетолог не видят оплаты бота. Декан и преподаватели не видят наполняемость групп. Нет единого окна.

```
МИР 1: Aisystant (Java LMS)              МИР 2: Бот (Python)
|- YooKassa (подписки, потоки)            |- YooKassa (семинары витрины)
|- Paybox                                 |- TG Stars (семинары)
|- Stripe (Aisystant Corp)                +- Neon: seminar_payments
|- Монета/PayAnyWay
|- Tilda/Ecwid
+- Aisystant PostgreSQL

         X НЕ СВЯЗАНЫ X
```

**Последствия:**
- Оплата через Tilda -> Алёна вручную отправляет invite -> задержка, ошибки
- Гиляна не видит оплаты через бота (Stars, ЮКасса семинаров)
- Маршрутка (кик неоплативших) невозможна -- бот не знает всех оплативших
- Юля и преподаватели не видят наполняемость потоков и семинаров в реальном времени
- Нет единого MRR/LTV -- данные в разных местах

<summary><b>2.1. Lifecycle семинара</b></summary>

Семинар проходит два режима. Переключение -- вручную или автоматически по дате завершения.

### Режим 1: Live (семинар идёт)

- Многоразовая пригласительная ссылка («маршрутка»)
- Все оплатившие вступают по этой ссылке
- **Накануне семинара (или за пару часов):** бот кикает неоплативших из чата
- После кика -- можно вступить по ссылке только после оплаты (тот же механизм, что при продаже записи: оплата -> одноразовая ссылка)

### Режим 2: Запись (семинар завершился)

- **Отключить** для всех кроме админов возможность добавлять/приглашать новых участников и генерировать вступительные ссылки
- Вступление **только по одноразовой ссылке** (после оплаты)
- Многоразовая ссылка деактивируется

### Переключение режима

| Событие | Действие бота |
|---------|--------------|
| Семинар завершился (дата или ручная команда) | 1. Отозвать многоразовую ссылку. 2. Отключить приглашения для не-админов (`setChatPermissions`). 3. Пометить семинар `mode: recording` |
| Новый семинар запланирован | Создать многоразовую ссылку, включить приглашения, `mode: live` |

### Два варианта допуска (оба рабочие)

| Вариант | Кто | Как | Бот знает? |
|---------|-----|-----|-----------|
| **A. Через бота** | Покупатель оплачивает -> бот генерирует одноразовую ссылку (`createChatInviteLink`, `member_limit=1`) | Автоматически | Да -- бот создал ссылку, видит вступление |
| **B. Через админа** | Админ создаёт одноразовую ссылку через интерфейс TG -> отправляет покупателю | Вручную | Да -- бот получает `chat_member` update с `invite_link.creator` = админ. Помечает `source: admin_invite` |

**Предпочтительный вариант: A** (автоматический учёт). Вариант B -- для исключительных случаев (ручной допуск).

<summary><b>3. Целевая схема (to-be, с Ory)</b></summary>

**Принципы:**
- **Не дублировать данные** -- каждый source-of-truth остаётся на месте
- **Ory = единый аккаунт** -- один аккаунт связывает Telegram, email и аккаунт Aisystant (заменяет ручную склейку `/link`)
- **Ory = единое управление доступом** -- платформа решает, кто к чему имеет доступ (чаты, функции, контент)

### Единый аккаунт (Ory)

Сейчас один человек может иметь 3 отдельных «аккаунта»: Telegram, email на сайте, Aisystant. Склейка -- вручную через `/link` в боте.

С Ory -- **один аккаунт на человека**:
- Регистрация на сайте -> Ory создаёт аккаунт (email)
- `/start` в боте -> Ory связывает telegram_id с тем же аккаунтом
- Привязка Aisystant -> миграция из LMS в тот же аккаунт

Все каналы оплаты видят одного человека. Нет путаницы «оплатил на сайте, но бот не знает».

### Архитектура

```
                         Ory (аккаунты + доступы)
                                |
                    +-----------+-----------+
                    v           v           v
            Aisystant PG     Neon        Directus
            (Payment         (семинары   (CRM UI)
             Registry)        бота)
                    |           |           |
                    +-----+-----+           |
                          v                 |
                    Metabase (аналитика) <---+
                          |
                          |-> Гиляна: все оплаты, сверка
                          |-> Алёна: воронка T0->T4, конверсия, когорты
                          |-> Юля: наполняемость потоков и семинаров
                          +-> Преподаватели: списки участников

              Activity Hub (WP-109)
                          |
                          +-> Billing adapter: платёж = событие
                              -> engagement -> ЦД -> аналитика
```

### Webhook-контракт (Aisystant -> бот)

Когда человек платит через Tilda/Монету/сайт, Aisystant уведомляет бот:

```
POST https://aistmebot-production.up.railway.app/webhook/workshop-payment
Header: X-Webhook-Secret: {секрет}

{
  "ory_id": "uuid",              // единый аккаунт (когда Ory развёрнут)
  "telegram_id": 123456789,      // до Ory -- Telegram ID (если известен)
  "tg_username": "username",     // опционально, для отправки в личку
  "email": "user@example.com",   // email покупателя
  "purpose": "SEMINAR",          // WORKSHOP | SEMINAR | INTERNSHIP
  "seminar_code": "SE-2026.2-T", // код семинара
  "stream_code": "INT-2026.3",   // код потока (если применимо)
  "amount": 5000,
  "currency": "RUB",
  "payment_id": "yookassa_xxx",  // защита от дублей
  "source": "tilda"              // tilda | moneta | yookassa | stripe | paybox | manual
}
```

**До Ory:** бот принимает `telegram_id` и/или `tg_username`. **После Ory:** бот получает `ory_id` и сам находит telegram_id через Ory.

### Обратный webhook (бот -> Aisystant)

Когда человек вступает в чат по одноразовой ссылке, бот уведомляет Aisystant:

```
POST https://aisystant.com/api/webhook/chat-join
Header: X-Webhook-Secret: {секрет}

{
  "telegram_id": 123456789,
  "chat_id": -100123456789,
  "seminar_code": "SE-2026.2-T",
  "invite_source": "bot" | "admin_invite",
  "joined_at": "2026-04-04T12:00:00Z"
}
```

Aisystant использует для связки tg_id с аккаунтом (до Ory -- по email, если известен; после Ory -- через Ory).

### Воронка входа (для маркетинга)

```
T0 (анонимный)
  | /start в боте или заход на сайт
  | -> Ory создаёт аккаунт
  v
T1 (зарегистрирован, без подписки)
  | Trial 30 дней -- полный доступ
  | -> бесплатные семинары, марафон
  v
T2 (подписка БР)
  | Оплата: YooKassa / Stripe / Монета / Stars
  | -> доступ: лента, консультации, заметки, планы, Сообщество IWE
  v
T3 (ЦД подключён)
  | Бесплатно -- заполнить цифровой двойник
  v
T4 (экзокортекс)
  | Бесплатно -- setup.sh / Gateway URL
```

**Metabase считает по Ory:** сколько на каждом тире, конверсия T0->T1->T2, retention, churn.

### Виды доступов и как они выдаются

| Что получает человек | При каком событии | Как выдаётся сейчас | Как будет с Ory |
|---------------------|-------------------|--------------------|--------------| 
| **Подписка БР** (лента, консультации, заметки) | Оплата подписки | Бот проверяет API Aisystant (cache 5мин) | Ory знает: тир = T2 -> доступ автоматически |
| **Чат Сообщество IWE** | Подписка БР активна | Бот одобряет join request | Ory: есть подписка -> одобрить |
| **Чат семинара** (оплата через бот) | Оплата Stars/ЮКасса | Бот сразу шлёт invite | Без изменений (уже работает) |
| **Чат семинара** (оплата через сайт) | Оплата на сайте | Гиляна/Алёна вручную | Webhook от Aisystant -> бот шлёт invite |
| **Видео семинара** | Оплата семинара | Ссылка в сообщении после оплаты | Без изменений |
