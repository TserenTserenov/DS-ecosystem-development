## Нерассмотренные вопросы → воскресенье (13 апр)

> Встреча 5 сфокусировалась на разделении Россия/мир и CRM. Ниже — вопросы, которые не обсуждались.

### 1. Инфраструктура за границей (~15 мин)

**Контекст (из встречи 5):** Принято решение разделять инфраструктуру Россия/мир. Решение по серверу в России принято (платформа Андрея). **Решение по инфраструктуре за границей — не принято.**

| Сервис | Зачем | Статус |
|--------|-------|--------|
| Activity Hub | Write-heavy, Railway дорого | Андрей: не AI-агент → не его платформа. Нужен отдельный сервер |
| Langfuse | Трейсинг prod AI-вызовов | Аналогично — не AI-агент |
| Agent Runner | Серверные агенты (Scout, ночные) | Подходит под платформу Андрея |

**Вопросы:**
1. Инфраструктура за границей: облако (какое?) или свой сервер?
2. Activity Hub + Langfuse — куда, если не на платформу Андрея?
3. Сетевая связность: RU-сервер ↔ зарубежный сервер ↔ Neon EU

### 2. Ory/Keto: статус Андрея (~5 мин)

**Контекст (из встречи 5):** Андрей сообщил, что клуб (DE-36) в работе у Натальи. Как только клуб готов — можно продвинуться.

**Вопросы (остались):**
- **Keto:** задеплоен? URL для `POST /check`?
- **Q-keto:** карта permissions (6 ns, 45 perm, 11 role-groups) — ревью с Пашей?
- **WP-187/WP-210:** Проверка подписки: Kratos traits vs Keto check? Единая точка auth для бота + VS Code + Web

### 3. Безопасность: архитектурные решения (~15 мин)

**Не обсуждалось.** Вопросы остаются:
1. **Encryption at rest:** pgcrypto / app-level / Vault?
2. **Inter-service auth:** стандартизуем?
3. Сертификации — SOC 2, ISO 27001?
4. **NEW (9 апр): Neon Authorize + Gateway + CI — согласование концепции мультитенантной изоляции данных.** Инцидент: утечка персональных данных через fallback в personal-knowledge-mcp. Подготовлена концепция Defense-in-Depth (3 слоя), АрхГейт пройден (7✅/0⚠️). Blocking dep: Ory JWT strategy. Связи: WP-212, WP-214, WP-215, WP-187, WP-121. → [Концепция](../../C.IT-Platform/C2.IT-Platform/C2.2.Architecture/Data-Governance/neon-authorize-rls-concept.md)
