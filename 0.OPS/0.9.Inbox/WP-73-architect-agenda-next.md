## 3. ADR по identity: формализация решения ory_id (~10 мин)

**Суть:** На встрече 4 принято ключевое решение — `ory_id` как единственный универсальный ID. Нужно формализовать как ADR.

**Решение (7 апр):**
- НЕ добавлять `lms_user_id` в Ory traits
- Везде переделать на `ory_id` (бот, ЦД, будущие системы)
- Если сервису нужен `lms_user_id` → запрашивать через LMS API по Ory-токену
- Каждый сервис создаёт локальную учётку при первом входе пользователя с `ory_id`

**Что нужно:** подтвердить текст ADR, Паша — план реализации в боте/ЦД.
