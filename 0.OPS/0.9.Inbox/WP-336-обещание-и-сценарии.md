# WP-336: Обещание пользователю и сценарии использования

---

<details open>
<summary><b>Обещание платформы пользователю</b></summary>

Aisystant IWE помогает специалисту работать системнее и расти профессионально — не через абстрактные курсы, а через поддержку в реальной работе.

**Что платформа гарантирует:**

- **Ты работаешь быстрее.** Hermes помнит твои паттерны ошибок и предлагает правильный метод в нужный момент — не после, а во время работы.
- **Ты видишь свой рост.** Платформа отслеживает реальные действия и показывает, как меняется твоё мастерство по шести измерениям (ритм, системное мышление, навыки, инструменты, интеграция, инициатива).
- **Ты контролируешь свои данные.** Ни один агент не имеет доступа ко всему сразу. Каждый сервис работает только в своей зоне. Ты можешь экспортировать, удалить или ограничить любой доступ в любой момент.
- **Интерфейс растёт вместе с тобой.** На ступени 1 — одна кнопка и рецепт. На ступени 5 — полная конфигурация, собственные инструменты, участие в развитии методов.
- **Ты не теряешь контекст.** Hermes помнит, что ты делал в прошлой сессии, какие методы освоил, что шло сложно. Следующая сессия начинается не с нуля.

**Что платформа НЕ делает:**

- Не заменяет твоё мышление — усиливает его (экзоскелет, не автопилот)
- Не хранит переписку и личные данные в агентах
- Не навязывает темп — ты сам решаешь, сколько и когда

<summary><b>Сценарии для Ученика (ступени 1–3)</b></summary>

### Сценарий У.1 — Первый день на платформе (ступень 1: Случайный)

**Кто:** специалист, только зарегистрировавшийся. Никогда не слышал о системном мышлении.

**Что происходит:**
1. Открывает LMS — видит один курс «С чего начать», одну задачу на день
2. Hermes предлагает: «Сегодня попробуй зафиксировать одно решение, которое принял на работе»
3. Человек пишет заметку прямо в IWE (VS Code или браузер)
4. Платформа фиксирует событие `capture_created` → начинается отсчёт ритма
5. В конце дня — короткий `day_close`: что сделал, что узнал. 3 минуты

**Ценность:** человек сделал первый осознанный шаг — без лекций, без теории, в процессе своей работы.

---

### Сценарий У.2 — Застрял на задаче (ступень 2: Практикующий)

**Кто:** пользователь, который ведёт еженедельный ритм, но сейчас потерял нить в большом проекте.

**Что происходит:**
1. Открывает IWE, пишет Hermes: «Я не знаю с чего начать, проект слишком большой»
2. Hermes находит паттерн: раньше похожая ситуация разрешилась декомпозицией на роли
3. Hermes предлагает чек-лист: «Кто владелец проблемы? Какой артефакт нужен на выходе?»
4. Пользователь отвечает на 3 вопроса → появляется первый конкретный шаг
5. Фиксируется `method_practiced` → cp_skl растёт

**Ценность:** из состояния «не знаю с чего начать» → конкретное действие за 10 минут.

---

### Сценарий У.3 — Прогресс невидим (ступень 2–3)

**Кто:** пользователь занимается 2 месяца, но чувствует «стою на месте».

**Что происходит:**
1. Открывает дашборд → видит динамику cp_rhy (ритм) за 8 недель
2. Видит: ритм вырос с 1.2 до 2.8 — это объективно
3. Hermes: «Твой bottleneck сейчас — cp_iwe (применение инструментов). Вот 2 задачи, которые это прокачают»
4. Пользователь выбирает задачу → ставит в план на неделю

**Ценность:** субъективное «стою на месте» заменяется объективной картиной роста и конкретным следующим шагом.

</details>

<summary><b>Сценарии для Профессионала (ступени 3–4)</b></summary>

### Сценарий П.1 — Применение методов в реальном проекте (ступень 3: Систематический)

**Кто:** менеджер, который освоил базовые методы и хочет применить системный подход к своей команде.

**Что происходит:**
1. Открывает IWE в VS Code во время подготовки к встрече с командой
2. Hermes видит контекст: «Ты готовишь встречу по проекту X»
3. Предлагает шаблон: «Определи проблему → роли → критерии успеха»
4. Менеджер заполняет — появляется структурированная повестка
5. После встречи фиксирует результат → `artifact_committed`
6. cp_int (интеграция) растёт: человек применяет методы не в учёбе, а в реальной работе

**Ценность:** системное мышление перестаёт быть учебной абстракцией — становится инструментом реальной работы.

---

### Сценарий П.2 — Настройка собственного сервиса (ступень 4: Дисциплинированный)

**Кто:** опытный пользователь, у которого есть специфический рабочий процесс — например, еженедельный анализ метрик.

**Что происходит:**
1. Открывает Marketplace сервисов
2. Находит «Weekly metrics summary» — но хочет изменить под свои метрики
3. Форкает сервис, меняет конфигурацию
4. Регистрирует в Coordinator с ограниченным scope: `["iwe_session_end", "day_close"]`
5. Verifier проверяет scope — подтверждает
6. Теперь сервис автоматически собирает его недельную статистику

**Ценность:** платформа становится расширяемой под конкретную работу конкретного человека.

---

### Сценарий П.3 — Работа с двух устройств (ступень 3+)

**Кто:** пользователь работает с ноутбука и планшета.

**Что происходит:**
1. С ноутбука открывает VS Code, работает 2 часа — Hermes накапливает паттерны в SQLite
2. Закрывает сессию → SQLite синхронизируется с Neon
3. На планшете открывает браузерную версию IWE
4. Hermes восстанавливает контекст: «В прошлой сессии ты работал над проектом Y. Продолжаем?»
5. Контекст актуален — нет потери состояния

**Ценность:** работа продолжается без разрыва контекста между устройствами.

<summary><b>Сценарии для Со-создателя (ступень 5)</b></summary>

### Сценарий С.1 — Разработка собственного метода (ступень 5: Проактивный)

**Кто:** опытный пользователь, который понял, что стандартный чек-лист закрытия дня не подходит для его работы.

**Что происходит:**
1. Открывает Hermes: «Хочу создать свой метод ежедневного review»
2. Hermes помогает формализовать: триггер → шаги → артефакт → критерий
3. Создаётся `DP.METHOD.USER.001` в его Persona
4. Метод начинает работать как чек-лист в day_close
5. Hermes отслеживает adoption: сколько раз применил, насколько эффективно
6. Пользователь решает поделиться с сообществом → метод публикуется в Marketplace

**Ценность:** пользователь становится автором инструментов — не просто потребителем.

---

### Сценарий С.2 — Передача знаний в сообщество (ступень 5)

**Кто:** пользователь, который освоил сложную практику и хочет помочь другим.

**Что происходит:**
1. Замечает, что его метод работы с большими проектами показал высокий success_rate у него
2. Платформа предлагает: «Хочешь оформить это как руководство для других?»
3. Он пишет гайд через IWE — Hermes помогает структурировать
4. Гайд публикуется в Knowledge Index сообщества
5. Другие пользователи на ступени 3+ могут его найти и применить
6. Автор получает feedback через community events

**Ценность:** цикл: личный рост → формализация → передача → рост сообщества.

</details>

<details>
<summary><b>Сценарии для Команды</b></summary>

### Сценарий К.1 — Общие методы в команде (Ф2+)

**Кто:** команда из 5 человек, работающая над одним продуктом.

**Что происходит:**
1. Руководитель создаёт «командное пространство» в IWE
2. Добавляет общие методы: стандарт декомпозиции задач, шаблон встречи
3. Каждый участник видит эти методы в своём Hermes (с правами read)
4. При работе над задачей — Hermes автоматически предлагает командный стандарт
5. Использование командного метода фиксируется → командная аналитика

**Ценность:** методы не остаются в голове руководителя — распределяются по всей команде.

---

### Сценарий К.2 — Онбординг нового участника (Ф2+)

**Кто:** новый человек в команде, первый день.

**Что происходит:**
1. Руководитель назначает «путь онбординга» из командного пространства
2. Hermes нового участника сразу знает: какие методы приняты, какие артефакты ожидаются
3. Первая неделя: выполняет 3 онбординговых задачи с подсказками Hermes
4. К концу недели Hermes показывает: «Ты освоил 2 из 4 командных методов»
5. Руководитель видит в аналитике: онбординг идёт в штатном режиме

**Ценность:** онбординг систематизирован — меньше нагрузки на руководителя, быстрее вхождение в поток.

</details>
