# Контуры системы (Platform Contours)

> Дополнительное архитектурное представление к модели ядер (01-kernels-model.md).
> Ядра описывают **функции** (кто что создаёт). Контуры описывают **организационные границы** (кто кого содержит и инстанцирует).

---

## 4-контурная модель

```
┌─────────────────────────────────────────────────┐
│  L1: Ecosystem                                  │
│  Платформа + сообщество + все IWE + МИМ         │
│  ┌─────────────────────────────────────────────┐│
│  │  L2: Platform (DP)                          ││
│  │  Инфраструктура + сервисы                   ││
│  │  ┌─────────────────────────────────────────┐││
│  │  │  L3: IWE Template                       │││
│  │  │  Шаблон для пользователя                │││
│  │  │  ┌─────────────────────────────────────┐│││
│  │  │  │  L4: Personal IWE                   ││││
│  │  │  │  Экземпляр пользователя             ││││
│  │  │  └─────────────────────────────────────┘│││
│  │  └─────────────────────────────────────────┘││
│  └─────────────────────────────────────────────┘│
└─────────────────────────────────────────────────┘
```

## Определения контуров

| Контур | Что | Примеры | Маппинг на ядра |
|--------|-----|---------|-----------------|
| **L1: Ecosystem** | Вся система: платформа + сообщество пользователей + их IWE + МИМ | Aisystant-Ecosystem, systemsworld.club, семинары, контент | B2 (Экосистема) |
| **L2: Platform (DP)** | Техническая инфраструктура и сервисы, к которым подключаются пользователи | Бот, MCP-серверы, Railway, Neon DB, Knowledge Index | C2 (ИТ-Платформа) |
| **L3: IWE Template** | Шаблон/формат персонального окружения, который получает пользователь | FMT-exocortex-template, CLAUDE.md, memory/, стратег-агент | Format-репо (FMT-*) |
| **L4: Personal IWE** | Конкретный экземпляр, настроенный пользователем и подключённый к Platform | CLAUDE.md пользователя, DS-strategy, личные Pack, WakaTime | A2 digital extension |

## Отношения между контурами

```
L1 Ecosystem  ──⊃──  L2 Platform      (экосистема включает платформу)
L2 Platform   ──→──  L3 Template       (платформа раздаёт шаблон)
L3 Template   ──→──  L4 Personal IWE   (шаблон инстанцируется в экземпляр)
L4 Personal   ──⟵──  L2 Platform       (экземпляр подключается к платформе)
```

**Инстанцирование:** Template → `setup.sh` → Personal IWE (fork + configure + connect).

**Обновление:** Platform → `template-sync.sh` → Template → `update.sh` → Personal IWE (standard/ файлы).

## Роли владельца (мета-роли)

Один человек может выступать в разных ролях в разных контурах:

| Мета-роль | Контур | Что делает | Governance |
|-----------|--------|-----------|-----------|
| **Разработчик экосистемы** | L1 | Стратегия, community, семинары, контент | DS-ecosystem-development |
| **Разработчик платформы** | L2 | Сервисы, инфраструктура, Pack, деплой | Pack DP, MAP.002, PROCESSES.md |
| **Разработчик шаблона** | L3 | Формат, протоколы, template-sync | FMT-exocortex-template |
| **Пользователь IWE** | L4 | Настройка под себя, работа, личные Pack | Личный CLAUDE.md, DS-strategy |

> **Важно:** Tseren сейчас выступает во всех 4 ролях. В будущем роли L1-L3 — команда, L4 — каждый пользователь.

## IntegrationGate (правило для всех контуров)

Перед установкой/интеграцией нового инструмента или агента в ЛЮБОМ контуре — **СТОП**:

1. **Тип:** инструмент или агент?
2. **Контур:** L1 (Ecosystem), L2 (Platform), L3 (Template), L4 (Personal)?
3. **Роли:** какие роли выполняет?
4. **Продукты:** что создаёт?
5. **Процессы:** в какие процессы встраивается?

Нет ответов → НЕ устанавливать. Сначала описание, потом интеграция.

| Контур | Governance интеграции |
|--------|----------------------|
| L2 (Platform) | Pack DP entity + MAP.002 сервис + PROCESSES.md |
| L3 (Template) | Описание в ONTOLOGY.md + README.md шаблона |
| L4 (Personal) | Описание в личном CLAUDE.md |

| Тир | Что доступно | Контуры |
|-----|-------------|---------|

| T5 (Administration) | + Исходный код, деплой | L2-L3 (разработка платформы/шаблона) |
