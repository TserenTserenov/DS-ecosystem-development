# JTBD-карты сегментов

## 0. Назначение документа

Связывает задачи (jobs) каждого сегмента аудитории с продуктами/методами IWE и метриками результата. Формат: «Когда я [ситуация], я хочу [запрос], чтобы [результат]».

Сегменты — из [Целевые аудитории 1.1.md](Целевые%20аудитории%201.1.md).

## 1. С1 — Индивидуальный практик

| Job (когда я…) | Запрос (я хочу…) | Продукт / метод IWE | Сценарий | Метрика результата |
|-----------------|-------------------|---------------------|----------|-------------------|
| Просыпаюсь утром и не знаю, за что хвататься | Видеть чёткий план дня с приоритетами | Day Open + DayPlan | SC.001 | План дня к 08:00, ≤3 приоритета |
| Заканчиваю неделю и не понимаю, что сделал | Видеть итоги и прогресс за неделю | WeekReport + WeekPlan | SC.002 | Еженедельный отчёт с % выполнения РП |
| Прочитал книгу / послушал лекцию | Зафиксировать знание, пока помню | Fleeting notes + Capture-to-Pack | SC.004 | Фиксация < 2 мин, extraction report ежедневно |
| Хочу развить конкретный навык | Получить персонализированный маршрут | Бот: лента знаний + марафон | SC.003 | Прогресс по Bloom-уровням, streak |
| Накопились заметки, задачи, идеи | Собрать всё в одну систему | Экзокортекс (репо + memory + протоколы) | — | Всё в одном workspace, ≤5 инструментов |
| Использую ChatGPT, но контекст теряется | ИИ, который помнит мой контекст и проекты | Claude Code + экзокортекс + ЦД | — | Контекст сохраняется между сессиями |
