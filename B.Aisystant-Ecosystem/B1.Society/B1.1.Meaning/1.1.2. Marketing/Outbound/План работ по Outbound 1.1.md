# План работ по Outbound 1.1

## 0. Назначение документа

Документ содержит план внедрения методологии outbound в три этапа:
- **Этап 1** — детальный, пилот с марафоном как входной точкой
- **Этап 2** — общий, масштабирование на все потоки
- **Этап 3** — общий, автоматизация и устойчивый ритм

---

## 1. Этап 1: Пилот с марафоном (детальный)

### 1.1 Цель этапа

Запустить минимально работающий outbound-процесс на **Потоке А (Созидатели)** с **Марафоном** как входной точкой. Получить первые конверсии, отладить процесс, выйти на устойчивый ритм.

**Марафон** — 4-дневная программа для людей, желающих выйти из состояния хаоса или тупика и начать системно двигаться вперёд. Формат с низким трением: не курс, а «начало нового стиля жизни». Репозиторий: [DS-marathon](https://github.com/alexpoaiagent-sudo/DS-marathon)

### 1.2 Критерии завершения этапа

- [ ] Проведено 50+ первых касаний
- [ ] Получено 10+ ответов (20%+)
- [ ] 5+ человек записались на марафон
- [ ] 2+ человека прошли марафон до конца
- [ ] Собрана обратная связь для следующей итерации
- [ ] Установлен еженедельный ритм (не рывками)

---

### 1.3 Задачи и рабочие продукты

#### Задача 1.1: Определить входное предложение

**Описание:** Зафиксировать, куда именно мы зовём людей через outbound.

**Рабочий продукт:**
```
📄 Рабочие продукты/01_Входное_предложение.md
```

**Содержание документа:**

**Входное предложение: Марафон «Выход из хаоса»**

| Параметр | Значение |
|----------|----------|
| Формат | 4 встречи |
| Для кого | Люди в состоянии хаоса/тупика, ищущие системный подход |
| Результат | Определено текущее состояние, установлен минимальный ритм, первые конкретные шаги |
| Трение на входе | Низкое — не нужно сразу платить/обязываться на долгий срок |
| Следующий шаг после марафона | Диагностика → Резидентура / Сопровождение |

**Альтернативные входные предложения (на будущее):**
- 15–20 минут диагностики
- Мини-практикум
- 3 вопроса для самодиагностики

**Содержание документа:**

| Параметр | Значение |
|----------|----------|
| Поток | А — Созидатели |

3. Нетворк основателей — 1–2 круги контактов
4. Участники профильных событий

**Количество контактов в пилоте:** 50–100

#### Задача 1.5: Разработать сценарий «Взаимодействие → Марафон»

**Описание:** Создать последовательность касаний для основного сигнала.

**Рабочий продукт:**
```
📄 Рабочие продукты/05_Сценарий_марафон.md
```

**Содержание документа:**

**Сценарий: Взаимодействие с контентом → Приглашение на марафон**

**Триггер:** Лайк/коммент к посту о системном мышлении, планировании, выходе из хаоса.

**Последовательность касаний:**

| День | Канал | Действие |
|------|-------|----------|
| 1 | LinkedIn | Первое сообщение (см. шаблон ниже) |
| 4 | LinkedIn | Повторное касание с артефактом (чеклист, 1 страница) |
| 7 | Email (если есть) | Развёрнутое письмо с кейсом/отзывом |
| 10 | LinkedIn | «Прощальное» сообщение |

**Шаблон первого сообщения:**

```
Привет, [Имя]!

Заметил твою реакцию на пост о [тема]. Похоже, тебе близка идея [конкретная идея].

Мы запускаем 4-дневный марафон для тех, кто хочет выйти из хаоса/тупика
и начать двигаться системно — не мотивационные качели, а устойчивый ритм.

Хочешь — пришлю 3 вопроса для самодиагностики или сразу ссылку на запись?
```

**Шаблон повторного касания:**

```
[Имя], отправляю мини-чеклист «3 признака, что пора менять подход» —
может быть полезно, даже если марафон пока не в планах.

[ссылка на PDF]

Если захочешь обсудить — напиши.
```

**Шаблон «прощального» сообщения:**

```
[Имя], не буду больше отвлекать.

Если тема станет актуальной позже — напиши, я на связи.
Или скажи, кому из знакомых это могло бы подойти.
```

**Цель сценария:** не «продать», а получить разговор и понять, подходит ли человек.

#### Задача 1.6: Настроить инструменты и процесс

**Описание:** Подготовить минимальный стек и зафиксировать ритм работы.

**Рабочий продукт:**
```
📄 Рабочие продукты/06_Инструменты_и_ритм.md
```

**Минимальный стек:**

| Функция | Инструмент |
|---------|------------|
| Реестр контактов | Google Sheets / Notion |
| Отправка сообщений | LinkedIn вручную |
| Шаблоны | Markdown-файлы |
| Календарь записи | Calendly |
| Хранилище артефактов | Google Drive / Notion Public |

**Еженедельный ритм:**

| День | Действие | Время |
|------|----------|-------|
| Пн | Собрать 30–50 новых контактов, разнести по уровням | 1–1.5 ч |
| Вт–Чт | Отправить 10–15 сообщений в день | 30–45 мин/день |
| Пт | Обработать ответы, назначить созвоны, обновить статусы | 30 мин |
| Пт | Ретроспектива: что сработало, что изменить (один элемент!) | 30 мин |

**Еженедельные вопросы для ретроспективы:**
1. Какой сегмент/сигнал дал лучший ответ?
2. Какая первая строка/предложение сработали?
3. На каком шаге «сыпется» (нет ответов / не записываются / не доходят)?
4. Что изменить на следующей неделе (ОДИН элемент)?

## 2. Этап 2: Масштабирование (общий)

### 2.1 Цель этапа

Расширить outbound на все три потока и увеличить объём операций при сохранении ритма.

### 2.2 Ключевые задачи

1. **Расширить Поток А** — добавить уровень 3 (широкий интерес)
2. **Запустить Поток Б (Провайдеры)** — адаптировать сценарии под привлечение наставников и методологов
3. **Запустить Поток В (Партнёры)** — разработать сценарии для компаний и организаций
4. **Стандартизировать процесс** — создать руководство для операторов
5. **Передать операции** — обучить ассистента/оператора
6. **Внедрить CRM** — перейти на полноценную систему

### 2.3 Рабочие продукты этапа

- Руководство оператора привлечения
- Матрица уровней для всех потоков
- Библиотека сценариев (5–10 штук)
- Шаблоны сообщений для всех потоков
- Панель метрик в CRM

### 2.4 Критерии перехода к Этапу 3

- Стабильный поток 50+ касаний в неделю
- Доля ответов 20%+, доля встреч 5%+
- Работают 3 потока параллельно
- Процесс документирован и передаваем
- Ритм устойчивый (не рывками)

---

## 3. Этап 3: Автоматизация и устойчивый ритм (общий)

### 3.1 Цель этапа

Автоматизировать рутину, оптимизировать конверсии, построить предсказуемый поток входов.

### 3.2 Ключевые задачи

1. **Автоматизировать сбор сигналов** — интеграции с LinkedIn, сайтом, Telegram
2. **Внедрить ИИ-ассистента** — генерация черновиков сообщений, оценка контактов
3. **Настроить автоматические последовательности** — email + LinkedIn
4. **Оптимизировать сценарии** — тестирование вариантов сообщений
5. **Интегрировать с экосистемой** — связь с путём созидателя
6. **Построить модель оценки** — прогнозирование вероятности конверсии

### 3.3 Рабочие продукты этапа

- Интеграции с источниками сигналов
- ИИ-агент квалификации
- Автоматические последовательности
- Результаты тестирования вариантов
- Модель оценки контактов

### 3.4 Критерии успеха

- Автоматизировано 70%+ рутинных операций
- Доля встреч 10%+
- Стоимость привлечения снижена на 30%+
- Поток масштабируется без линейного роста команды
- Каждую неделю — новые контакты и касания (устойчивый ритм)

---

## 4. Критерии «Outbound развёрнут»

По итогам всех этапов outbound считается развёрнутым, если выполняются 6 условий:

| # | Критерий | Этап |
|---|----------|------|
| 1 | Есть 2–4 чётких входных предложения (куда зовёте) | 1 |
| 2 | Есть реестр контактов с уровнями | 1 |
| 3 | Есть 3–5 источников сигналов | 1 |
| 4 | Есть минимум 2 последовательности касаний | 2 |
| 5 | Есть правила статусов и еженедельная ретроспектива | 1 |
| 6 | Установлен ритм: каждую неделю новые контакты и касания | 1–2 |
