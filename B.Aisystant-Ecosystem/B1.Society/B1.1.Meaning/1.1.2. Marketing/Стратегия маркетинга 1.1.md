# Стратегия маркетинга IWE

## 0. Назначение документа

Маркетинговая стратегия экосистемы IWE: какие идеи доносим, кому, через какие каналы и с какой частотой. Отвечает на НЭП #1 (Масштабирование) и #4 (Признание).

Опирается на: [Целевые аудитории](Целевые%20аудитории%201.1.md), [JTBD-карты](JTBD-карты%20сегментов%201.1.md).

## 1. Позиционирование

**IWE (Intellectual Work Environment)** — среда для организации интеллектуальной работы. IDE — для кода. IWE — для мышления.

**Одно предложение:** IWE помогает собрать знания, планы и действия в единую систему, усиленную ИИ, — чтобы развиваться осознанно, а не хаотично.

**Ключевое отличие от конкурентов:**
- Не ещё один таск-менеджер (Notion, Obsidian) — а **среда**, где знания связаны с действиями
- Не ещё один ИИ-чатбот — а **экзоскелет для мышления**, который усиливает, а не заменяет
- Не курс — а **система непрерывного развития** с персонализацией

## 2. Ключевые идеи по сегментам

### С1. Индивидуальный практик — «Наведи порядок в голове»

**Главная идея:** Ты уже много знаешь и делаешь. Проблема не в знаниях — в том, что они не собраны в систему. IWE — это твоя личная система, где всё связано.

**Подидеи:**
1. **«Утро без хаоса»** — ИИ собирает твой план дня из задач, календаря и приоритетов. Ты просыпаешься и знаешь, за что браться.
2. **«Знания не теряются»** — Записал заметку в Telegram за 30 секунд → она автоматически попадает в систему, связывается с проектами и знаниями.
3. **«Не просто читать, а применять»** — Прочитал книгу → персональный маршрут применения: задания, проверка, обратная связь от ИИ.
4. **«Один workspace — вместо пяти приложений»** — Заметки, планы, знания, ИИ-ассистент — всё в одном месте. ИИ помнит контекст между сессиями.

**Тон:** дружелюбный, практичный. Без терминологии. Показывать результат, не архитектуру.

**Триггерные ситуации:** начало года («пора навести порядок»), выгорание («бегу, но не продвигаюсь»), новый проект («нужна система»).

---

### С2. Методолог / наставник — «Масштабируй свою экспертизу»

**Главная идея:** Ты не можешь клонировать себя. Но можешь создать систему, которая доставляет твои знания и отслеживает прогресс — системно, а не вручную.

**Подидеи:**
1. **«Видь прогресс, а не самоотчёты»** — Цифровой двойник ученика показывает реальное применение знаний, не субъективные оценки.
2. **«Один метод → сто учеников»** — Загрузи свою методику в Pack → бот доставляет её каждому ученику персонально.
3. **«Наставник фокусируется на главном»** — Рутина (доставка контента, проверка ДЗ, напоминания) автоматизирована. Ты — на том, что требует человека.

**Тон:** профессиональный, ориентированный на результат. Кейсы и метрики.

**Триггерные ситуации:** «не могу взять больше клиентов», «даю знания, но люди не применяют», «нет системы сопровождения».

---

### С3. Техлид / архитектор — «IDE для интеллекта»

**Главная идея:** Ты строишь системы на работе. Почему твоя личная интеллектуальная работа организована хуже, чем твой код? IWE — инженерный подход к мышлению.

**Подидеи:**
1. **«Экзокортекс: версионируй знания как код»** — Git для знаний. Pack-репо. Онтология. Context engineering для ИИ.
2. **«АрхГейт для жизненных решений»** — Те же принципы оценки архитектуры (масштабируемость, эволюционируемость) — для решений за пределами кода.
3. **«ИИ, который усиливает мышление, а не заменяет»** — Экзоскелет, не протез. Claude Code + MCP + memory = ИИ знает твой контекст и помогает думать, а не думает за тебя.
4. **«От заметки до Habr-поста за день»** — Конвейер публикаций: зафиксировал знание → адаптировал → опубликовал.

**Тон:** технический, конкретный. Архитектурные диаграммы, код, сравнения с инженерными практиками.

**Триггерные ситуации:** «попробовал Claude/Cursor, хочу больше», «знания по проекту в голове, а не в системе», «хочу делиться знаниями».

---

### С4. Организация / команда — «Развитие, которое работает» (горизонт)

**Главная идея:** Обучение без системы применения — выброшенные деньги. IWE связывает знания с действиями и показывает реальный прогресс команды.

**Подидеи:**
1. **«ИПР, который не формальность»** — На основе реальных данных, не галочек.
2. **«Метрики развития»** — Не «прошёл курс», а «применяет знание на уровне X».
3. **«Онбординг за дни, не за месяцы»** — Новый сотрудник получает персональный маршрут в контексте команды.

**Тон:** бизнес-ориентированный. ROI, метрики, сравнение с текущими затратами на L&D.

> **Примечание:** С4 — горизонт 2026-2027. Контент создаётся по мере готовности B2B-инфраструктуры.

## 4. Контент-конвейер (content atomization)

```
Событие / результат / инсайт
       ↓
Fleeting note (< 2 мин)
       ↓
DS-Knowledge-Index (source-of-truth текст)
       ↓
   ┌───┴───┐───────┐───────┐
   TG      Discourse  Habr   LinkedIn
  (тизер)  (полный)  (техн.) (проф.)
```

**Принцип:** Один текст — несколько адаптаций. Source-of-truth живёт в Knowledge Index. Адаптация меняет тон и глубину, не суть.

| Формат адаптации | Канал | Длина | Особенности |
|-----------------|-------|-------|-------------|
| Тизер | TG | 3-5 предложений | Крючок + ссылка на полный текст |
| Полный пост | Discourse | 500-1500 слов | Структура, примеры, ссылки |
| Техническая статья | Habr | 1500-3000 слов | Код, диаграммы, архитектура |
| Профессиональный пост | LinkedIn | 200-500 слов | Инсайт + CTA |
