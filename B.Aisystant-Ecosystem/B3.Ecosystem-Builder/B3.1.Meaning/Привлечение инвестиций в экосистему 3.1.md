# Привлечение инвестиций в экосистему

## 0. Назначение документа

Этот документ описывает **стратегию и набор документов**, необходимых для привлечения инвестиций в экосистему развития. Служит руководством для подготовки к раундам финансирования и взаимодействия с инвесторами.

**Адресаты:**
- Основатели и команда фандрайзинга
- Финансовый директор (CFO)
- Юридическая команда
- Консультанты по привлечению инвестиций

- Полный чек-лист документов для инвесторов

### 1.2 Инвестиционный тезис

**Почему инвестировать в платформу Aisystant:**

1. **Рынок:** Глобальный EdTech $250B+ (2025), AI-enhanced learning — самый быстрорастущий сегмент. Lifelong learning — основной тренд (от «курсов» к «среде развития»)
2. **Провал рынка:** MOOC completion 5-10%, transfer-to-practice <15%. ИИ обесценивает контент — выживает тот, кто создаёт **инфраструктуру практики**
3. **Решение:** Платформа развития интеллекта — открытый контент + платная практика (модель Red Hat). Персонализация через ЦД + ИИ, качество 1-on-1 наставничества по цене MOOC ($10-30/мес)
4. **7 структурных преимуществ:** FPF-методология (10+ лет IP), ЦД (switching costs), open-content модель (конкурент не может повторить), ИИ-персонализация, token flywheel, multi-surface, Global Core + Local Edge — см. [[Конкурентное преимущество и бизнес-модель 2.1]]
5. **Defence stack:** 5 уровней защиты — Product → Business Model → Methodology IP → Data Moat → Community
6. **Архитектура готова:** 3-слойная платформа, 12 принципов, 4-фазный roadmap (WP-73), ЭМОГССБ 8.3/10
7. **Юнит-экономика:** LTV/CAC 20-50x, gross margin 70-85%, payback 1-2 мес

### 1.4 Куда пойдут инвестиции (Use of Funds)

> Связь с roadmap: WP-73 (4 фазы, 31 подсистема). Seed-раунд покрывает Фазы 0-1 + запуск Фазы 2.

**Шесть направлений:**

| # | Направление | Доля | Что входит | Связь с roadmap |
|---|-------------|------|-----------|-----------------|
| 1 | **Разработка платформы** | 40% | Web App MVP, ORY SSO, Event Bus, ЦД Event Store, Billing Service, мобильная адаптация | Фаза 0 + Фаза 1 (WP-73) |
| 2 | **Команда** | 25% | Frontend (React/Next.js), Backend (Node.js), DevOps, Product Manager | Capacity для Фаз 0-2 |
| 3 | **Маркетинг и продажи** | 20% | Семинары/вебинары, контент-маркетинг (SEO), партнёрства с организациями, первые корпоративные пилоты, community building | Воронка T1→T2→T3 |

### 3.2 Детальное содержание ключевых слайдов

#### Слайд «Проблема»

**Содержание:**
- MOOC completion rate 5-10% → 90%+ бросают, transfer-to-practice <15%
- ИИ обесценивает контент: ChatGPT/Claude объяснят любую тему бесплатно
- $250B EdTech-рынок продаёт **доступ к контенту**, но контент не меняет поведение
- Коучинг персонализирован ($300+/час) но не масштабируется. MOOC масштабируется но не персонализирован
- **Пустующая ниша:** персонализированная среда непрерывного развития, масштабируемая через ИИ

#### Слайд «Решение»

**Содержание:**
- **Платформа развития интеллекта** (не курсы, не LMS): открытый контент + платная практика
- Два ключевых актива:
  1. **Цифровой двойник (ЦД)** — 5 проекций когнитивного профиля (skill_mastery, memory_decay, engagement, misconceptions, qualifications)
  2. **ИИ-Проводник** — строит маршрут из данных ЦД, не из «средней» траектории
- Результат: качество 1-on-1 наставничества по цене MOOC ($10-30/мес)
- Визуальная схема: бесплатно (контент) → платно (практика + ИИ + наставник + ЦД)

#### Слайд «Продукт»

**Содержание:**
- **Web App** — «банковское приложение для развития»: дашборд ЦД, обучение, рабочая тетрадь
- **TG-бот** — быстрый канал (90M пользователей в RU), тренажёр, ИИ-ассистент
- **ИИ-агенты**: Проводник (маршрут), ДЗ-чекер (мгновенная проверка), Консультант (вопросы)
- **Multi-surface**: один бэкенд → Web App, TG-бот, CLI, Discord
- Скриншоты: дашборд ЦД, рабочая тетрадь, панель наставника

### 4.2 Unit Economics

**Ключевые метрики:**

| Метрика | Формула | Целевое значение |
|---------|---------|------------------|
| **LTV** | ARPU × Avg Lifetime | >$X |
| **CAC** | Marketing Spend / New Customers | <$Y |
| **LTV/CAC** | LTV / CAC | >3x |
| **Payback Period** | CAC / Monthly ARPU | <12 мес |
| **Gross Margin** | (Revenue - COGS) / Revenue | >70% |

## 8. Term Sheet и условия

### 8.1 Ключевые параметры Term Sheet

| Параметр | Описание | На что смотреть |
|----------|----------|-----------------|
| **Valuation** | Pre-money оценка | Адекватность рынку и метрикам |
| **Investment Amount** | Размер инвестиции | Достаточность для milestones |
| **Instrument** | SAFE / Convertible / Equity | Простота vs. защита |
| **Discount** | Скидка при конвертации | Стандарт 15-25% |
| **Valuation Cap** | Потолок оценки при конвертации | Не слишком низкий |
| **Pro-rata Rights** | Право участия в следующих раундах | Стандартно для лидов |
| **Board Seats** | Места в совете директоров | Не больше 1 на seed |
| **Liquidation Preference** | Приоритет при ликвидации | 1x non-participating стандарт |
| **Anti-dilution** | Защита от размытия | Broad-based weighted average |
| **Vesting** | Vesting основателей | 4 года с 1 год cliff |

### 8.2 Красные флаги в Term Sheet

**Избегать:**
- [ ] Liquidation preference >1x
- [ ] Full participating preferred
- [ ] Full ratchet anti-dilution
- [ ] Excessive board control
- [ ] Pay-to-play provisions (на ранних стадиях)
- [ ] Redemption rights
- [ ] Cumulative dividends

### 8.3 Шаблон SAFE (для ранних стадий)

**Параметры Y Combinator SAFE:**
- Valuation Cap: $X
- Discount: Y% (опционально)
- Pro-rata: Да/Нет
- MFN: Да/Нет

## 10. Специфика привлечения для экосистемы

### 10.1 Уникальные аспекты pitch'а

**Что подчеркивать:**

1. **Экономика вклада:**
   - Токеномика как инструмент engagement и retention
   - Снижение CAC через viral loops
   - Дополнительный revenue stream

2. **Цифровой двойник:**
   - Data moat — накопление данных о траекториях
   - Персонализация как конкурентное преимущество
   - Переносимость и lifetime value

3. **Network effects:**
   - Больше участников → больше контента → больше ценности
   - Peer-to-peer learning и менторство
   - Marketplace динамика

4. **Impact story:**
   - Цивилизационная амбиция
   - Демократизация образования
   - Культурная трансформация

### 10.2 Риски и митигации

| Риск | Митигация | Как представлять |
|------|-----------|------------------|
| Регуляторные риски токена | Utility token, не security; юридическая экспертиза | Показать legal opinion |
| Зависимость от ИИ-провайдеров | Multi-provider стратегия; собственные модели | Архитектурная гибкость |
| Конкуренция с EdTech гигантами | Нишевая дифференциация; community moat | Позиционирование |
| Сложность product-market fit | Итеративная валидация; метрики engagement | Текущий traction |

### 10.3 Метрики для EdTech/Community инвесторов

**Engagement метрики:**
- DAU/MAU ratio
- Time spent per session
- Content creation rate
- Peer interaction rate

**Learning outcomes:**
- Completion rate
- Transfer to practice rate
- Certification rate
- NPS

**Community health:**
- Active contributors %
- Mentor-to-mentee ratio
- Referral rate
- Churn by cohort
