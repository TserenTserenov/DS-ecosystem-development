<summary><b>1. Стратегия привлечения</b></summary>

## 1.1 Текущая стадия

**Первый раунд.** 11 лет на рынке, 500 подписчиков, $240K ARR @ $0 CAC, 0 внешних инвестиций. Одна подписка БР (T2–T4, цена растёт постепенно) + LLM-биллинг (отдельный поток: базовый лимит в подписке, превышение оплачивается отдельно). Ищем капитал для международного масштабирования и роста ARPU. Структура: priced equity round, Delaware C-Corp. Размер и оценка — предмет обсуждения. При согласовании условий хотели бы обсудить возможность founder secondary (11 лет bootstrapping без внешнего капитала).

**Путь пользователя (T0→T4):** /start в боте (5 мин) → регистрация email (5 мин) → подписка БР (5 мин) → заполнение ЦД (20 мин, включено) → GitHub OAuth + Gateway URL (5 мин, включено). Около 40 минут — от Telegram до полной рабочей среды IWE.

**Принципы:**
- Пользователь — хозяин своих данных (GitHub, может забрать и уйти)
- Одна подписка = все тиры T2–T4 (нет отдельных платных уровней)
- ИИ как сервис: базовый лимит в подписке, LLM-биллинг сверх лимита — отдельный поток дохода
- Минимальный барьер: нетехническому пользователю не нужно знать git/код

## 1.2 Ценностное предложение для инвестора

**Тезис:** Anthropic и OpenAI строят интеллект для машин. Мы строим интеллект для людей. Это не EdTech — это Intelligence Platform.

**Почему сейчас:**
1. AI обесценил контент — ценность сместилась к мышлению и методологии
2. 30-летняя методология (FPF) + AI = экзоскелет интеллекта, не замена
3. Организация знаний (Pack-архитектура) делает AI на порядок точнее
4. Модель «Red Hat интеллекта» — контент открыт, среда и практика платные
5. 11 лет organic traction доказывают PMF

**Unfair advantage:**
- Методология FPF не воспроизводится за деньги (30+ лет, 5400+ текстов)
- Open-content моат — конкурент не откроет контент без потери бизнеса
- Data moat через цифровой двойник — switching cost растёт с каждым месяцем

<summary><b>3. IP/Legal структура</b></summary>

## 3.1 Кому принадлежит IP

| Актив | Текущий владелец | Целевая структура |
|---|---|---|
| **Методология FPF** (5400+ текстов) | Основатель-методолог (физ. лицо) | Лицензия от автора → юрлицо (exclusive, perpetual) |
| **Код платформы** (LMS, бот, IWE) | Команда разработки (физ. лица) | IP assignment → юрлицо. Work-for-hire agreement |
| **Контент курсов** | Наставники + основатели | Creative Commons (открытый) + платная инфраструктура |
| **Pack-знания** (доменные пакеты) | Сообщество + основатели | Open source (MIT/CC BY-SA). Моат — не контент, а структура |
| **Бренд «Aisystant»** | Не зарегистрирован | Зарегистрировать ТМ в ключевых юрисдикциях |
| **Данные пользователей** (ЦД) | Платформа | GDPR/CCPA compliance. Data Processing Agreement |

## 3.2 Юридическое лицо

**Текущее состояние:** ИП (РФ). Недостаточно для привлечения инвестиций.

**Варианты:**

| Вариант | Плюсы | Минусы | Рекомендация |
|---|---|---|---|
| **Delaware C-Corp** | Стандарт для VC, SAFE/priced equity, cap table tools | Дорого в обслуживании, double taxation | ✅ Рекомендован для seed |
| **Singapore PTE LTD** | Азиатский хаб, налоговые льготы, ASEAN | Менее привычно для US VC | Альтернатива |
| **Estonia e-Residency** | EU access, дешёво, digital-first | Ограниченно для крупных раундов | Промежуточный вариант |
| **ООО (РФ)** | Просто | Санкции, ограничения для иностранных инвесторов | ❌ Не рекомендован |

**Следующие шаги:**
1. Консультация с юристом (Stripe Atlas / Firstbase для Delaware)
2. IP assignment agreements от всех контрибьюторов
3. Регистрация бренда
4. Founder vesting agreement (4 года, 1 год cliff)

## 3.3 Лицензирование open-source

| Компонент | Лицензия | Стратегия |
|---|---|---|
| FPF-Spec.md | Open (CC BY-SA или MIT) | Открытая методология = моат (конкурент не скопирует без 30 лет опыта) |
| Pack-шаблоны (SPF) | Open (MIT) | Формат открыт, содержание — работа участника |
| Код LMS | Проприетарный | SaaS, не распространяется |
| Код бота | Проприетарный | SaaS |
| IWE (экзокортекс) | Open (шаблон FMT) + проприетарный (платформа) | Шаблон = open. Hosted platform = платная |

## 3.4 Compliance

| Требование | Статус | Действие |
|---|---|---|
| **GDPR** | Частично | Data Processing Agreement, Privacy Policy |
| **CCPA** | Нет | При выходе на US рынок |
| **ToS** | Нет | Создать Terms of Service |
| **Privacy Policy** | Нет | Создать |
| **Cookie Policy** | Нет | При запуске Web App |
| **COPPA** | N/A | Целевая аудитория 18+ |
