**Версия 1.0 от 11 мая 2026 г.**

**Оператор:** ИП Церенов Цырен Иванович (ОГРНИП — указать).  
**Контакт по защите данных (DPO):** info@system-school.ru.

---

## 2. Какие данные мы собираем

### 2.1. Идентификаторы (обязательно для работы сервиса)

| Данные | Источник | Назначение |
|--------|---------|-----------|
| `telegram_id`, `first_name`, `last_name` | Telegram | Идентификация пользователя в боте |
| `ory_id` (UUID) | Ory Hydra OAuth | Кросс-платформенная идентификация (Web, MCP) |
| `email` (опционально) | Регистрация подписки | Связь по подписке, чеки |

### 2.2. Данные интеграций (с явного согласия — OAuth)

| Интеграция | Что собираем | Хранение |
|-----------|-------------|---------|
| **GitHub OAuth** | `github_user_id`, `username`, список выбранных репозиториев | До отзыва (через /disconnect) |
| **Google Calendar** | `refresh_token`, события календаря (для Day Open) | До отзыва |
| **WakaTime** | Время кодирования по проектам | До отзыва |
| **Активность в IWE** | Что и когда делал (для прогресса развития) | 90 дней (rolling) |

### 2.3. Контент пользователя

| Данные | Назначение | Хранение |
|--------|-----------|---------|
| Сообщения боту | Q&A с консультантом | 30 дней (история сессии) |
| Загруженные документы (личные) | Индексация для /personal-mcp | До удаления пользователем |
| Личные репозитории (через GitHub App) | База знаний для руководства | До отзыва доступа |

### 2.4. Платёжные данные

- Реквизиты карт **не хранятся** у нас. Платежи через ЮKassa (PCI DSS у них).
- Мы храним: `transaction_id`, `amount`, `status`, `telegram_id` (binding) — 7 лет (налоговое требование РФ).

## 3. Согласие на обработку через AI

**3.1. Базовая обработка** (Claude Anthropic): сообщения боту обрабатываются через Claude API для генерации ответов. Anthropic — partner с DPA (Data Processing Agreement). Anthropic не использует наши данные для обучения моделей.

**3.2. Эмбеддинги через OpenAI:** при загрузке личных документов через `/scaffold` они индексируются через OpenAI Embeddings API. Это требует **явного согласия** при первом использовании (галочка в боте). Без согласия — `/scaffold` недоступен.

**3.3. Локальные модели:** в roadmap (опция для пользователей с повышенными требованиями приватности).

## 5. Кому передаём данные

**Передаём:**
- **Anthropic Claude** — содержимое сообщений (с DPA, не используется для обучения)
- **OpenAI Embeddings** — содержимое личных документов (только с явного согласия §3.2)
- **YooKassa** — данные транзакций (необходимо для платежей)
- **Налоговые органы РФ** — суммы и даты транзакций (по требованию)

**Не передаём** третьим лицам в рекламных или маркетинговых целях.

## 6. Ваши права (152-ФЗ + GDPR baseline)

### 6.1. Право на доступ
- Команда `/mydata` в боте → JSON с вашими данными

### 6.2. Право на исправление
- `/profile-edit` в боте

### 6.3. Право на удаление (Right to be Forgotten)
- `/mydata-delete` в боте → запускается процесс удаления (до 30 дней)
- Что удаляется: персональные идентификаторы, контент, история. Что остаётся: агрегированная статистика (без привязки к личности), платёжные записи (требование ФНС, 7 лет).

### 6.4. Право на отзыв согласия
- Любая интеграция: `/integrations` → отключить
- Подписка: `/unsubscribe`
- Согласие на OpenAI: можно отозвать в `/privacy-settings`

### 6.5. Право на жалобу
- Внутри: info@system-school.ru
- Регулятор РФ: Роскомнадзор

## 8. Безопасность

Меры защиты:
- TLS in-transit (HTTPS, WSS)
- AES-256 at-rest (Neon)
- Row-Level Security (изоляция данных пользователей в БД)
- JWT-верификация (Ory) на каждом запросе
- Rate limiting (защита от brute force и DDoS)
- Дневной аудит безопасности
